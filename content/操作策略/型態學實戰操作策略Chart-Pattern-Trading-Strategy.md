# 型態學實戰操作策略Chart-Pattern-Trading-Strategy

> 型態辨識只是起點，操作策略才是關鍵——目標價計算、進出場時機、成交量確認、指標共振、六大誤區避坑，型態學從「看圖說故事」升級為可執行的交易系統

## 核心概念

型態學的價值不在於辨識型態（那是基本功），而在於：

1. **什麼時候進場**——型態完成確認的時機點
2. **進場後目標在哪**——測量法則計算目標價
3. **怎麼確認型態有效**——成交量與技術指標共振
4. **什麼情況下型態會失敗**——六大誤區與避坑方法

只會辨識型態但不會操作，等於看到紅燈但不會煞車。本篇把型態學從「辨識」升級到「操作」。

## 型態分類速查表

| 類型 | 型態 | 方向意義 | 操作策略 |
|------|------|----------|----------|
| **底部反轉** | W底、頭肩底、三重底、圓弧底、V型底 | 空轉多 | 突破頸線做多 |
| **頂部反轉** | M頭、頭肩頂、三重頂、圓弧頂、V型反轉（空） | 多轉空 | 跌破頸線做空/減碼 |
| **中繼型態** | 旗形、三角收斂、箱型、楔形 | 延續原趨勢 | 順勢突破加碼 |

## 目標價計算法則

型態學最重要的實戰工具之一是**測量法則**：型態的幅度等於突破後的目標幅度。

### 頭肩型態目標價

```
頭肩頂目標價 = 頸線 - (頭部 - 頸線)
頭肩底目標價 = 頸線 + (頸線 - 頭部)
```

**範例**：頸線100元，頭部120元
- 頭肩頂目標價 = 100 - (120 - 100) = **80元**
- 頭肩底目標價 = 100 + (100 - 80) = **120元**（底部80元）

### W底/M頭目標價

```
W底目標價 = 頸線 + (頸線 - 底部)
M頭目標價 = 頸線 - (頭部 - 頸線)
```

**範例**：頸線100元，底部80元
- W底目標價 = 100 + (100 - 80) = **120元**
- M頭目標價（頭部120元）= 100 - (120 - 100) = **80元**

### 三角收斂目標價

```
目標價 = 突破點 ± 三角形底邊寬度
```

### 重要提醒

- 目標價是**最小目標**，實際走勢可能超過
- 目標價是**機率參考**，不是保證
- 搭配 [[支撐壓力]] 和 [[斐波那契擴展Fibonacci-Extension]] 交叉驗證更可靠

## 進場時機四步確認法

型態辨識出來後，不要急著進場。遵循四步確認：

### Step 1：型態完成確認

- **底部型態**：價格突破頸線（帶量）
- **頂部型態**：價格跌破頸線（帶量）
- **中繼型態**：順原趨勢方向突破

### Step 2：成交量確認

| 型態 | 正確量能 | 危險訊號 |
|------|----------|----------|
| W底突破頸線 | 爆量突破 | 無量突破（假突破風險高） |
| M頭跌破頸線 | 帶量跌破 | 無量跌破（可能假跌破） |
| 頭肩底突破 | 右肩量縮+突破爆量 | 左肩量最大+突破無量 |
| 頭肩頂跌破 | 左肩量最大+跌破帶量 | 突破無量 |

**核心原則**：突破必須帶量確認！無量突破是假突破的最大特徵。參考 [[成交量確認原則Volume-Confirmation]]

### Step 3：回測確認

- 型態突破後，價格常會回測頸線
- **底部型態**：回測頸線不破 → 更穩的買點
- **頂部型態**：反彈回頸線不過 → 更穩的賣點
- 回測確認的進場點雖然晚一點，但勝率明顯提高

### Step 4：技術指標共振

型態突破搭配指標共振，勝率從 40-50% 提升到 65-70%：

| 指標 | 底部型態確認 | 頂部型態確認 |
|------|-------------|-------------|
| [[MACD指標實戰判讀]] | 黃金交叉 | 死亡交叉 |
| [[背離Divergence進階實戰]] | RSI<30 低檔反轉 | RSI>70 超買反轉 |
| [[均線判斷]] | 站上季線/半年線 | 跌破季線/半年線 |
| [[量價關係九種型態高低檔判讀]] | 突破帶量+回測量縮 | 跌破帶量+反彈量縮 |

## 底部型態選股實戰SOP

底部型態是 CANSLIM 選股法中技術面最重要的特徵。歐尼爾說過：「幾乎大漲的強勢股都來自於紮實的底部型態。」

### 選股三步驟

1. **掃描底部型態**：尋找 W底、頭肩底、三重底、V型底、盤整底
2. **確認量價結構**：左腳帶量、右腳量縮、突破爆量
3. **等待突破確認**：帶量突破頸線才進場，不猜測

### 進場時機優先順序

| 優先順序 | 進場點 | 風險 | 報酬 |
|----------|--------|------|------|
| 1 | 回測頸線不破 | 低 | 高 |
| 2 | 突破頸線當下 | 中 | 高 |
| 3 | 右腳形成時（預判） | 高 | 最高 |

### 停損設定

- **頸線下方 2-3%** 設停損
- 如果突破後隔天跌破頸線 → 假突破，立即出場
- 參考 [[假突破三道過濾]]

## 型態學六大誤區

### 誤區一：強行對號入座

**問題**：投資人主觀地將價格走勢套用成特定型態，即使圖形並不完整。

**案例**：某投資人觀察個股連續三天拉回後反彈，即判斷為W底型態，但忽略兩次低點間無明顯頸線、也無足夠時間對稱性，進場後發現只是反彈結束、趨勢繼續下跌。

**對策**：型態必須完整且對稱，不完整的型態不要強行套用。

### 誤區二：忽略成交量確認

**問題**：只看價格突破型態，不確認成交量。

**案例**：W底突破頸線看似成功，但成交量並未放大，隔天即被大單拉回，出現假突破，投資人只能急忙停損。

**對策**：所有反轉型態的突破都必須帶量確認，無量突破一律視為假突破。參考 [[成交量確認原則Volume-Confirmation]]

### 誤區三：單一型態孤立判斷

**問題**：僅依型態做進出場決策，忽略整體趨勢、技術指標、消息面。

**案例**：個股呈現明顯頭肩底型態，技術型態完整，投資人據此買進，但忽略了大盤正處空頭趨勢、重要經濟指標疲軟，導致型態無效、價格持續破底。

**對策**：型態必須結合大盤趨勢、[[多時間框架分析]]、籌碼面、消息面綜合判斷。參考 [[三面合一投資系統整合技術面籌碼面基本面]]

### 誤區四：忽略時間與比例對稱性

**問題**：有效的型態需要一定的時間與空間比例，左右肩對稱、W底兩低點間距合理。

**案例**：某個股一日內劇烈下跌再上漲，形成類似頭肩底型態，但左肩僅為5分鐘K線、右肩卻為60分鐘K線，比例失衡，行情迅速反轉下跌。

**對策**：型態的時間對稱性和比例對稱性必須合理，變形過大的型態可信度低。

### 誤區五：忽視關鍵支撐壓力位

**問題**：型態雖成形，但突破位置剛好遇到前波壓力區（如跳空缺口、前高壓力），突破失敗機率高。

**案例**：W底突破頸線，但頸線位置正處於前高壓力帶，加上大盤轉弱，結果突破失敗反而變成「誘多陷阱」。

**對策**：型態辨識必須結合 [[支撐壓力]] 判斷，頸線位置如果有前波壓力或支撐，型態成功率會打折。

### 誤區六：過度依賴歷史重複性

**問題**：「歷史會重演」是技術分析的基礎假設，但並不代表完全複製。不同時期的市場結構、資金流向與總體經濟背景差異，皆可能影響型態的表現。

**案例**：投資人認為台股大盤出現與2008年金融海嘯前相似的M頭型態，因此提前大幅放空，結果市場並未崩盤，反而是產業輪動造成短期整理，誤判行情方向。

**對策**：型態的目標價和成功率是統計機率，不是保證。搭配 [[勝率與賺賠比WinRate-and-RiskReward]] 管理，永遠設好停損。

## 型態學與操作策略整合

### 順勢操作原則

型態完成後的操作方向應該**順著原趨勢方向**：

| 原趨勢 | 型態類型 | 操作方向 |
|--------|----------|----------|
| 上漲趨勢 | 中繼型態（旗形、三角收斂） | 順勢做多 |
| 上漲趨勢 | 頂部反轉（M頭、頭肩頂） | 反轉做空/減碼 |
| 下跌趨勢 | 中繼型態 | 順勢做空/觀望 |
| 下跌趨勢 | 底部反轉（W底、頭肩底） | 反轉做多 |

### 部位控制

參考 [[部位控制2%法則Position-Sizing-2-Percent-Rule]]：

- 型態突破進場：標準部位
- 回測確認進場：可加碼至1.5倍
- 型態失敗出場：堅決停損，不抱幻想

### 搭配 [[交易日誌與交易檢討Trading-Journal]]

每次型態操作都應記錄：
1. 型態類型與完成時間
2. 進場理由（突破/回測/指標共振）
3. 目標價與停損價
4. 實際結果與檢討
5. 型態成功率統計（建立個人型態資料庫）

## 型態學的優缺點總結

### 優點
1. **直觀好上手**：用眼睛就能辨識，不需要複雜計算
2. **提供明確交易訊號**：突破頸線就是進場點、目標價可計算
3. **適用多種商品和環境**：股票、期貨、外匯、加密貨幣都適用

### 缺點
1. **主觀性強**：同一張圖不同人可能看到不同型態
2. **事後辨識容易、事前辨識困難**：型態完成前很難確認
3. **假突破風險**：型態突破後失敗的比例不低
4. **需要搭配其他工具**：單獨使用勝率不足
