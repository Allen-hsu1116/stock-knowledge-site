# 網格交易Grid-Trading

> 把區間交易自動化——像漁網一樣，在震盪行情中自動收割每一次波動的利潤

## 核心概念

網格交易（Grid Trading）是區間交易的自動化版本，將資金分為多份，按固定價差（網格間距）在區間內分散掛單：價格下跌一定比例時買進一單位，上漲一定比例時賣出一單位，隨商品波動不斷執行。

本質是系統化的「低買高賣」——不依賴技術指標判斷進出場時機，而是用**價格本身**觸發交易。

### 與區間交易的關係

網格交易是[[區間交易策略Range-Trading]]的程式化實現：

| 面向 | 區間交易 | 網格交易 |
|------|---------|---------|
| 進場判斷 | 技術指標確認關鍵價位 | 固定價差自動觸發 |
| 操作頻率 | 較低 | 較高（每格都交易） |
| 人工介入 | 需要 | 高度自動化 |
| 適合者 | 有看盤時間 | 無法全天盯盤 |

## 實戰應用

### 網格交易設定四要素

#### 1. 網格上下限

設定價格區間的最高點和最低點。超出上限部位清空後繼續漲，就無法再獲利；跌破下限資金用盡還續跌，就會被套牢。

**設定方法**：
- 統計方法：以歷史價格分佈的合理範圍為上下限
- 支撐壓力：以明確的支撐/阻力位為邊界
- ATR 法：當前價 ± N 倍 ATR

#### 2. 網格大小（間距）

每格之間的價格差距，決定交易頻率和單次利潤。

| 標的波動性 | 建議網格大小 | 邏輯 |
|-----------|------------|------|
| 高波動（當沖熱門股） | 0.5-1% | 觸發頻率高 |
| 中波動（績優股） | 1-2% | 平衡頻率和利潤 |
| 低波動（ETF） | 0.3-0.5% | 小幅波動也要抓到 |

**原則**：網格大小 > 手續費成本（0.385%），否則白忙一場。

#### 3. 起始部位

第一次進場買進的比例，通常建議 **50%** 資金作為起始部位。

- 太少（20%）：行情上漲時賺不夠
- 太多（80%）：行情下跌時無法加碼
- 50%：進可攻退可守

#### 4. 每格交易量

每次觸發買賣的數量，影響總部位上限和風險。

- 總部位上限建議 10 單位（例：10 張）
- 每格 1 單位，漲賣 1 張、跌買 1 張
- 部位越大風險越高，需配合停損

### Python 實作範例（富果 API）

```python
# 記錄部位成本
position_list = []

# 記錄損益
profit_record = []

# 第一根 K 棒收盤價為基準價
base_price = close_price_list[0]

# 網格區間 0.5%
grid_range = 0.005

# 起始買進 5 張
n = 5
for i in range(n):
    position_list.append(close_price_list[0])

for price in close_price_list[1:]:
    # 漲 1 格且還有部位 → 賣 1 張
    if price >= base_price*(1+(1-len(position_list)+n)*grid_range) and len(position_list) > 0:
        profit_record.append((price - position_list[0])*1000)
        del position_list[0]
    # 跌 1 格且部位未滿 → 買 1 張
    elif price <= base_price*(1-(len(position_list)+1-n)*grid_range) and len(position_list) < 10:
        position_list.append(price)

# 尾盤清空所有部位
for pos in position_list:
    profit_record.append((close_price_list[-1]-pos)*1000)
```

### 實作結果（2022-09-30 元太 8069）

- 設定：網格 0.5%、起始 5 張、上限 10 張
- 不計交易成本，日內報酬 27,500 元
- 每筆交易獲利相對固定
- 上漲行情只能抓住格內獲利，超出部分無法賺取

### 進階優化方向

1. **多標的分散**：一次在多檔股票同時使用網格策略，降低單一標的風險
2. **振幅篩選**：用前一日振幅作為當日執行網格策略的依據，提高勝率
3. **動態網格**：根據即時波動率（ATR）調整網格大小，波動大時放大間距，波動小時縮小間距
4. **時間過濾**：避開開盤 15 分鐘和尾盤 15 分鐘的劇烈波動

## 注意事項

### 單邊上漲——賣光部位後踏空

當走勢由震盪轉為一段上漲趨勢時，策略會逐步出場減碼，賣完所有部位後還持續上漲，就無法在後續行情中獲利。

**這是網格交易最痛的缺點**——你賣掉的股票繼續漲，而且你已經沒有部位了。

### 單邊下跌——資金用盡被套牢

遇到一段下跌趨勢，策略會逐步進場加碼，用光所有資金後還持續下跌，就會因為資金用盡而被套牢。

**防禦**：
- 設定總停損線（如跌破下限 5% 全部出清）
- 不要把全部資金投入單一網格策略

### 網格大小選擇不當

- 太大：觸發次數少，利潤有限
- 太小：手續費吃掉利潤，且震盪不到就無法成交
- **建議**：網格大小至少是手續費的 2-3 倍（0.8-1.2%）

### 不適合的市場

- **明確趨勢市場**：單邊走勢是網格交易的剋星
- **低流動性標的**：掛單無法成交
- **極低波動標的**：波動不到網格大小，策略閒置

### 手續費侵蝕

頻繁交易的手續費累積驚人：
- 每天來回 10 次 = 手續費 3.85% 的本金
- 月累計可能吃掉 50%+ 的利潤
- **必須計算扣除手續費後的淨利**

## 網格交易成功清單

- [ ] 選擇震盪頻繁的標的（日內振幅 > 3%）
- [ ] 網格大小 > 手續費的 2-3 倍
- [ ] 設定總停損線
- [ ] 不投入全部資金
- [ ] 多標的分散執行
- [ ] 定期檢視是否從震盪轉為趨勢
- [ ] 計算扣除手續費後的淨利
