# XGBoost技術指標整合策略

> 用 XGBoost 機器學習模型強化技術指標信號，綜合多維度市場變數提升交易準確度

## 核心概念

XGBoost（eXtreme Gradient Boosting）是一種基於梯度提升決策樹（GBDT）的機器學習演算法，在量化交易領域被廣泛使用。它的核心優勢在於能綜合多個特徵（技術指標數值、價量資料等），學習高維度的非線性關係，從而優化交易信號的篩選與決策邏輯。

### 為什麼需要整合？

傳統技術指標策略有兩個核心問題：
1. **固定參數限制**：如一目均衡表使用 9-26-52 參數，無法靈活適應不同市場環境
2. **震盪行情假信號**：技術指標在盤整期容易產生虛假信號，導致頻繁交易和虧損

XGBoost 能綜合不同市場變數（價格動能、成交量變化、技術指標數值等），找出數據與未來報酬間的關聯度，從而過濾掉低品質信號。

### 整合方式

1. **純技術指標策略**：只依賴技術指標信號交易
2. **技術指標為主的混合策略**：技術指標觸發信號，ML 模型輔助判斷加減倉幅度
3. **ML 為主的混合策略**：ML 模型觸發信號，技術指標輔助確認
4. **純 ML 策略**：完全依賴 XGBoost 模型信號

## 實戰應用

### 特徵工程

將技術指標的數值作為 ML 模型的輸入特徵。以一目均衡表整合為例：
- Tenkan_sen（轉換線）、Kijun_sen（基準線）
- Senkou_Span_A、Senkou_Span_B（雲帶前緣）
- Chikou_Span（遲行線）
- 加上基本價量：開高低收 + 成交量

### 標籤定義

定義目標變數：
- 未來 N 天漲幅超過 X% → Buy（1）
- 未來 N 天跌幅超過 X% → Sell（2）
- 其餘 → Hold（0）

實例：未來 5 天漲幅 > 3% 為 Buy，跌幅 > 3% 為 Sell。

### 模型參數建議

- n_estimators=500（樹的數量）
- max_depth=5（控制樹深度防過擬合）
- learning_rate=0.05
- subsample=0.8（用 80% 數據訓練每棵樹）
- colsample_bytree=0.8
- eval_metric="mlogloss"（多類別分類）

### 部位管理邏輯

- 首次買進信號 → 50% 部位
- 雙重信號確認（技術+ML 同向）→ 加倉 30%
- 單一信號確認 → 加倉 10%
- 賣出信號 → 減倉 20%
- 槓桿限制 100%（不加槓桿不做空）

### 績效比較（6446 藥華藥實證）

| 策略 | 累積報酬率 |
|------|-----------|
| Buy and Hold | 61.24% |
| 純技術指標 | 15.44% |
| 技術為主混合 | 17% |
| ML 為主混合 | 137.52% |
| 純 ML 策略 | 141.81% |

ML 策略大幅優於純技術指標，主因是 XGBoost 能更早辨識漲勢並提早建倉。

## 注意事項

### 過擬合風險

當測試集股價突破訓練集歷史最高價時，模型會進入未知區間，可能持續輸出單一信號（如全部 Sell）。這是 ML 模型的根本限制。

### 解法

1. **設計警示線**：當股價超出訓練集區間時，停用或更換模型
2. **混合策略優先**：不要只依賴 ML 信號，搭配技術指標做綜合判斷
3. **特徵重要性觀察**：XGBoost 可輸出特徵重要性，技術指標特徵若排在前面，代表加入特徵確實有幫助
4. **波動度與趨勢斜率監控**：如果測試集的結構性特徵（波動度、均線斜率）與訓練集差異過大，模型可能失效
5. **時間序列切分**：切勿隨機切分訓練/測試集，必須按時間順序切分

### 適用範圍

- 適合有明確趨勢的個股
- 單一個股策略需要驗證跨股通用性
- 期貨或大盤指數因結構較穩定可能更適合
