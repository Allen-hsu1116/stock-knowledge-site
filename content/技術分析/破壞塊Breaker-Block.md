# 破壞塊 Breaker Block

> 訂單塊被突破後反轉角色——失效的支撐變壓力，失效的壓力變支撐，是聰明錢完成流動性獵取後的反向防線

## 核心概念

破壞塊（Breaker Block, BB）是 SMC 聯明錢概念中的進階工具。當一個訂單塊（Order Block）被價格突破失效後，它的角色會反轉：

- **看漲訂單塊被跌破** → 變成**看跌破壞塊**（未來價格回到此區域會遇到壓力）
- **看跌訂單塊被突破** → 變成**看漲破壞塊**（未來價格回到此區域會遇到支撐）

### 訂單塊為什麼會失效？

訂單塊失效的核心原因是：**聰明錢的真實目的不是在那個價位建立長期部位，而是利用那個區域獲取對面的流動性。**

- 原始訂單塊看起來像建倉 → 但聰明錢的真實目的是「Buy to Sell」或「Sell to Buy」
- 聰明錢在訂單塊區域建倉 → 推動價格突破對面的止損密集區 → 取得流動性 → 然後反向操作
- 結果：訂單塊被突破，角色反轉成為破壞塊

## 實戰應用

### 破壞塊辨識三步驟

1. **找到失效的訂單塊**：觀察價格是否已經反向突破了一個之前的訂單塊
2. **確認流動性獵取**：訂單塊失效前，是否有流動性被掃（Liquidity Sweep）？這是關鍵——沒有流動性獵取的突破可能只是普通突破
3. **等待回測進場**：價格回到破壞塊區域時，觀察反應（引線、拒絕型K線），確認後進場

### 破壞塊交易策略

**看漲破壞塊做多**：
- 前提：一個看跌訂單塊被向上突破
- 進場：價格回測破壞塊區域 + 小時間框架出現 CHoCH 確認
- 止損：設在破壞塊下方
- 止盈：下一個流動性區域

**看跌破壞塊做空**：
- 前提：一個看漲訂單塊被向下突破
- 進場：價格回測破壞塊區域 + 小時間框架出現 CHoCH 確認
- 止損：設在破壞塊上方
- 止盈：下一個流動性區域

### 破壞塊 vs 訂單塊

| 特徵 | 訂單塊 (OB) | 破壞塊 (BB) |
|------|------------|------------|
| 定義 | 大幅單邊運動前最後一根反向K線 | 被突破失效的訂單塊 |
| 角色 | 原始支撐/壓力 | 反轉後的壓力/支撐 |
| 可靠性 | 較高（聰明錢成本區） | 中等（需確認流動性獵取） |
| 進場時機 | 首次回測 | 角色反轉後首次回測 |
| 前提條件 | 無需突破 | 必須已被突破 |

## 注意事項

- **不是所有失效的訂單塊都是有效的破壞塊**：必須伴隨流動性獵取才有意義——聰明錢「買是為了賣」或「賣是為了買」
- **破壞塊的效力會遞減**：被回測次數越多，效果越弱
- **搭配 FVG 確認**：伴隨 FVG 的破壞塊更可靠
- **大時間框架的破壞塊更重要**：4小時或日線的破壞塊比5分鐘的有意義得多
- **破壞塊是反向操作區**：不要在破壞塊區域做原本方向——如果原來是看漲訂單塊被突破，那它現在是壓力不是支撐
- **避免孤立使用**：破壞塊要放在完整的 SMC 框架中使用——先確認大結構方向、流動性、再找破壞塊
