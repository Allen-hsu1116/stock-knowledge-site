# 威科夫方法進階：吸籌派發五階段判讀

> 威科夫吸籌與派發各分五個階段（A→E），精準判讀每個階段的特徵事件與量價行為，是找到 Spring 買點與 UTAD 賣點的關鍵。

## 核心概念

威科夫方法的精華不在於辨識「市場在漲還在跌」，而在於**精準定位市場處於吸籌或派發的哪個階段**。吸籌與派發各有五個階段（A→E），每個階段有明確的事件序列和量價特徵。判讀正確，就能在 Spring（彈簧效應）時建倉、在 UTAD（派發後假突破）時離場。

## 吸籌五階段詳解（Accumulation Phase A→E）

### 階段 A：下跌趨勢停止

**事件序列**：PS → SC → AR → ST

| 事件 | 全名 | 特徵 | 實戰意義 |
|------|------|------|----------|
| PS | Preliminary Support 初步支撐 | 振幅放大、成交量放大，跌勢中首次出現明顯承接 | 跌勢可能接近尾聲的早期訊號 |
| SC | Selling Climax 賣出高潮 | 恐慌性拋售，極長黑K+巨量，低檔大量換手 | 專業人士開始吸收恐慌籌碼 |
| AR | Automatic Rally 自動反彈 | 賣壓枯竭後自然反彈，短空回補 | AR 高點決定吸籌區間上緣 |
| ST | Secondary Test 二次測試 | 回測 SC 低點，振幅和成交量縮小 | 成功的 ST 確認 SC 是真正底部 |

**關鍵**：SC 的最低點、ST 的低點、AR 的高點三者決定了吸籌的交易區間（TR）。

**特殊情況**：若下跌不劇烈，階段 A 可能沒有明顯的 PS 和 SC，振幅和成交量不會突然放大。但 E大建議：有明確 PS → SC → AR → ST 的圖形，代表有大機構主力正在吸納籌碼，後續行情更值得期待。

**再吸籌（Re-accumulation）**：上漲過程中的橫盤震盪也可以是再吸籌，但階段 A 不會出現明顯的 SC 和 ST，看起來更像派發的階段 A。階段 B→E 表現類似，但時間較短、幅度較小。

### 階段 B：主力建倉

**核心特徵**：
- 主力機構低價建倉，屯籌待漲
- 屬於因果法則中的「因」
- 建倉時間可能很長（一年以上）
- 價格在支撐線和阻力線之間波動

**量價演變**：
- 早期：波動範圍大、成交量大（主力測試上下方力量）
- 中期：下行波段的成交量逐步降低（浮籌被吸走）
- 末期：賣盤衰竭，成交量持續萎縮 → 準備進入階段 C

**主力行為**：
- 低價吸籌：在支撐附近靜靜買入
- 高位做空試探：測試上方賣壓
- 向上洗盤：清除上方的浮動籌碼

### 階段 C：賣盤測試（最重要的判讀點）

**兩種可能的測試方式**：

1. **Spring 彈簧效應**（最經典）
   - 價格跌破 TR 支撐線，然後快速回到區間內
   - 這是**空頭陷阱**：跌破支撐意味著下跌趨勢又開始，但實際上終結了下跌趨勢
   - 把最後的賣家或空頭套住

2. **支撐線上方測試**
   - 測試發生在支撐線上方，沒有跌破
   - 判讀較困難，容易與階段 B 的回調混淆

**成交量是關鍵**：
- **低成交量 Spring**：預示股票已準備拉升，是**最佳初步建倉時機**
- **高成交量 Spring**：賣壓仍重，需要再觀察

**Spring 之後的確認**：SOS（Sign of Strength 強勢信號）出現，確認分析正確。

### 階段 D：需求成為主趨勢

**核心特徵**：
- 市場從「因」（吸籌）過渡到「果」（上漲）
- 交易量和波動性增加
- 價格至少漲到 TR 上沿

**量價模式**：
- 上漲時：放量、振幅大（SOS）
- 回調時：縮量、振幅小（LPS）

**LPS 是最佳加倉點**：
- 最後支撐點可能出現不只一次
- 每次 LPS 都是買入或加倉的好機會
- 價格回調縮量 = 賣壓耗盡

### 階段 E：突破上行

**核心特徵**：
- 股價離開 TR，需求控制市場
- 回調都是短暫的、不明顯的
- 趨勢已確立，追多風險相對低

**特殊情況：腳踏石（Stepping Stone）**
- 在新高位置形成新的調整區間
- 有人獲利了結，也有機構再次吸籌
- 預示股價還會有新高
- 這是再吸籌的一種形式

## 派發五階段詳解（Distribution Phase A→E）

派發是吸籌的鏡像，主力從買方轉為賣方。

### 階段 A：上漲趨勢停止

**事件序列**：PSY → BC → AR → ST

| 事件 | 全名 | 特徵 |
|------|------|------|
| PSY | Preliminary Supply 初步供給 | 上漲中首次出現明顯賣壓，量增 |
| BC | Buying Climax 買入高潮 | 最後一波瘋狂追漲，巨量長紅 |
| AR | Automatic Reaction 自動回落 | BC 後自然回落 |
| ST | Secondary Test 二次測試 | 回測 BC 高點，量縮振幅小 |

**關鍵**：BC 的高點、AR 的低點、ST 的高點決定派發的交易區間。

### 階段 B：主力出貨

- 主力高位逐步派發
- 價格在支撐線和阻力線之間波動
- 早期波動大、成交量大
- 末期上行波段成交量降低（買盤減弱）
- 多次向上洗盤試探（測試買盤強度）

### 階段 C：買盤測試

**UTAD（Upthrust After Distribution）**：派發中最關鍵的事件
- 價格突破 TR 阻力線，然後快速回落
- 這是**多頭陷阱**：突破意味著上漲趨勢開始，但實際上是最後出貨
- 吸引散戶追多，主力趁機倒貨

**低成交量 UTAD**：主力出貨已接近完成，後續下跌機率高。

### 階段 D：供給成為主趨勢

- 價格下跌時放量、振幅大（SOW 弱勢訊號）
- 反彈時縮量、振幅小（LPSY 最後供給點）
- LPSY 是最後的減倉/做空機會

### 階段 E：跌破下行

- 股價離開 TR，供給控制市場
- 反彈都是短暫的
- 趨勢已反轉向下

## 吸籌 vs 派發對照表

| 維度 | 吸籌 | 派發 |
|------|------|------|
| 位置 | 下跌末期/底部 | 上漲末期/頂部 |
| 階段 A 事件 | PS → SC → AR → ST | PSY → BC → AR → ST |
| 階段 C 陷阱 | Spring（跌破支撐） | UTAD（突破阻力） |
| 陷阱方向 | 空頭陷阱 | 多頭陷阱 |
| 階段 D 主力方向 | LPS（買點） | LPSY（賣點） |
| 量價確認 | 漲放量/跌縮量 | 跌放量/漲縮量 |
| 結果 | 突破上行 | 跌破下行 |

## 實戰判讀 SOP

### Step 1：確認市場結構
- 先判斷大方向：上漲趨勢還是下跌趨勢？
- 找到 TR（交易區間）的上下緣

### Step 2：辨識階段 A 事件
- 底部：找 SC（巨量長黑K + 自動反彈）
- 頂部：找 BC（巨量長紅K + 自動回落）
- ST 確認後，TR 範圍確立

### Step 3：觀察階段 B 的量價變化
- 追蹤成交量趨勢：逐步萎縮 = 浮籌減少
- 注意上洗/下洗的頻率和力度

### Step 4：等待階段 C 的陷阱
- 底部等 Spring：跌破支撐 → 快速回升 → 低量 = 最佳買點
- 頂部等 UTAD：突破阻力 → 快速回落 → 低量 = 最佳賣點

### Step 5：階段 D 加碼/減碼
- 底部 LPS 加碼
- 頂部 LPSY 減碼或做空

### Step 6：階段 E 順勢操作
- 突破後順勢持有
- 注意腳踏石（再吸籌/再派發）

## 威科夫三大法則在階段判讀中的應用

### 供需法則
- 階段 C 的 Spring/UTAD 測試的就是剩餘供給/需求
- Spring 低量 = 供給枯竭 → 上漲
- UTAD 低量 = 需求枯竭 → 下跌

### 因果法則
- 階段 B 的長度決定突破後的幅度
- **「橫有多長，豎有多高」**
- TR 時間越長，突破後行情越大

### 努力與結果法則
- SC/BC 的巨量對應大波幅 → 趨勢停止的訊號
- Spring/UTAD 低量對應小波幅 → 陷阱確認
- SOS 放量大波幅 → 真正趨勢啟動

## 注意事項

1. **Spring/UTAD 不一定出現**：階段 C 有時候測試發生在 TR 內部，需要更細膩的判讀
2. **階段判讀有延遲**：往往要事後才能確認處於哪個階段，實戰中需要邊觀察邊調整
3. **多時間框架**：週線看大結構、日線看事件細節
4. **搭配籌碼面**：法人買賣超、融資融券變化可輔助確認
5. **現代市場差異**：高頻交易和演算法可能製造假的 Spring/UTAD，需要量價確認
6. **再吸籌/再派發**：上漲中的橫盤不一定是派發，也可能是再吸籌的腳踏石
