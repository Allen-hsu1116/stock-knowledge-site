# 過度擬合Overfitting量化判斷

> 用數學告訴你策略是真的好還是只是運氣好——PSR、DSR、CSCV三大工具量化過擬合風險

## 核心概念

過度擬合（Overfitting）的量化判斷是將「策略績效好是不是運氣」從感覺問題變成數學問題。三個核心工具：

### 概率夏普比 PSR（Probabilistic Sharpe Ratio）

PSR是策略真實夏普比超過基準（通常為0）的概率。不只是看SR大小，還考慮：

- **SR值越高** → PSR越高
- **採樣點T越多** → PSR越高
- **偏度越大** → PSR越高（正偏度代表大賺小賠）
- **峰度越小** → PSR越高（極端值少代表更穩定）

公式概念：PSR = Z[(SR - SR*) × √(T-1) / √(1 - γ₃×SR + (γ₄-1)/4 × SR²)]

### 收縮夏普比 DSR（Deflated Sharpe Ratio）

DSR是PSR的升級版，解決了PSR沒考慮多重測試的問題：

- PSR的基準是SR*=0，但如果你測了K個策略，K個策略中最好的SR本來就會比0高
- DSR把基準從0調整為E[max(SR)]——多重測試下最佳SR的期望值
- **DSR本質上是考慮多重測試後，真實SR超過0的概率**

測試次數N越多，DSR要求的基準SR₀越高，越難通過。

### CSCV組合對稱交叉驗證

CSCV計算PBO（Probability of Backtest Overfitting）：

1. 將績效時間序列切成S個塊
2. 任選S/2塊當IS，其餘當OOS → 共有C(S,S/2)種組合
3. 每種組合中，IS最佳策略在OOS低於中位數 → 計為過擬合
4. 統計過擬合頻率 = PBO估計值

**PBO判斷標準**：
- PBO < 0.1 → 策略穩健，過擬合風險極低
- PBO 0.1-0.3 → 中等風險，需謹慎
- PBO > 0.5 → 高度過擬合，策略不可信

**CSCV缺點**：
- 只能對一組策略算PBO，不能評價單一策略
- 對SR_IS與SR_OOS關係的預測有偏
- 對於平均回報率相同的策略組，SR_OOS與SR_IS可能呈負相關

## 實戰應用

### 過擬合檢測SOP

```
Step 1: 計算策略SR，確認 > 0
Step 2: 計算PSR(SR*=0)，確認 > 0.95
Step 3: 記錄總測試次數N（包含所有參數組合）
Step 4: 計算DSR，確認 > 0.95
Step 5: 用CSCV計算PBO，確認 < 0.3
Step 6: IS/OOS分段驗證，OOS夏普衰退 < 50%
```

### 參數高原識別法

不只看最佳參數，要看整個參數空間：

- **參數高原**：某區域參數績效都不錯，周圍也穩定
- **參數孤島**：單一參數績效突出，周圍暴跌
- 選高原中心的參數，而非孤島頂峰

### FinLab實驗結果

Bailey等人用CSCV方法測試策略：

**差策略**：每100個樣本，74個最佳化後效果低於中位數 → PBO=74%，嚴重過擬合

**好策略**：每100個樣本，只有4個最佳化後效果低於中位數 → PBO=4%，策略穩健

### AI/ML模型防範過擬合三層控制

**數據層面**：
- 樣本外測試是黃金法則：將歷史數據嚴格分為訓練集、驗證集、測試集三部分
- 用PCA或Autoencoder降維，只保留最重要的特徵——輸入維度越高，過擬合風險越大
- 增加數據量讓模型學習更普遍的規律

**模型層面**：
- 簡化模型：線性模型或決策樹可能比深度神經網路更具泛化能力
- Dropout：訓練時隨機關閉部分神經元，迫使模型學習更分散、更穩健的特徵
- L1/L2正則化：在損失函數加入權重大小態罰項，限制模型複雜度
- 早停法（Early Stopping）：驗證集性能不再提升時停止訓練

**策略層面**：
- 滾動回測（Walk-forward Analysis）：將歷史數據分成多個區間，每個區間都做訓練和測試
- 蒙地卡羅模擬：隨機重採樣歷史數據，成千上萬次回測評估穩定性

### 量化交易過擬合的特殊挑戰

- **市場非平穩性**：金融市場統計特性隨時間變化，過去有效的模式未來可能失效
- **低信噪比**：金融數據充滿噪音，有效信號微弱，模型容易把噪音當信號
- **交易成本放大效果**：過擬合導致頻繁交易，真實手續費+滑價會迅速吃掉利潤

## 注意事項

- **PSR不考慮多重測試**：只適合單一策略評估，多策略比較必須用DSR
- **DSR假設收益近似常態**：偏度和峰度極端時可能不準確
- **CSCV只評估策略組不評估單策略**：無法告訴你「這個策略過擬合了沒」
- **貝葉斯方法考慮策略相關性**：CSCV忽略策略間相關性，貝葉斯方法引入協方差矩陣更精確但計算量大
- **PBO低不代表策略一定賺**：只代表「策略不是純靠運氣」，但市場結構變化仍會讓策略失效
- **回測曲線過於完美是過擬合的最大紅旗**：批判性看待，先懷疑再驗證
- **從簡單假設開始**：簡單模型先證明有效性，再逐步增加複雜度
- **模型是訊號不是聖盃**：將AI模型視為提供預測訊號的工具，配合嚴格資金管理才能穩健獲利
