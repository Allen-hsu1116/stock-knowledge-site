# 移動停利停損 Trailing Stop

> 止損點隨價格往有利方向移動但不會回頭，鎖住獲利也控制虧損——核心是情緒管理不是保證獲利

## 核心概念

移動停利停損（Trailing Stop），又稱追蹤停止、動態停利、移動鎖利。是一種出場機制：觸發點會隨著價格往有利方向移動，但價格反向時觸發點維持不變。當價格觸及觸發點時自動出場。

**關鍵區別**：
- 傳統停損停利：根據進場成本設定固定出場價
- 移動停利停損：根據最高（低）價設定動態出場價

## 三情境分析：為什麼 Trailing Stop 優於傳統停損

以「追蹤停損 15%」為例，進場後三種情境：

1. **直線下跌**：追蹤停損 = 傳統停損。進場後直接跌 15% 出場，兩者結果相同
2. **狹幅橫盤**：追蹤停損 ≥ 傳統停損。波動未超過 15%，兩者都沒觸發；若之後上漲，追蹤停損點會跟著上移，永遠不會比傳統停損差
3. **向上噴出後回落**：追蹤停損 >>> 傳統停損。股價漲 50% 後回落，傳統停損只能看紙上富貴歸零；追蹤停損 15% 在 +35% 時獲利出場

**結論**：追蹤停損在所有情境下都 ≥ 傳統停損，最差情況打平，最好情況多賺幾十%。

## 實戰應用

### 三種設定方式

**1. 固定百分比**
- 從最高點拉回 X% 時出場
- 最常用，適合大多數場景
- 例：股價100→150，設10%，出場點=150×0.9=135

**2. ATR 倍數**
- 從最高點拉回 N 倍 ATR 時出場
- 波動大停損大、波動小停損小，最適應市場
- 例：近期ATR=80點，設2倍，最高點11057，出場點=11057-(80×2)=10897

**3. 固定點數**
- 從最高點拉回 N 點時出場
- 僅適合極短線，一般不建議
- 不同價位層級用同一點數不合理（指數2000點和10000點停損50點意義完全不同）

### 停損結合移動停利

組合策略：百分比停損 2% + 移動停利 3 倍 ATR

- 進場初期：百分比停損生效
- 隨價格上漲：移動停利價格超過百分比停損
- 移動停利接管後：停損價格只會向上移動
- 當移動停利價格超過進場成本：交易進入不虧損狀態

### 觸發價設定參考

| 操作週期 | 建議設定 | 備註 |
|----------|----------|------|
| 當沖 | 不適合 | 應有更敏感的出場方式 |
| 短線 | 3～10% | 依標的波動度調整 |
| 波段 | 10～20% | 波動大的標的設寬一點 |

**觀察方法**：先觀察標的近3個月上下震盪幅度，設定值應大於常態波動範圍。

## 注意事項

### 五大陷阱

1. **設太小被洗出場**：設定值小於標的常態波動，頻繁觸發錯過行情
2. **市場不配合你的風控**：標的常態波動5%，停損設2%等於找死
3. **出場後反轉向上**：長期必定發生，這是Trailing Stop的代價
4. **回測不一定更好**：有明確進出場策略的系統，加上Trailing Stop幾乎不會改善績效
5. **不等於最佳出場點**：Trailing Stop是風險管理工具，不是利潤最大化工具

### 市場先生回測結論

絕大多數有明確進出場策略的系統，加上停損停利及Trailing Stop，幾乎都不會變得更好。理由：
1. 原策略的出場邏輯可以代替停損停利
2. 停損停利避開某些虧損，但也會錯過某些獲利

**Trailing Stop的真正價值**：減少情緒影響、增加操作彈性、幫助無法盯盤的人保持紀律。

### 趨勢行情 vs 盤整行情

**趨勢行情**：移動停損發揮最大功效，價格長時間朝單一方向移動，停損點步步抬高鎖定利潤。

**盤整行情**：挑戰較大，價格來回拉鋸，移動停損可能頻繁觸發。經驗法則是設定距離應略大於常態短期波動範圍，或暫時不使用等到趨勢明朗。

### 固定止損 vs 移動止損比較

| 策略類型 | 機制 | 適用情況 | 優點 | 缺點 |
|---------|------|---------|------|------|
| 固定止損 | 進場時設定固定止損價 | 震盪行情、短線 | 簡單明確、控制初始風險 | 無法保護浮動獲利 |
| 移動止損 | 隨價格變動調整停損距離 | 趨勢行情、中長線波段 | 保護利潤、自動調整、情緒影響小 | 可能過早出場、參數依賴 |

### 停利 + 移動止損雙保險組合

可以同時設置停利價和移動止損：
- 例：100元買進，設定120元停利，同時設置5%移動止損
- 行情直奔120以上 → 120自動賣出賺20%
- 價格沒到120就回跌5% → 移動止損在約95元處止損
- 保證「該賺則賺，否則虧損有限」

### 手動移動止損階梯法

1. 進場100元，固定止損90元
2. 價格漲到110 → 手動把止損提高到100（保本）
3. 價格漲到120 → 止損提到110
4. 每創新高就階段性跟進鎖利

### 使用時機

- 不想長期盯盤
- 行情波動劇烈
- 自己出場容易受情緒影響
- 帳面已有獲利想鎖住

### 程式交易實作（富果 API 範例）

核心邏輯：
1. 從歷史 K 線取得策略起始日到最近交易日的最高價
2. 盤中即時價格若高於歷史最高價，更新最高價
3. 當現價 ≤ 最高價 × (1 - 停利比例) 時，觸發平倉

```python
# 核心判斷邏輯
if self.now_price >= self.high_price:
    self.high_price = self.now_price  # 更新高點

if self.now_price <= self.high_price * (1 - self.stopProfit_ratio):
    self.sell_total_shares(qty)  # 觸發出場
```

進階變化：最高點回檔 N% 賣出一半進行分批出場，而非一次全部出清。

### Chandelier Exit（吊燈停損）進階應用

Chandelier Exit 是 Trailing Stop 的 ATR 進階版，用波動度取代固定百分比：
- 做多：Chandelier Exit = 最高價 - ATR(n) × 乘數
- 做空：Chandelier Exit = 最低價 + ATR(n) × 乘數
- 常用參數：ATR 期間 14~22、乘數 2.5~4.0
- 核心優勢：波動大時停損距離遠（不怕被洗），波動小時停損距離近（鎖住更多利潤）

詳細內容見 [[Chandelier-Exit吊燈出場指標]]

### 盤整行情使用 Trailing Stop 的技巧

1. **暫停使用**：盤整期不使用移動停損，等趨勢明朗再啟用
2. **拉大距離**：設定值大於盤整區間的平均震盪幅度，避免被小波動洗出
3. **分段移動**：先將止損移到保本價，之後只在明顯高點下方設置，不追蹤每一細小新高
4. **結合技術位**：只在關鍵支撐/壓力位移動停損，而非每個小波動都跟

### 手動移動止損進階：階梯法

1. 進場100元，固定止損90元
2. 價格漲到110 → 手動把止損提高到100（保本）
3. 價格漲到120 → 止損提到110
4. 每創新高就階段性跟進鎖利
5. 優點：比自動移動止損更有彈性，可以結合技術分析判斷關鍵位置
6. 缺點：需要紀律和盯盤，容易因情緒不執行
