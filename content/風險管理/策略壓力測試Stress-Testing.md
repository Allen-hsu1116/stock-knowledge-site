# 策略壓力測試 Stress Testing

> 不測壓力就上線，等於蒙眼開車——用極端情境檢驗策略是否能在風暴中存活

## 核心概念

策略壓力測試是將交易策略暴露在各種極端市場情境下，檢驗其是否能存活。回測只告訴你「過去正常行情下的表現」，壓力測試告訴你「最壞情況下你還在不在」。

### 為什麼需要壓力測試？

- 回測的平均績效不代表極端情況的表現
- 黑天鵝事件不是例外，是常態——每 5-10 年就會出現一次
- 策略在平穩市場賺的錢，可能一次極端事件就全賠回去
- 2008 金融海嘯、2020 COVID 暴跌、2022 升息衝擊，都是壓力測試的活教材

## 實戰應用

### 壓力測試五情境

| 情境 | 描述 | 檢驗重點 |
|------|------|----------|
| 閃崩 | 2008/2015/2020型急跌 | 停損能否執行？流動性是否足夠？ |
| 長期熊市 | 2000-2002/2008型緩跌 | 策略空頭時的最大回撤？能撐多久？ |
| 波動率暴漲 | VIX從15飆到80 | 選擇權策略的 Vega 風險、部位是否過大？ |
| 流動性乾涸 | 2008信用凍結型 | 能否出場？滑價成本多高？ |
| 相關性趨同 | 所有資產同時下跌 | 分散是否真的有效？對沖是否失效？ |

### 壓力測試 SOP 五步驟

1. **列出極端情境**：從歷史事件挑選，也設計「想像但合理」的情境
2. **計算各情境下策略損益**：不是看平均，是看最差情況
3. **評估存活機率**：最大回撤是否超過可承受範圍？保證金是否足夠？
4. **調整策略參數**：降低槓桿、收緊停損、增加現金比例
5. **定期重新測試**：市場結構會變，壓力測試也要更新

### 歷史壓力事件清單（台股為主）

| 事件 | 期間 | 台股跌幅 | 特徵 |
|------|------|----------|------|
| 亞洲金融風暴 | 1997-1998 | ~40% | 匯率危機、外資撤離 |
| 網路泡沫 | 2000-2002 | ~65% | 科技股崩盤、長期熊市 |
| SARS | 2003 | ~20% | 短期恐慌、V型反轉 |
| 金融海嘯 | 2008-2009 | ~55% | 系統性風險、流動性枯竭 |
| 歐債危機 | 2011 | ~25% | 外資大賣、政策不確定性 |
| COVID-19 | 2020/3 | ~30% | 極速暴跌、V型反轉 |
| 升息衝擊 | 2022 | ~30% | 估值壓縮、科技股重挫 |

### 危機組合設計

設計壓力測試組合時，不只是看單一事件，而是組合多個不利因素：

- **溫水煮青蛙**：緩慢下跌 + 低波動（最危險，因為不覺得需要行動）
- **閃電崩盤**：急速暴跌 + 流動性消失（停損可能失效）
- **流動性陷阱**：相關性趨同 + 對沖工具失靈（分散失效）
- **政策衝擊**：突發法規 + 市場結構改變（因子失效）

## 注意事項

### 壓力測試的四大盲點

1. **歷史不會重複**：過去的壓力事件只是參考，下次的危機一定不同
2. **低估尾部風險**：真正的極端事件往往超出我們的想像力
3. **相關性假設失效**：壓力時所有資產相關性趨同，分散效果消失
4. **倖存者偏差**：只看活下來的例子，忽略已經消失的市場/策略

### 回測績效衰退率

回測績效通常比實盤好 30-50%，這是正常的「衰退率」。如果壓力測試中策略的衰退超過 50%，代表策略可能不夠穩健。

### 與 Walk-Forward Analysis 的互補

壓力測試關注「極端情境下的存活」，WFA 關注「參數穩定性」。兩者搭配使用：
- WFA 驗證策略在正常市場下的穩定性
- 壓力測試驗證策略在極端市場下的存活能力
- 只有通過兩者檢驗的策略才值得上線
