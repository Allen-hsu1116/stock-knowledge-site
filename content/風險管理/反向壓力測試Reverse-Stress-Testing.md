# 反向壓力測試 Reverse Stress Testing

> 別問「跌 30% 會怎樣」，問「什麼組合會讓我虧 500 萬」——反向找出致命弱點

## 核心概念

傳統壓力測試是正向的：給定一個情境，算損失多大。反向壓力測試反過來：**先設定一個不可承受的損失門檻，再找出哪些情境組合會觸發它**。

這個方法源自一個殘酷的事實：**你沒想到的風險，殺傷力最大**。正向壓力測試只能測你想得到的情境，反向壓力測試逼系統告訴你最脆弱的點在哪裡。

### 為什麼需要反向壓力測試？

- 正向壓力測試受限於想像力——你只測你想到的情境
- 反向壓力測試讓組合「自己招供」——什麼組合會讓我爆掉？
- 英國審慎監理局（PRA）已要求金融機構執行反向壓力測試
- 個人投資者同樣需要：你的致命組合是什麼？

### 正向 vs 反向壓力測試

| | 正向壓力測試 | 反向壓力測試 |
|---|---|---|
| 問題 | 「如果大盤跌 30%，我會虧多少？」 | 「什麼事件組合會讓我虧 500 萬？」 |
| 起點 | 給定情境 | 給定不可承受損失 |
| 終點 | 算出損失金額 | 找出觸發情境 |
| 優勢 | 直覺、容易溝通 | 發現你想不到的弱點 |
| 盲點 | 受限於想像力 | 可能找到多種等效情境，難以排序 |

## 實戰應用

### 反向壓力測試五步驟

1. **定義不可承受損失門檻**：帳戶虧損多少會導致無法繼續交易？例如總資產的 40%、或具體金額如 200 萬
2. **列出所有風險因子**：股票持倉、利率敏感度、匯率曝險、信用利差、流動性、槓桿
3. **數學搜索觸發組合**：哪些因子同時變動多少%，會剛好達到門檻？
4. **評估情境合理性**：找到的組合是否「嚴重但合理」？還是純粹理論上的不可能？
5. **制定預防措施**：針對每個合理觸發組合，設計防禦動作

### 台股投資人反向壓力測試範例

**情境設定**：
- 帳戶總值：500 萬
- 持股：台積電 60%、聯發科 20%、現金 20%
- 使用融資，融資比例 30%
- 不可承受損失門檻：200 萬（40%）

**搜索結果——觸發 200 萬虧損的情境組合**：

| 情境 | 台積電跌幅 | 聯發科跌幅 | 融資追繳 | 總虧損 |
|------|-----------|-----------|---------|--------|
| A | -35% | -30% | 是 | 195 萬 |
| B | -25% | -40% | 是 | 188 萬 |
| C | -20% + 利率+2% | -25% + 利率+2% | 是 | 210 萬 |
| D | -15% + 匯率-10% | -15% + 匯率-10% | 否 | 180 萬 |

**發現**：
- 單純股價下跌需要跌幅超過 35% 才觸發（情境 A）——看似安全
- 但加入利率或匯率衝擊後，股價跌幅只需 15-20% 就能觸發（情境 C、D）——**隱藏的脆弱點**
- 融資追繳是加速器，一旦觸發會強制在低點賣出，擴大虧損

### 反向壓力測試的三個層次

#### 第一層：單一資產衝擊
最簡單——某檔股票跌多少% 會觸發門檻。這跟傳統壓力測試差不多。

#### 第二層：多資產聯動
加入相關性——當兩檔股票同時下跌，虧損不是簡單相加，而是被相關性放大。見 [[相關性崩潰Correlation-Breakdown]]。

#### 第三層：系統性連鎖
加入二階效果——股價下跌 → 融資追繳 → 強制賣出 → 進一步下跌 → 流動性枯竭。這是最危險也最難建模的層次。

## 注意事項

### 反向壓力測試的陷阱

1. **找到太多情境**：可能有多種等效組合同時觸發門檻，難以決定優先處理哪個。解法：用歷史機率排序，優先處理「最可能發生」的致命組合
2. **情境不合理**：數學搜索可能找到理論上能觸發但實際不可能的組合。解法：加入合理性過濾，排除不可能的因子組合
3. **忽略二階效果**：只算一階衝擊，忽略強制平倉、流動性枯竭等連鎖反應。解法：至少做第三層次的連鎖分析
4. **門檻設定錯誤**：門檻設太高等於沒測，設太低則每個情境都觸發。解法：門檻應該是「無法承受」的損失，不是「不舒服」的損失
5. **只做一次**：市場和持倉都在變，上個月的反向壓力測試結果可能已經過時。解法：至少每月重做，持倉重大改變後立即重做

### 與正向壓力測試的互補

反向壓力測試不是取代正向壓力測試，而是互補：
- **正向**：我知道的風險，評估衝擊多大
- **反向**：我不知道的風險，找出它是什麼
- 兩者搭配使用才能覆蓋「已知的已知」和「未知的未知」

### 個人投資人的最低標準

不需要金融機構那麼複雜，但至少做：
1. 定義你的「無法承受損失」門檻
2. 用 Excel 或試算表算：各持股跌多少會觸發
3. 加入融資/槓桿效果
4. 每月更新一次
