# 程式交易停止時機與回收係數 EA Stop Timing and Recovery Factor

> EA有壽命，超過最大回撤或回收係數恢復期限仍未恢復就是停止訊號——承認策略失效比死撐更安全

## 核心概念

### EA的有效期限

再怎麼優異的程式交易策略（EA）都無法永遠有效。因為面對「會變動的市場」，「EA則不會變動」。即使一向佔有優勢的EA，也無法保證過了明天依然能持續保有優勢。

**核心認知**：EA是有壽命的。若無法接納這一點來運用EA，很可能在撤退時機與結果上帶來巨大差異。

### 停止運用的兩大基準

#### 基準一：最大回撤突破法

進行10年以上長期回測，以其最大回撤作為基準。

**操作邏輯**：
1. 回測10年以上的策略績效
2. 找出歷史最大回撤（Maximum Drawdown）
3. 實際運用時，若超過此最大回撤 → 停止使用該EA

**範例**：回測最大回撤為10%，實際運用虧損超過10% → EA優勢不再適用

**為什麼要10年以上**？因為不論好的市況、壞的市況，往往每10年風水輪流轉。10年以上的回測才能涵蓋完整的市場週期。

#### 基準二：回收係數（Recovery Factor）確認法

回收係數是衡量「從回撤中恢復能力」的指標：

```
回收係數（RF）= 總損益（盈虧）÷ 最大回撤
```

**數值意義**：數值越大，代表從回撤中恢復的可能性越高。

**適當標準**：
- 1年間的測試：RF ≥ 1.0（意味著從回撤恢復需要1年）
- 10年間的測試：RF ≥ 10.0

**範例**：
- 1年回測期間，RF = 1.0 → 恢復需要1年
- 1年回測期間，RF = 2.0 → 恢復需要半年
- 若過了RF計算的恢復期間仍未恢復 → EA優勢不再適用，應考慮停止

## 實戰應用

### 停止EA的決策流程

```
1. 建立基準 → 10年以上回測，確定最大回撤和回收係數
2. 即時監控 → 每日檢查實際虧損是否超過最大回撤
3. 觸發停止 → 虧損超過最大回撤，立即停止EA
4. 等待恢復 → 觀察RF計算的恢復期間，是否自然恢復
5. 恢復判斷 → 若恢復期間內恢復，可重新啟動；否則永久停止或重新設計
```

### 回收係數計算實例

假設一個EA的10年回測結果：
- 總損益：+500萬
- 最大回撤：-50萬
- 回收係數 = 500 ÷ 50 = 10.0 ✅（符合10年標準）

實際運用6個月後：
- 累計虧損超過50萬（超過回測最大回撤）→ 停止EA
- 或恢復期間 = 500 ÷ 50 ÷ 12個月 × 6個月評估，仍持續虧損 → 停止EA

### 策略失效的漸進過程

策略失效不是瞬間發生的，而是一個漸進過程：

1. **超額報酬遞減**：策略仍有正報酬，但超額報酬逐漸縮小
2. **趨近隨機**：策略報酬接近隨機，不再有統計顯著性
3. **開始虧損**：策略持續虧損，侵蝕過去累積的利潤

**關鍵**：在你虧掉大部分資金之前，用量化方法偵測到失效訊號，比猜測市場方向更重要。

## 注意事項

- **回測期不足**：只用1-2年回測就上線，最大回撤可能被嚴重低估
- **市場結構改變**：市場微結構改變（如交易時間、稅率、法規）可能讓EA完全失效，與回測無關
- **過度優化回測**：為了通過最大回撤測試而調整參數，是曲線套入（overfitting），不是真正的策略改善
- **心理障礙**：停止EA最大的敵人不是市場，是交易者「不甘心」的心理。承認策略失效需要紀律
- **恢復不等於有效**：EA從回撤恢復不代表策略重新有效，可能只是市場恰好回到有利狀態
