# 策略容量 Strategy Capacity

> 每個策略能承載的最大資金規模，超過容量上限後報酬率必然下降

## 核心概念

策略容量（Strategy Capacity）是指一個交易策略在維持預期報酬率的前提下，能夠有效運作的最大資金規模。超過這個上限，策略的邊際報酬率會因為衝擊成本、滑價和流動性限制而顯著下降。

簡單說：**策略容量 = 資金利用率能維持100%的最大資金量**。當你投入的資金無法被充分利用（因為市場流動性不夠、滑價太大、或資金被凍結），策略就達到了容量上限。

### 策略容量 vs 部位大小

- **部位大小（Position Sizing）**：在策略容量內，決定每次交易投入多少資金
- **策略容量（Capacity）**：整個策略能承載的總資金上限
- 部位大小是戰術問題，策略容量是戰略問題

## 實戰應用

### 為什麼策略容量很重要？

1. **Alpha 衰減的加速器**：超過容量的策略不只是賺少了，而是會賠錢——因為你的大單本身就是價格的衝擊源
2. **零和博弈的致命傷**：容量不足的策略（如當沖、套利），本質是零和博弈，規模擴大後邊際收益遞減極快
3. **基金規模的隱形殺手**：很多基金績效隨規模膨脹而下滑，根本原因就是超過策略容量

### 影響策略容量的關鍵因素

1. **市場流動性**
   - 標的日均成交量越大，策略容量越大
   - 小型股策略容量遠小於大型股策略
   - 非主流交易時段流動性驟降，容量也下降

2. **交易頻率**
   - 高頻策略每次利潤極小，對滑價容忍度極低，容量很小
   - 波段策略每次利潤較大，可容忍較大滑價，容量較大
   - 換手率越高，交易成本占比越重，容量越受限

3. **衝擊成本**
   - 交易成本隨資金規模呈**非線性增長**，而策略收益呈**線性增長**
   - 當衝擊成本超過預期利潤，策略就失效了
   - 在台股，衝擊成本可能超過手續費和交易稅

4. **策略持倉時間**
   - 持倉時間越短，對成交價格越敏感
   - 日內策略的容量遠低於持有數週的波段策略

### 粗略估計策略容量的方法

**方法一：資金利用率法（做市商策略範例）**

1. 統計策略有效市場條件持續時間（如「4小時內不發生大波動」）
2. 統計小資金（如1萬美元）下，資金完全被凍結/佔用的平均時間（如10分鐘）
3. 策略容量 ≈ 有效時間 / 凍結時間 × 測試資金 = 4小時/10分鐘 × 1萬 = 24萬美元

**方法二：報酬率衰減觀察法**

1. 從小資金開始運行策略
2. 逐步增加資金，觀察報酬率變化
3. 當報酬率開始顯著下降的臨界點，即為策略容量
4. 例：200萬→日報酬0.2%，300萬→日報酬0.12%，容量約在200-300萬之間

**方法三：回測交易量限制法**

- 回測時加入成交量限制（如單筆交易不超過日成交量5%）
- 逐步放大資金規模，觀察何時績效開始顯著衰減
- ⚠️ 回測會忽略冰山訂單，可能低估容量
- ⚠️ 回測若不考慮成交量，可能嚴重高估容量

### 不同策略類型的典型容量

| 策略類型 | 典型容量 | 原因 |
|----------|----------|------|
| 高頻做市 | 數百萬~數千萬 | 每次利潤極小，滑價容忍度低 |
| 日內動能 | 數千萬~數億 | 需要快速進出，衝擊成本敏感 |
| 波段趨勢 | 數億~數十億 | 持倉時間長，對成交價較不敏感 |
| 價值投資 | 數十億~數百億 | 換手率極低，流動性要求低 |
| 套利策略 | 通常很小 | 利潤空間窄，容量極易飽和 |

### 策略容量與因子衰退的關係

- 策略容量是因子衰退的**物理限制**
- 當一個因子被過度使用（策略擁擠），不只是 Alpha 衰減，策略容量也會因為參與者爭搶流動性而下降
- 容量不足的策略公開教學 = 暴露對手盤操作方式，加速衰退
- 參見 [[因子衰退Factor-Decay]]

## 注意事項

- **策略容量不是固定值**：市場流動性會波動，容量也會跟著變動
- **短期規模增速比絕對規模更危險**：突然湧入的大資金對市場的衝擊遠大於緩慢增長
- **回測無法精確估算容量**：冰山訂單、隱藏流動性在回測中看不到
- **策略參數變動會改變容量**：每次下單大小、持倉時間等參數一變，容量估算就失準
- **分散到多個標的可擴展容量**：但要注意標的間的相關性，相關性高時分散效果有限
- **管理人的投資能力是否體現在當下規模**：比看絕對規模更重要的判斷標準
