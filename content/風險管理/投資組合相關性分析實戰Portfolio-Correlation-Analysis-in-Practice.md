# 投資組合相關性分析實戰 Portfolio Correlation Analysis in Practice

> 50檔股票不代表50種風險——如果相關性太高，你的分散投資只是幻覺；相關性是看不見的隱形殺手

## 核心概念

投資組合相關性分析是將相關性理論轉化為可操作的投資組合管理工具。不同於[[相關性風險Correlation-Risk]]的理論框架和[[相關性崩潰Correlation-Breakdown]]的危機機制，本文聚焦於**日常投資組合管理中如何實際計算、評估、運用相關性**來降低風險。

### 分散投資的幻覺

兩個同樣持有10檔股票的投資組合：
- **組合A**：10檔科技股，平均相關性0.8 → 實質風險分散效果極低
- **組合B**：跨產業混合，平均相關性0.3 → 風險大幅降低

同樣10檔，B的風險遠低於A。差異不在數量，在相關性。

### 常見資產相關性實例

| 資產配對 | 相關性 | 分散效果 |
|----------|--------|----------|
| Apple vs Microsoft | ~0.75 | 差（同為大型科技股） |
| S&P 500 vs 黃金 | ~0.0 至 -0.2 | 佳（危機避風港） |
| 股票 vs 債券 | 歷史約0.0，近年波動大 | 不穩定 |
| Bitcoin vs Ethereum | ~0.85 | 差（高度相關） |
| 油股 vs 油價 | ~0.6-0.8 | 差（邏輯關聯） |
| 美股 vs 新興市場 | ~0.6 | 中等 |
| REITs vs 股票 | ~0.5-0.6 | 部分分散 |
| 價值股 vs 成長股 | ~0.7 | 幫助有限 |

## 實戰應用

### 1. 投資組合相關性三步驟評估

**Step 1：計算加權平均相關性**

計算投資組合中所有持倉兩兩之間的相關性，以部位大小加權平均。

| 加權平均相關性 | 分散品質 | 行動 |
|----------------|----------|------|
| < 0.6 | 良好分散 | 維持配置 |
| 0.6 - 0.8 | 中等分散 | 考慮加入低相關資產 |
| > 0.8 | 分散不足（集中風險） | 必須調整 |

**Step 2：識別相關性聚類**

將持倉按相關性分組：
- 科技股聚在一起
- 金融股一起移動
- 債券與股票負相關
- 黃金接近零相關

發現聚類後，每個聚類視為**一個風險單位**，而非多個獨立持倉。

**Step 3：加入低相關資產**

改善分散的具體行動：
- 重倉美國大型科技股 → 加入小型價值股、國際股、債券
- 全股票 → 加入債券、黃金、房地產
- 全成長股 → 加入價值股、高股息股

### 2. 相關性加權部位配置（Correlation-Weighted Position Sizing）

這是傳統固定百分比部位管理的進階版：**新部位的相關性越低，可以配置越大；相關性越高，配置越小**。

```
部位大小 = 基礎權重 × 相關性調整係數

相關性調整係數：
- 與組合相關性 0.2 → 乘數 1.5（可放大至 5%）
- 與組合相關性 0.5 → 乘數 1.0（正常 3%）
- 與組合相關性 0.8 → 乘數 0.5（縮小至 2%）
```

**實戰範例**：
- 你已持有5檔電子股，想再加碼台積電（相關性~0.8）→ 只配置2%
- 你想加入黃金ETF（相關性~0.0）→ 可配置5%
- 你想加入高股息金融股（相關性~0.3）→ 可配置4%

### 3. 理想投資組合相關性結構

| 層級 | 佔比 | 與核心相關性 | 目的 |
|------|------|-------------|------|
| **核心持倉** | 60-70% | 彼此0.4-0.6 | 穩定報酬 |
| **分散器** | 20-30% | 與核心 -0.2 至 +0.3 | 降風險 |
| **機會性部位** | 10-15% | 可以較高 | 追求超額報酬 |

**核心持倉**選擇中等相關性的優質資產，不要全部高度相關。
**分散器**選擇低相關甚至負相關的資產（債券、黃金、現金），是風險保護層。
**機會性部位**可以承擔較高相關性，但必須有明確的報酬理由。

### 4. 滾動相關性監控

相關性不是靜態的，需要持續監控：

| 期間 | 用途 | 注意事項 |
|------|------|----------|
| 30日 | 短期趨勢變化 | 雜訊多，需搭配其他訊號 |
| 90日 | 中期結構變化 | 適合偵測相關性崩潰早期訊號 |
| 1年 | 長期模式 | 較穩定但反應慢 |

**實戰訊號**：當30日滾動相關性大幅偏離1年值（差異>0.3），代表相關性結構正在改變，需要關注。

### 5. 相關性壓力測試

模擬最壞情境：**假設所有資產相關性飆升至0.9（危機情境）**，計算投資組合最大損失。

```
壓力測試步驟：
1. 將相關性矩陣全部替換為 0.9
2. 用新的相關性矩陣計算組合 VaR
3. 比較：正常VaR vs 崩潰VaR
4. 如果崩潰VaR超過可承受損失 → 降低部位
```

這個測試每季至少做一次，大型調整後也要重做。

### 6. 五種常見相關性錯誤

**錯誤1：不同名稱＝不同風險**
Apple、Amazon、Google、Microsoft、Meta——名稱不同，但都是大型科技股，相關性0.75+。持5檔不算分散。

**錯誤2：忽略產業相關性**
銀行、保險、資產管理——商業模式不同，但都是金融業，產業內高度相關。

**錯誤3：忽略因子相關性**
所有成長股不管哪個產業都高度相關，所有價值股也一樣。因子暴露比產業分類更重要。

**錯誤4：忽略加密貨幣相關性**
Bitcoin和Ethereum相關性0.85+，同時持有分散效果極低。而且加密貨幣與科技股的相關性也在上升。

**錯誤5：只用歷史相關性**
歷史相關性在常態下可靠，但在regime change時完全失效。必須用壓力測試補充，模擬危機時的相關性結構。

### 7. 搭配其他風險指標

相關性單獨看不夠，要搭配：

| 組合 | 意義 | 風險等級 |
|------|------|----------|
| 高波動率 + 高相關性 | 危險 | 🔴 |
| 高Beta + 高相關性 | 放大市場風險 | 🔴 |
| 大部位 + 高相關性 | 存在性風險 | 🔴 |
| 低波動率 + 低相關性 | 安全 | 🟢 |

## 注意事項

1. **相關性在 brokerage statement 上看不到**：你的帳戶顯示50個持倉、50個代碼，看起來很分散。但相關性揭露真相——那50個持倉可能實質只有5個風險單位。

2. **相關性是動態的**：今天低相關的資產，明天可能高相關。定期（至少每月）重新計算，不要用去年的相關性做今天的決策。

3. **相關性加權部位不是萬能**：它降低相關性風險，但不消除。在[[相關性崩潰Correlation-Breakdown]]時，所有調整都可能來不及。

4. **過度優化相關性的陷阱**：如果為了追求最低相關性而買入不理解的資產，可能引入新的風險。相關性分析是工具，不是策略本身。

5. **台股的特殊限制**：台股電子股佔比超過60%，即使跨產業配置，系統性風險仍然偏高。考慮跨市場（美股、債市）配置才是真正降低相關性的有效方法。
