# 回測驗證Backtesting陷阱

> 回測是驗證策略的必要工具，但十個陷阱讓你以為找到聖盃其實找到的是地雷——認識陷阱比跑回測更重要

## 核心概念

回測（Backtesting）是量化策略研發的核心環節，但回測結果好 ≠ 未來績效好。一個常見的悲劇是：回測遠勝大盤、高夏普率、低回撤的策略，實盤卻急轉直下。原因不是策略不好，而是回測過程掉進了陷阱。

## 十大回測陷阱

### 1. 數據有誤（Garbage In, Garbage Out）
- 數據缺失、錯誤值、誤植數字，回測結果必定偏誤
- 即使向專業廠商購買的數據也可能有錯
- **解法**：回測前做data cleaning，用其他來源交叉驗證

### 2. 樣本不足
- 交易筆數太少，統計結果缺乏意義
- 統計學共識：至少30個樣本才有參考價值
- **解法**：中長期策略需更長回測期間，高頻策略幾個月就夠

### 3. 回測區間不完整
- 只回測牛市期間 → 過度樂觀
- 只回測趨勢期 → 不知道盤整時會虧多少
- **解法**：至少涵蓋一次牛市、熊市、盤整，經歷完整景氣循環

### 4. 數據不合時宜
- 法規變更、市場結構改變、科技進步
- 1970年前黃金數據（與美金掛鉤）不適合現代
- **解法**：了解數據的歷史背景，判斷對未來的參考價值

### 5. 未排除極端值
- 一次閃崩大賺、一次超大行情，拉高整體績效
- 移除極端獲利後績效變差很多 → 策略只靠少數意外
- **解法**：排除極端值重看績效，策略要能承受意外而非仰賴意外

### 6. 忽略交易成本
- 回測軟體預設成本為0
- 當沖/高頻策略手續費吃掉全部利潤
- 滑價對順勢策略影響更大
- **解法**：手續費+交易稅+滑價全算進去，寧可保守估計

### 7. 程式碼錯誤
- **邏輯矛盾**：價值投資策略同時設停損，跌了要買還是賣？
- **未來資料偷看（Look-Ahead Bias）**：用3月GDP判斷3月股價，但GDP要5月才公布
- 績效好到太誇張 = 程式寫錯
- **解法**：挑幾筆交易出來核對，檢查訊號時間點

### 8. 過度最佳化 Overfitting
- 測試幾百個參數組合，選歷史最好的
- 歷史上每段時期的最佳參數都不一樣
- **解法**：選參數高原不選參數孤島，IS/OOS分段驗證

### 9. 曲線套入 Curve Fitting
- 不斷加條件讓歷史績效更好
- 條件越多越貼合過去，越不適應未來
- **解法**：策略核心邏輯簡單就好，用多種商品測試同一策略

### 10. 缺乏可執行性
- 漲停鎖住買不到、低流動性成交不了、不能做空的標的
- 回測看得到價格 ≠ 現實成交得到
- **解法**：考慮成交量、漲跌停、能否做空、法規限制

## 實戰應用

### 回測可信度自檢表

| 檢查項目 | 通過標準 |
|---------|---------|
| 數據清洗 | 已檢查缺失/錯誤/異常值 |
| 樣本數 | 交易筆數≥30 |
| 區間覆蓋 | 至少一次牛熊市+盤整 |
| 極端值 | 排除後績效差異<20% |
| 交易成本 | 手續費+稅+滑價已計入 |
| Look-Ahead | 訊號時間點均已驗證 |
| 參數穩定性 | 參數高原，非孤島 |
| OOS驗證 | IS/OOS績效一致 |
| 可執行性 | 成交量足夠、無法規限制 |
| 預期衰退 | 預期實盤衰退30-50% |

### 回測結果過於漂亮的紅旗

- 夏普比 > 3.0（異常高）
- 最大回撤 < 5%（幾乎不存在）
- 勝率 > 80%（太穩定了）
- 資金曲線近乎直線上升
- 任何時期都賺錢（沒有不適合的市場）

**看到以上任何一個 → 先檢查程式碼，再檢查數據，最後才相信策略**

## 注意事項

- **回測是必要的但不足夠的**：沒有回測就上場是蒙眼登山，但回測通過也不代表一定能到山頂
- **最危險的陷阱是Look-Ahead Bias**：因為它最難發現，而且會製造最漂亮的假績效
- **交易成本容易被低估**：滑價在極端行情可能比預期大5-10倍
- **回測區間選擇本身就是偏誤**：選擇從哪一年開始回測，本身就是一種參數最佳化
- **實盤永遠比回測差**：不是因為策略不好，而是回測無法模擬所有現實摩擦
