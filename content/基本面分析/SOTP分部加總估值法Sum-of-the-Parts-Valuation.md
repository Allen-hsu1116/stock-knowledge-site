# SOTP分部加總估值法Sum-of-the-Parts-Valuation

> 將多元化企業拆解為獨立業務部門，分別用最適合的方法估值後加總，再用來發現隱藏價值與對抗多元化折價的估值方法。

## 核心概念

### 什麼是 SOTP？

SOTP（Sum of the Parts Valuation），又稱「分部加總估值法」或「拆解分析（Break-up Analysis）」，是將一家多元化公司拆解為數個獨立的業務部門（Segments），分別計算每個部門作為獨立實體的價值，最後加總並扣除企業層面的調整項，求得整體內含價值。

### 為什麼需要 SOTP？——對抗「多元化折價」

市場通常對經營多種不相關業務的集團給予較低估值，稱為**多元化折價（Conglomerate Discount）**。Berger & Ofek（1995）的開創性研究證實，已開發市場的集團企業平均折價 13-15%，意思是集團市值只有其各部門獨立價值之和的 85%。

就像一個便當，市場有時因為不喜歡其中一道菜，就對整個便當打對折；SOTP 就是要把每一道菜單獨拿出來重新定價。

**多元化折價六大成因**：
1. **內部資本配置低效** — 績效好的部門補貼績效差的部門
2. **管理焦點分散** — 高層無法充分理解所有業務
3. **複雜度與不透明** — 投資人因看不清而給折價
4. **投資人偏好純粹曝險** — 基金經理人傾向自行組合標的
5. **代理成本** — 管理層優先考慮職位安全或帝國擴張
6. **內部交叉補貼** — 資源從高回報轉向低回報部門

**重要例外**：Berkshire Hathaway 持續以約 1.5 倍淨值交易，證明卓越管理可以克服甚至反轉多元化折價。

### SOTP 公式

```
Total Enterprise Value = Σ(各業務部門估值)
Equity Value = Total Enterprise Value - 淨負債 - 非營業負債 + 非營業資產
每股價值 = Equity Value / 稀釋後流通股數
```

## 實戰應用

### Step 1：識別業務部門

根據財報附註中的部門別報告（ASC 280 / IFRS 8），識別可報告部門。判斷標準：
- 該部門營收 ≥ 合併營收 10%
- 該部門利潤或虧損 ≥ 利潤或虧損較大者的 10%
- 該部門資產 ≥ 合併資產 10%

### Step 2：對各部門獨立估值

每個部門使用最適合的估值方法：

| 部門類型 | 建議估值方法 | 常用倍數 |
|----------|------------|---------|
| 製造業 | EV/EBITDA | 6-12× |
| 軟體/SaaS | EV/Revenue 或 PEG | 4-8× 營收 |
| 金融（銀行） | Price-to-Book | 1.0-1.5× |
| 金融（保險） | Price-to-Book 或內涵價值法 | 0.8-2.0× |
| 生技（早期） | rNPV（風險調整淨現值） | 依臨床階段 |
| 基礎設施/公用 | EV/EBITDA 或 DCF | 12-15× |
| 礦業/能源 | EV/EBITDAX 或 EV/Reserves | 6-8× |

**找同業比對公司**的要件：
- 同產業或子產業
- 營收成長率 ±5% 以內
- EBITDA 利潤率 ±5% 以內
- 營收規模在 0.5 倍以內

### Step 3：加總各部門估值

將各部門的 Enterprise Value 相加，得到 Total Enterprise Value。

### Step 4：扣除淨負債與調整項

```
Implied Equity Value = Total TEV - 淨負債 - 企業總部成本 - 少數股權 + 非營業資產
```

**企業調整項**：
- **總部開支（Corporate Overhead）**：大型集團通常為營收的 3-5%，可分配至各部門或作為獨立負項（以 5-8× 資本化）
- **少數股權**：非 100% 控股的部門，僅計算持股比例的價值
- **反綜效（Dis-synergy）**：實際拆分時各部門獨立營運的額外成本，通常為各部門營運成本的 0-15%

### SOTP 適用情境

1. **跨產業集團**：不同部門風險/回報特徵差異大，無法用單一倍數
2. **企業重組**：評估分拆（Spin-off）、剝離（Split-off）、股權分拆上市（Carve-out）的效益
3. **維權投資（Activist Investing）**：Elliott Management 推動 Honeywell 三方拆分即為 SOTP 驅動
4. **生技公司**：多個臨床階段資產需獨立計算 rNPV
5. **控股公司**：投資組合型公司（如 Berkshire）需 look-through 估值

### 經典拆分案例

| 案例 | 年份 | 結果 |
|------|------|------|
| **GE 三方拆分** | 2021-2024 | 拆分後三家公司合計市值為拆分前的 **4 倍** |
| **eBay 分拆 PayPal** | 2014 | Carl Icahn 推動，PayPal 上市後市值超過 eBay |
| **J&J 分拆 Kenvue** | 2023 | 約 $41B 估值，J&J 減少 1.91 億股並獲 $13.2B 現金 |
| **Kellogg 拆分** | 2023 | Kellanova 被 Mars 以 $30B 收購，WK Kellogg 被 Ferrero 以 $3.1B 收購，合計 $33.1B |
| **Motorola 拆分** | 2007-2011 | Mobility 被 Google 以 $12.5B 收購（溢價 63%） |

### 台灣實務特點

台灣多數採「母子分割 + 子公司 IPO」模式（equity carve-out），而非純 spin-off 或 split-off。

台大康美雪（2009）研究發現，台灣分割公告的異常報酬偏低（約前後三日）、多數未達統計顯著，原因：
- 國內分割常把高 ROI 部門切出去，投資人擔心金雞母被移轉
- 子公司上市須等待一年以上，分割利益難及時反映
- 公司治理與資訊揭露不如國外成熟

### 多元化折價的量化計算

```
折價率 = (SOTP 價值 - 市值) / SOTP 價值 × 100%
```

**折價幅度判斷**：
- **大折價（15-30%）**：資本配置差、透明度低、歷史收購破壞價值、弱部門拖累
- **小折價（5-10%）**：管理層強、透明度高、綜效明確、各部門績效好
- **無折價或溢價**：Berkshire 等級的卓越管理

## 注意事項

### 五大限制

1. **部門數據有限**：上市公司很少提供足夠的部門別資訊來建立完整模型
2. **假設過度寬鬆**：資訊不足迫使使用更寬泛的假設，降低估值可信度
3. **綜效無法分配**：跨部門的成本節省和綜效難以拆分歸屬
4. **轉撥計價扭曲**：內部交易可能不反映市價，虛增部門利潤
5. **反綜效被忽略**：實際拆分時的額外成本常被低估

### 六大常見錯誤

1. **未分配企業總部成本** — 高估價值 5-10%
2. **重複計算公司間營收** — 內部交易被雙重計入
3. **各部門倍數不一致** — 混用不同基礎的倍數
4. **忽略少數股權** — 非全資部門未按持股比例計算
5. **同業選錯** — 用了不對口的 peer group
6. **使用過時數據** — comparable 資料未更新

### 實際拆分時的調整

建模實際拆分場景時，必須加入：
- **反綜效調整**：5-15%（獨立營運的額外成本）
- **交易成本**：投行費用、法律成本、分離費用，約交易價值的 2-5%
- **稅務影響**：美國 IRC Section 355 允許免稅分拆（需滿足四項條件）
