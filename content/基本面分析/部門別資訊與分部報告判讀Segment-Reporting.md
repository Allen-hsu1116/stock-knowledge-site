# 部門別資訊與分部報告判讀 Segment Reporting

> 合併報表是一塊大餅，部門別資訊是切開後的每一片——看懂每片餅的獲利能力，才能精準估值

## 核心概念

部門別資訊（Segment Information），又稱分部報告，是將公司依業務別或地區別拆分揭露營收、損益、資產等財務數據的資訊。IFRS 8「營運部門」要求強制揭露，是看懂多角化公司最重要的工具。

### 為什麼重要

1. **隱藏的成長引擎**：合併報表掩蓋個別部門的好壞
2. **隱藏的地雷**：虧損部門可能被獲利部門掩蓋
3. **SOTP估值的基礎**：分部加總估值需要部門別數據
4. **資源配置判斷**：看公司把資源配置在哪個部門

### IFRS 8 揭露要求

- 採管理法（Management Approach），以內部管理報告為基礎劃分部門
- 必須揭露：部門營收（外部+部門間）、部門損益、部門資產、部門資本支出、部門折舊攤銷
- 部門負債為選擇性揭露

## 實戰應用

### 三步判讀框架

**第一步：辨識部門**

看合併財報附註或年報的「營運部門」揭露：
- 公司有幾個營運部門？
- 每個部門的業務內容是什麼？
- 部門劃分基礎是什麼（產品別、地區別、客戶別）？

**注意**：有些公司劃分過粗（只分「產品」和「其他」），這本身就是紅旗。

**第二步：計算關鍵比率**

對每個部門計算：
1. 營收佔比 = 部門營收 ÷ 合併營收
2. 營業利益率 = 部門營業利益 ÷ 部門營收
3. 資產佔比 = 部門資產 ÷ 合併資產
4. 資產週轉率 = 部門營收 ÷ 部門資產
5. ROA = 部門營業利益 ÷ 部門資產

**第三步：趨勢分析**

- 哪個部門營收成長最快？
- 哪個部門利潤率在改善/惡化？
- 資本支出正投向哪個部門？
- 部門利潤率趨勢是否可持續？

### SOTP估值的基礎

公司價值 = Σ（各部門價值）- 多角化折價/ + 協同溢價

- 成長部門：用P/E或P/S
- 穩定部門：用P/E或EV/EBITDA
- 虧損部門：用P/S或資產淨值
- 金融部門：用P/B

### 台積電部門別判讀實例

台積電依技術節點劃分部門別資訊：
- 先進製程（5nm及以下）營收佔比上升 = 技術領先優勢擴大
- 先進製程毛利率高但初期資本支出大 = 看資本支出vs利潤率的時間差
- 成熟製程穩定貢獻現金流 = 公司的「現金牛」

## 注意事項

### 五大紅旗

1. **部門劃分過粗**：「其他」部門佔比超過20%，可能隱藏重要資訊
2. **部門間消除異常大**：內部交易佔各部門合計>30%，可能有移轉定價問題
3. **虧損部門被掩蓋**：合併獲利好看但個別部門嚴重虧損
4. **營收集中度過高**：單一部門佔>70%，實質上是單一業務公司
5. **業務別與地區別不匹配**：某地區暴增但無對應部門成長

### 多角化折價

- 市場對多角化公司通常給予13-15%的折價
- 投資人可以自己分散投資，不需要公司分散
- 但部門間有協同效應時（1+1>2），可能享有溢價

### 台灣公司特色

1. 年報必須揭露營運部門資訊（IFRS 8）
2. 部分公司依產品別揭露月營收，可做更即時追蹤
3. 法說會常提供比年報更詳細的部門別數據
4. 外資分析師常自行推估各部門預估
