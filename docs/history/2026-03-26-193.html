二、Straddle 與 Strangle（跨式與勒式）

是什麼

這是波動率策略——不預測漲跌方向，只預測「波動大小」。

跨式 Straddle：買 / 賣同一履約價的 Call + Put 勒式 Strangle：買 / 賣不同履約價的 Call + Put

四種策略總覽

| 策略 | 組成 | 使用時機 | 風險特性 | |------|------|----------|----------| | Long Straddle | 買 Call + 買 Put（同履約價） | 預期大波動 | 損失有限，成本高 | | Long Strangle | 買 Call + 買 Put（不同履約價） | 預期大波動 | 成本較低，需要更大波動 | | Short Straddle | 賣 Call + 賣 Put（同履約價） | 預期盤整 | 勝率高，風險無限 | | Short Strangle | 賣 Call + 賣 Put（不同履約價） | 預期盤整 | 勝率高，風險更大 |

Long Straddle 買進跨式

使用時機： - 重大政策/經濟數據發布前（利率、失業率） - 重要選舉前 - 重大意外後（可能繼續崩盤或大反彈） - 盤整末期（三角收斂快突破）

優點： - 不用賭方向，漲跌都賺 - 大波動報酬可觀 - 風險可預期（最大虧損 = 權利金） - 不需高保證金

缺點： - 成本是單邊兩倍 - 雙倍時間價值衰減

實戰技巧：短期持有幾天就平倉，不必抱到結算。

Short Straddle 賣出跨式

使用時機： - 預期進入盤整 - 成交量低，預期小漲小跌

優點： - 勝率高，天天收時間價值 - 盤整市場很常見

缺點： - 大漲大跌風險無限 - 資金需求高 - 太安逸容易忽視風險，一次意外就畢業

流動性風險：極端大跌時賣權變深價內，可能難以平倉。

Straddle vs Strangle 差異

核心差異：履約價相同 vs 不同

| | Straddle | Strangle | |---|----------|----------| | 履約價 | Call 和 Put 同一履約價 | Call 和 Put 不同履約價 | | 成本 | 較高 | 較低 | | 獲利門檻 | 波動較小即可獲利 | 需要更大波動 | | 風險（賣方） | 風險較集中 | 風險更分散但範圍更大 |

進階關聯

Short Straddle + 價外買 Call + 價外買 Put = Iron Condor（鐵兀鷹） Short Strangle + 價外買 Call + 價外買 Put = Butterfly（蝴蝶）

實戰心法

買方策略：重大事件前佈局，事件後平倉賺波動率上升。 賣方策略：越接近結算，時間價值收越快，抱久一點（但看到危險要跑）。

近年變化：大家都事件前買雙買，事件後停損 → 隱波事件前上升、事件後下降。 → 實務改為：更早買進，事件前平倉；事件後改賣方賺權利金。
