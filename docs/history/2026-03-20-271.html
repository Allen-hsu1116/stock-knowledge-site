2026-03-20 · ⚠️ 風險管理

二、CVaR（條件風險價值）進階概念

CVaR 定義 - CVaR = Conditional Value at Risk = 期望尾端損失（Expected Shortfall） - 核心問題：當損失超過 VaR 時，預期會虧多少？ - CVaR 計算 VaR 點以下的平均損失

VaR vs CVaR 比較

| 特性 | VaR | CVaR | |------|-----|------| | 定義 | 特定信賴水準下的最大損失 | 超過 VaR 後的平均損失 | | 關注點 | 情況會變多糟糕？ | 最糟糕發生時預期虧多少？ | | 次可加性 | ❌ 不滿足 | ✅ 滿足 | | 尾部資訊 | 不考量 | 充分考量 | | 一致性風險測度 | 否 | 是 |

為什麼 CVaR 更好？

VaR 的缺陷： - 不考慮超過閾值後的損失嚴重度 - 不滿足次可加性（組合 VaR 可能大於個別 VaR 之和） - 無法正確反映分散風險的好處

CVaR 的優勢： - 考慮尾端損失的嚴重度 - 滿足次可加性，符合一致性風險測度 - 對估計誤差更敏感 - 更適合風險管理和資本配置

CVaR 公式概念

直觀理解： - CVaR 把 VaR 以下的損失全部加權平均 - 正態分配下：(CVaR - VaR) / VaR ≈ 15%（99% 信賴水準） - 峰度越高，差距越大

計算方法： - 把尾端分成 n 等分 - 計算對應 n-1 分的 VaR - 求期望值

實例（FRM 範例，n=5，常態分配）：

| 信賴區間 | 96% | 97% | 98% | 99% | |---------|-----|-----|-----|-----| | VaR | 1.7507 | 1.8808 | 2.0537 | 2.3263 | | CVaR（平均） | 2.003 |
