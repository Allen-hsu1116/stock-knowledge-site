相關性風險×分散失效×壓力測試實戰

一、相關性風險（Correlation Risk）

### 是什麼 相關性風險是指「你以為分散了，但其實沒有」的風險。在正常時期，不同資產的相關係數可能很低（如股票0.3、商品0.2），讓你覺得組合很分散。但危機來時，所有相關係數急速飆升趨近1，所有資產同時下跌，分散完全失效。

為什麼相關性會在危機時飆升？

流動性機制：

  • 危機時，投資人面臨追繳保證金（Margin Call），需要現金
  • 他們不是「想賣什麼」就賣什麼，而是「能賣什麼」就賣什麼
  • 有流動性的資產首先被拋售（不論基本面），因為急需變現
  • 所有資產通過同一個管道——「對現金的迫切需求」——被連結在一起

歷史證據：

  • 2008年金融海嘯：股票跌、債券跌、商品跌、房地產崩、新興市場崩。原本相關係數0.3的資產飆到0.9以上
  • 2020年3月：S&P 500三週跌34%，投資級公司債同時下跌，黃金也下跌，連美國公債都出現異常波動
  • 結論：危機時，相關性趨近1是常態而非例外

相關性矩陣的謊言

怎麼做：
1. 你用歷史數據算出來的相關性矩陣，絕大部分來自正常市場環境
2. 平靜的日子遠多於危機的日子，所以平均相關性被「正常日」稀釋
3. 你用正常時期的數據設計防護，卻在危機時期需要這個防護——這就是矛盾

實例：

  • 情境：你計算股票與商品的相關係數為0.3，覺得分散效果不錯
  • 現實：0.3是正常時期的數值，危機時可能變成0.9
  • 你以為的分散：0.3相關性 → 風險降低70%
  • 實際的分散：危機時0.9相關性 → 風險幾乎沒降低
  • 你的相關性矩陣量測的是「沒事發生時」的行為，卻用來防護「出事時」的情境

風險平價基金的致命缺陷

  • 風險平價策略對低波動資產（如債券）加槓桿，使風險貢獻與高波動資產（如股票）相等
  • 危機時波動率飆升，必須同時減碼所有資產
  • 這個「同時減碼」行為本身就是一種強制賣壓，讓相關性進一步飆升
  • 越是精巧的風險平價策略，在危機時越是加劇相關性飆升

適用時機：每個投資人都必須理解相關性風險，特別是在建構投資組合時

二、分散失效（Diversification Failure）

### 是什麼 分散失效是指「理論上分散的組合，在危機時全部一起下跌」的現象。這不是理論上的擔憂，而是歷史上反覆出現的紀錄。

三次危機的教訓

1929年大蕭條：

  • 存活者的特徵：持有現金或短期政府票券
  • 他們不需要在底部賣出股票，因為有其他流動性來源
  • 分散不是跨資產類別（這些類別會一起下跌），而是跨流動性特徵

2008年金融海嘯：

  • 表現最好的組合：要麼持有明確的尾部避險工具（如VIX Call），要麼持有同儕認為「過多」的現金
  • 這些現金部位在牛市時看起來是拖累，在崩盤時成為生存保障
  • 分散失效打擊了所有假設相關性會保持穩定的人

2020年疫情崩盤：

  • 最快復甦的組合：能在下跌時加碼的組合（有乾火藥）
  • 能進攻的前提：在危機前就有防禦性配置
  • 那些滿倉的人承受了完整的回撤
### 共同特徵 三個歷史案例的共同教訓： 1. **不依賴相關性假設在壓力下維持穩定** 2. **維持流動性緩衝，避免被迫賣出** 3. **持有結構性避險工具，專為相關性飆升而設計**

過度分散的陷阱（Diworsification）

是什麼：過度分散在相同類型的標的上，結果Alpha被稀釋，風險卻沒有進一步降低。

怎麼做（正確的分散）：
1. 每種策略內精選標的：只挑最有信心的5-10支股票，不是什麼都買一點
2. 跨策略分散：在不同類型的Alpha之間分散（動能、價值、趨勢追蹤、套利等）
3. 追求低相關性的策略組合：不同策略之間相關性低，才是真正的分散

實例：

  • ❌ 錯誤分散：持有30支科技股 → 相關性極高，分散效果差
  • ✅ 正確分散：持有5支精選科技股 + 5支精選金融股 + 5支精選消費股 + 短期公債 + 黃金 → 跨產業、跨資產類別，真正分散

數學基礎：

  • Grinold的主動管理基本法則：IR = IC × √BR
  • 關鍵：BR是「獨立」機會數，不是持股數
  • 30支同產業股票的BR可能只有5-10，因為它們高度相關

研究證據：

  • 哈佛商學院研究：基金經理的「最佳點子」股票，每年超額報酬比其他持股高2.8-4.5%
  • AQR研究：結合5個低相關性因子策略，夏普比率約1.4，遠高於單一策略的0.7
  • 結論：分散的正確做法是「每個籃子裝最優質的雞蛋，同時放在不同類型的籃子裡」

三、壓力測試實戰（Stress Testing）

### 是什麼 壓力測試是模擬極端但合理的市場情境，評估你的投資組合在這些情境下的表現。不是預測未來，而是**確認你的組合在最壞情況下能不能活下來**。 ### 為什麼需要壓力測試？ 1. 回測只告訴你「過去的平均表現」，壓力測試告訴你「極端情況會多慘」 2. VaR（風險值）假設常態分布，但極端事件不是常態的 3. 相關性在壓力時會飆升，只有壓力測試能模擬這種情境

五大壓力情境

情境1：股市崩盤

模擬方式：

  • 股票下跌30-50%
  • 公司債利差擴大200-400基點
  • 黃金上漲5-15%（避險需求）
  • 政府公債初期波動，後期穩定（資金避風港）

怎麼做：
1. 計算你的股票部位在下跌40%時的虧損金額
2. 計算債券部位能否提供足夠緩衝
3. 計算整體組合的回撤是否超過你能承受的範圍
4. 如果超過，減少股票曝險或增加避險部位

實例：

  • 情境：投資組合60%股票、30%債券、10%黃金
  • 股票跌40% → 整體虧損60%×40% = 24%
  • 債券跌5% → 整體虧損30%×5% = 1.5%
  • 黃金漲10% → 整體獲利10%×10% = 1%
  • 組合總回撤：-24% - 1.5% + 1% = -24.5%
  • 判斷：如果你的最大可承受回撤是20%，這個組合需要調整

情境2：利率急升

模擬方式：

  • 利率上升200-300基點
  • 債券價格下跌（長期債跌幅更大）
  • 成長股因折現率上升而大跌
  • 金融股可能受益（利差擴大）
  • 房地產面臨更高融資成本

怎麼做：
1. 計算債券部位的存續期間（Duration）
2. 存續期間×利率變動 = 大約的價格變動
3. 例：存續期間7年的債券，利率上升2% → 價格下跌約14%
4. 評估你的成長股曝險是否過高

實例：

  • 情境：持有100萬長期公債ETF（存續期間7年）和50萬科技股
  • 利率上升3%：公債下跌約21%（7×3%），虧損21萬
  • 科技股下跌約25%，虧損12.5萬
  • 總虧損：33.5萬，佔總資金150萬的22.3%
  • 修正：降低長期債部位，增加短期債和浮動利率債

情境3：地緣政治衝突

模擬方式：

  • 油價飆漲50-100%
  • 相關區域股市暴跌
  • 避險貨幣（日圓、瑞士法郎）升值
  • 黃金上漲
  • 軍工和能源股受益

怎麼做：
1. 評估你的區域曝險比例
2. 評估能源和原物料曝險
3. 確認是否有足夠的避險資產（黃金、避險貨幣）
4. 確認流動性緩衝是否足夠

情境4：經濟長期衰退

模擬方式：

  • 股票持續下跌2-3年
  • 企業獲利大幅下降
  • 失業率飆升
  • 流動性枯竭，信用利差擴大
  • 資產相關性飆升

怎麼做：
1. 計算你的組合在連續3年負報酬下的資金狀況
2. 評估你是否需要在不利的時間點被迫賣出資產
3. 確認你的現金緩衝能否支撐2-3年的生活開銷
4. 評估你的配息收入是否會被削減

實例：

  • 情境：退休族，靠配息生活，年支出60萬
  • 組合：500萬股票ETF（配息率3%）+ 300萬債券ETF（配息率2.5%）+ 200萬現金
  • 股票3年跌40%，配息率從3%降到1.5%
  • 債券配息穩定但價格下跌
  • 年配息收入從22.5萬降到14.25萬
  • 需從本金提取45.75萬/年，3年共137.25萬
  • 200萬現金只夠撐4.3年 → 安全邊際不足，需要調整

情境5：黑天鵝事件

模擬方式：

  • 所有資產同時下跌（相關性趨近1）
  • 流動性瞬間蒸發
  • 市場暫停交易可能
  • 融資強制平倉

怎麼做：
1. 模擬你的組合在所有資產同時下跌30%時的損失
2. 這是最壞情況下的壓力測試
3. 如果你使用融資，計算是否會被強制平倉
4. 評估你的現金能否在市場暫停時支撐生活

壓力測試SOP

Step 1：列出所有部位

  • 股票、債券、ETF、選擇權、現金等
  • 計算每個部位的佔比和相關性

Step 2：選擇3-5個壓力情境

  • 至少包含：股市崩盤、利率急升、黑天鵝
  • 根據你持有的標的加入特定情境（如持有大量科技股就加科技泡沫情境）

Step 3：計算每個情境下的損益

  • 使用歷史數據或假設數值
  • 重點看最大回撤和流動性影響
  • 危機時相關性要設為0.8-1.0（不是平時的0.3）

Step 4：評估可承受性

  • 最大回撤是否超過你的承受範圍？
  • 是否會被迫在不利的時間點賣出？
  • 是否有足夠的現金緩衝？

Step 5：調整組合

  • 如果某個情境的損失不可承受，減少該風險因子的曝險
  • 增加在危機時表現良好的資產（現金、短期公債、避險工具）
  • 確保每個情境下你都能活下來

四、危機組合設計：兩種市場，兩種架構

正常市場架構

  • 最佳化夏普比率和平滑報酬
  • 跨資產分散，追求風險調整後報酬最大化
  • 這是多數人熟悉的投資組合理論

危機市場架構

  • 最佳化最大回撤限制和避免被迫賣出
  • 現金部位比「最佳化」告訴你的還要多
  • 持有在正常時期看似「浪費」的避險工具
  • 牛市時會落後同儕，但崩盤時會存活

兩種架構的選擇不是二選一，而是同時存在：

  • 核心組合（80-90%）：正常市場架構，追求合理報酬
  • 避險組合（10-20%）：危機市場架構，確保生存
  • 這跟之前學的槓鈴策略完全一致

怎麼做：
1. 回測告訴你「平均來說能賺多少」
2. 壓力測試告訴你「最壞情況會虧多少」
3. 兩者結合才能做完整的風險評估
4. 只有回測沒有壓力測試，就像只知道平均體重但不知道最重的能有多重

心得

今天學到的最有衝擊力的概念是「相關性矩陣的謊言」。我們用歷史數據算出的相關性，絕大部分來自正常市場環境。但在危機時——也就是我們最需要分散保護的時候——相關性飆升到接近1，分散完全失效。這就像買了一把傘，晴天時看起來很好用，下雨時卻發現傘是紙做的。

這跟之前學的槓鈴策略完美呼應。塔雷伯說中間風險最危險，現在我理解為什麼了——中間風險的資產在正常時期看似分散，危機時卻跟股票一起跌。真正有效的分散不是「買一堆不同的股票」，而是「持有在危機時行為不同的資產」：現金、短期公債、黃金、VIX。這些在正常時期看起來是「拖累」，但在危機時是「救命」。

過度分散（Diworsification）的概念也讓我重新思考。FinLab那篇研究說得很清楚：30支科技股的分散效果可能不如5支精選科技股+5支精選金融股+5支精選消費股。真正的分散是「跨策略、跨因子」的分散，不是「同類標的越多越好」。Grinold的基本法則IR = IC × √BR告訴我們，增加持股數如果相關性高，獨立投注數（BR）不會增加，IR也不會提升。

壓力測試則是把之前學的「最壞情況會多糟」具體化。以前只在腦中想「如果股市大跌怎麼辦」，現在有了系統化的框架：選情境、算損益、評可承受性、調組合。特別是「危機時相關性設為0.8-1.0」這個原則，比用歷史相關性更能反映真實風險。

最後，三個世紀的危機教訓都指向同一個結論：先求生存，再求報酬。1929年的存活者有現金，2008年的存活者有尾部避險，2020年的存活者有乾火藥可以加碼。設計組合時，先把最壞情況想清楚，確保活下來，然後才追求最佳化報酬。這不是悲觀，是數學。
