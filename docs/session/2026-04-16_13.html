一、過擬合（Overfitting）— 回測完美實盤腰斬的元凶

1. 過擬合是什麼

是什麼：策略過度貼合歷史數據，把市場噪音和隨機波動也當成規律學進去了，導致回測績效亮眼但實盤慘不忍睹。

怎麼發生：
1. 反覆調參追求最佳歷史績效 → 參數被「訓練」到只適合那段歷史
2. 參數越多，過擬合風險越高 → 像考試背答案，換題目就掛了
3. 歷史績效好 ≠ 未來績效好 → 績效好可能只是運氣，不是實力

2. 過擬合的四大成因

成因一：過度優化 — 反覆調整參數讓回測績效最大化
成因二：數據不足 — 回測期間太短或不具代表性
成因三：測試偏誤 — 下意識選擇有利數據集、重複使用OOS資料
成因四：策略過度複雜 — 規則太多、條件太細

二、Walk Forward Optimization（前向優化）

3. WFO 核心概念

將歷史數據切分為多個訓練期（IS）+ 測試期（OOS），滾動前移驗證策略穩健性。訓練期2-4年、測試期3-6個月。

4. WFO vs 傳統 Hold-Out 驗證

WFO將績效時間序列切成S份做排列組合驗證，能計算「過擬合機率」。

FinLab 實證：

  • 差策略：74% 過擬合機率
  • 好策略：4% 過擬合風險

三、參數高原 vs 孤島參數

5. 高原參數與孤島參數

  • 高原參數：附近區域績效都差不多 → 穩健、抗干擾
  • 孤島參數：只有一個精確數值表現好 → 極度敏感、過擬合

選高原參數，不要選孤島參數。

四、WFO 結果判讀與策略失效判斷

6. WFO 績效判讀四指標

1. OOS 整體報酬率
2. OOS 勝率穩定性
3. OOS 最大回撤（不超過IS的2倍）
4. OOS/IS 效率比（>0.5及格，>0.7優秀）

7. 策略失效判斷

  • 連續虧損超過歷史最長連虧 → 降低部位到50%
  • MDD超過歷史1.5倍 → 暫停策略，重新WFO驗證
  • 月報酬連續3個月為負 → 重新優化參數
  • 市場結構改變 → 重新設計策略邏輯

五、回測數據解讀

8. TradingView 回測關鍵數據

  • 夏普比率>1.0及格，>2.0優秀
  • 勝率40%+盈虧比2:1 優於 勝率70%+盈虧比0.5:1
  • 指標≠策略：指標只提供訊號，策略是完整交易規則

心得

這回合學到WFO前向優化，這東西根本就是交易界的「壓力測試」。最實用的概念是「參數高原 vs 孤島參數」——不要選那種只在某個精確數值才有效的參數組合。WFO的核心邏輯很直覺：不要只驗證一次，要滾動驗證很多次。OOS/IS效率比 > 0.5才算過關。FinLab的過擬合機率計算方法很厲害，74%過擬合機率的策略直接丟垃圾桶。策略失效判斷也很重要——知道何時該停損策略本身，比知道何時停損單筆交易更重要。
