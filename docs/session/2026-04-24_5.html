回測驗證×過度擬合×前進分析

一、回測的本質與陷阱

### 回測是什麼 回測（Backtesting）是用歷史市場數據模擬交易策略的過去表現，驗證策略是否有效。聽起來很合理，但這裡藏著一個致命陷阱：**回測的目的是「驗證」而不是「調整」**。

回測的三大陷阱

陷阱1：存活者偏差

  • 你只看到現在還存在的策略，看不到已經被淘汰的
  • 100個隨機策略中，總有幾個在歷史數據上表現優異
  • 但這只是統計上的必然，不代表策略有真正的優勢

陷阱2：過度擬合

  • 對歷史數據反覆調整參數，直到回測結果完美
  • 結果策略「記住」了歷史的噪音，而非捕捉了市場的規律
  • 實盤表現遠不如回測，甚至虧損

陷阱3：未來函數（Look-ahead Bias）

  • 回測時不小心使用了「當時還不知道」的資訊
  • 例如：用當季財報數據回測當季策略，但財報是季末才公布的
  • 這種錯誤讓回測績效虛高，實盤完全無法複製

實例：

  • 情境：你開發了一個均線策略，回測5年數據，年化報酬率40%
  • 問題：你在回測中調了37次參數，每次都看結果再微調
  • 結果：策略完美貼合過去的噪音，實盤第一個月就虧5%
  • 原因：你不是找到規律，你是把參數硬塞進歷史數據

二、過度擬合（Overfitting / Curve Fitting）

### 是什麼 過度擬合是指策略「過度貼合」歷史數據，捕捉到的是噪音而非規律。回測結果完美，但實盤表現崩潰。 ### 為什麼會過度擬合？ 1. **參數太多**：每多一個參數，就多一個「貼合噪音」的機會 2. **反覆調整**：看完回測結果就改參數，改了再看，循環往復 3. **數據太短**：樣本數不足，少數極端事件就被當成規律 4. **邏輯太複雜**：加了太多過濾條件，試圖捕捉每一個細微波動 ### 過度擬合的比喻 想像你在準備考試：
  • 正常學習：理解原理，能舉一反三 → 好的策略
  • 背答案：把考古題的答案全部背下來，換一個題型就全錯 → 過度擬合

怎麼判斷是否過度擬合？

判斷1：樣本內外績效差距

  • 在歷史數據（樣本內）上績效極好
  • 在未見過的數據（樣本外）上績效大幅下滑
  • 如果樣本外績效比樣本內差超過30%，大概率過度擬合

判斷2：參數高原 vs 孤島

  • 參數高原：最佳參數附近的小幅調整，績效變化不大 → 策略穩健
  • 孤島參數：最佳參數一微調，績效就崩潰 → 過度擬合

實例：

  • 情境A：策略最佳參數是MA=20，但你測試MA=18和MA=22時，績效只差5%
  • 情境B：策略最佳參數是MA=17，但MA=16和MA=18時績效差了50%

判斷3：策略邏輯是否說得通

  • 如果策略的邏輯無法用簡單語言解釋為什麼有效，大概率是過度擬合
  • 例：「當RSI=23.7且MACD柱狀體在-0.032且成交量是均量的1.47倍時買入」→ 這些精確數字就是過度擬合的產物

過度擬合的機率量化（Bailey et al. 方法）

怎麼做：
1. 將回測績效時間序列切成S份
2. 任選S/2份當作樣本內（IS），其餘當作樣本外（OOS）
3. 在IS找到最佳參數，測試在OOS的表現
4. 重複大量組合，觀察最佳參數在OOS的排名分布

判讀：

  • 如果最佳參數在OOS的排名大部分位於前半段 → 策略穩健
  • 如果最佳參數在OOS的排名大部分位於後半段 → 過度擬合
  • FinLab的實驗：74%的機率最佳化後效果比中位數差 → 這就是過度擬合

三、樣本外測試（Out-of-Sample Test）

### 是什麼 將歷史數據分為「樣本內（In-Sample, IS）」和「樣本外（Out-of-Sample, OOS）」兩部分。IS用來開發和優化策略，OOS只用來驗證，絕不用來調整。 ### 怎麼做 1. **資料分割**：將歷史數據分為兩段
  • 樣本內（IS）：2015-2020年，用來開發策略和優化參數

2. IS優化：在IS期間找到最佳參數
3. OOS驗證：用最佳參數跑OOS期間，觀察績效
4. 比較：如果OOS績效比IS差超過30%，策略可能過度擬合

樣本外測試的隱藏陷阱

陷阱1：反覆偷看

  • 第一次OOS結果不好 → 去改策略 → 再跑一次OOS
  • 改了3次後OOS終於好看 → 你已經把OOS也變成IS了
  • 解法：OOS只能跑一次，跑完就不准再調整

陷阱2：邏輯最佳化取代參數最佳化

  • 參數不調了，但改了策略邏輯又重新跑OOS
  • 這跟調參數一樣，都是對OOS數據的過度擬合
  • 解法：策略邏輯也只准在IS期間修改

陷阱3：數據洩露（Look-ahead Bias）

  • 在回測時使用了「當時還不知道」的資訊
  • 例如：用2022年的整年數據計算指標，但回測的是2022年1月的交易
  • 解法：確保每個時間點只用「到那個時間為止」的數據

實例：

  • 情境：你開發了一個均線策略，IS期間（2018-2022）年化報酬25%
  • OOS期間（2023-2024）年化報酬18%
  • OOS/IS比率 = 18/25 = 72%，衰退28%
  • 判讀：衰退在合理範圍（<30%），策略穩健
  • 如果OOS年化只有8%，比率 = 8/25 = 32%，衰退68% → 過度擬合

四、前進分析（Walk Forward Analysis）

### 是什麼 前進分析是目前最被推崇的策略驗證方法，由Robert Pado提出。核心概念是**模擬真實交易環境**——先用歷史數據優化策略，再用「未來」的數據驗證，然後時間窗口往前移，重複這個過程。

前進分析 vs 傳統回測

比較傳統回測前進分析
優化全部歷史數據只用訓練期間
驗證同一段數據獨立的測試期間
問題容易過度擬合降低過度擬合風險
真實度低（看到全貌才下單）高（模擬真實決策過程）

前進分析五步法

Step 1：設定時間窗口

  • 訓練期（IS）：例如2年（如2020-2021）
  • 測試期（OOS）：例如6個月（如2022上半年）
  • 比例建議：訓練期與測試期比例至少3:1

Step 2：在訓練期優化策略

  • 在2020-2021年的數據上找到最佳參數
  • 確保參數落在「高原」而非「孤島」

Step 3：在測試期驗證策略

  • 用Step 2找到的最佳參數，跑2022上半年的數據
  • 不調整任何參數，只看結果

Step 4：窗口前移

  • 訓練期往前移6個月：2020下半年-2022上半年
  • 測試期往前移6個月：2022下半年
  • 重複Step 2-3

Step 5：彙整所有OOS結果

  • 把所有測試期的結果串起來
  • 這就是策略在「模擬真實環境」下的績效
  • 如果OOS績效穩定且為正，策略才有實盤價值

實例：

  • 訓練期2年、測試期6個月，跑5輪前進分析

  • 如果某輪OOS/IS < 50%，表示該期間策略失效，需要調查原因

前進分析效率比（Walk Forward Efficiency）

公式：

WFE = OOS年化報酬 / IS年化報酬 × 100%

判讀：

  • WFE > 50%：策略穩健，可考慮實盤
  • WFE 30-50%：策略有部分有效性，需謹慎使用
  • WFE < 30%：策略大概率過度擬合，不建議實盤

五、避免過度擬合的七大對策

對策1：增大樣本規模

  • 樣本數太少，噪音的影響就大
  • 交易次數至少要有100筆以上才有統計意義
  • 如果策略每月只交易1-2次，需要10年以上的歷史數據

怎麼做：

  • 回測期間至少3-5年
  • 交易頻率太低的策略，考慮在多個標的上測試
  • 樣本規模越大，統計信賴度越高

對策2：策略不要太複雜

  • 邏輯越複雜，越容易過度擬合
  • 需要最佳化的參數越少越好，理想情況不超過3-4個
  • 根據你想捕捉的行情設定邏輯，不要亂加過濾條件

怎麼做：

  • 先用最簡單的邏輯測試，確認有正期望值
  • 再逐步加入條件，每加一個都要確認OOS績效是否提升
  • 如果加了條件反而OOS變差，果斷移除

對策3：選擇參數高原而非孤島

  • 最佳化時，不要選最高績效的參數
  • 選擇績效穩定的參數區域（高原）
  • 高原上的參數即使微調，績效變化不大

怎麼做：

  • 畫出參數-績效3D圖或熱力圖
  • 觀察最佳參數附近是否形成「高原」
  • 如果只有單一參數績效突出（孤島），果斷放棄

實例：

  • MA參數15-25之間績效都穩定在年化15-18% → 參數高原，選MA=20
  • MA=17時年化25%，但MA=16和18時年化只有5% → 孤島，不選

對策4：不選最好結果

  • 最佳化結果呈現「山形」分布
  • 最高峰（最好結果）可能是噪音造成的
  • 選擇山峰附近（高原），忽略微小的差異

對策5：前進分析驗證

  • 用前進分析取代單一樣本外測試
  • 確保策略在不同時間段都有穩定表現
  • 前進分析的OOS績效才是策略真實能力的指標

對策6：K折交叉驗證

  • 將數據切成K份（通常5或10）
  • 每次用K-1份訓練，1份測試
  • 重複K次，確保每份數據都當過測試集
  • 優點：比單一樣本外測試更充分利用數據

怎麼做：

  • 將5年數據切成5份
  • 第1次：用第1-4份訓練，第5份測試
  • 第2次：用第1-3+5份訓練，第4份測試
  • 依此類推，5次測試結果的平均就是策略估計績效

注意：K折交叉驗證在時間序列中要小心「未來函數」問題，必須確保訓練數據的時間都在測試數據之前。

對策7：組合分割交叉驗證（CPCV）

  • 由Marcos Lopez de Prado提出
  • 解決前進分析和K折交叉驗證只能驗證一條路徑的問題
  • 生成多條回測路徑，得到夏普比率的分布而非單一數值
  • 更能評估策略的穩健性

怎麼做：

  • 將數據切成N份，任選S份當IS，其餘當OOS
  • 組合數 = C(N, S)，可以產生大量驗證路徑
  • 觀察最佳參數在所有OOS路徑中的排名分布
  • 如果大部分路徑都排名前列 → 策略穩健
  • 如果排名分散或偏後 → 過度擬合

六、回測績效的正確解讀

回測績效一定會衰退

  • 回測績效 → 實盤績效的衰退率通常在30-50%
  • 原因：滑價、手續費、流動性、市場衝擊、過度擬合
  • 預期實盤績效 = 回測績效 × (1 - 衰退率)
  • 回測年化20% → 實盤預期10-14%

五個回測績效指標必看

1. 夏普比率（Sharpe Ratio）

  • 年化報酬率 / 年化波動率
  • > 1.0 可接受，> 2.0 優秀，< 0.5 不值得
  • 高夏普比率的策略加槓桿後優於高報酬低夏普的策略

2. 最大回撤（MDD）

  • 從高點到低點的最大跌幅
  • > 30% 需要極強的心理素質才能承受
  • 建議控制在 15-20% 以內

3. 盈虧比（Profit Factor）

  • 總獲利 / 總虧損
  • > 1.5 可接受，> 2.0 優秀
  • < 1.2 的策略交易成本就吃掉大部分利潤

4. 交易次數

  • 至少100筆以上才有統計意義
  • 少於30筆的回測結果不可信

5. OOS/IS比率

  • 樣本外績效 / 樣本內績效
  • > 70% 策略穩健
  • 50-70% 可接受
  • < 50% 大概率過度擬合

回測常見錯誤清單

錯誤描述正確做法
未來函數用了當時不知道的數據確保每個時間點只用已知數據
忽略交易成本不計手續費、滑價至少加入0.1-0.3%交易成本
生存者偏差只看還在上市的股票包含已下市股票
過度最佳化反覆調參追求完美只調2-3次，之後用OOS驗證
起止點偏見特定起止日期結果特別好測試不同起始日的穩健性
忽略流動性假設無限量可以買賣加入成交量限制和滑價

七、實戰SOP：從回測到實盤的驗證流程

### Phase 1：策略開發（只用IS數據） 1. 用簡單邏輯建立策略原型 2. 在IS期間優化參數（最多調整3輪） 3. 確認參數落在高原而非孤島 4. 記錄IS績效指標 ### Phase 2：樣本外驗證（只用OOS數據，不准調整） 1. 用IS最佳參數跑OOS數據 2. 計算OOS/IS比率，目標 > 70% 3. 如果比率 < 50%，回到Phase 1重新設計 4. 只允許跑一次OOS，不准反覆修改再驗證 ### Phase 3：前進分析 1. 設定訓練期（2年）和測試期（6個月） 2. 跑至少5輪前進分析 3. 確認每輪OOS績效都為正 4. 計算WFE（前進分析效率比），目標 > 50% ### Phase 4：紙上交易（Forward Testing） 1. 用模擬帳戶跑1-3個月 2. 記錄每筆交易的滑價、成交狀況 3. 對比回測績效和紙上交易績效 4. 如果衰退超過50%，需要重新檢討 ### Phase 5：小資金實盤 1. 用最小部位進行實盤交易 2. 至少交易50筆以上再評估 3. 比較實盤與回測的差距 4. 如果差距在預期範圍內（30-50%衰退），逐步放大部位

心得

回測驗證這個主題可能是風險管理中最被忽略的。很多人花大量時間開發策略、優化參數，卻從不驗證策略是否過度擬合。結果就是「回測之神，實盤之鬼」。

最讓我印象深刻的是「參數高原vs孤島」這個概念。以前我一定會選回測績效最好的參數，但現在理解了：最好的參數可能是噪音造成的假象，穩健的參數高原才是真正的策略信號。這跟之前學過的很多概念呼應——不要追求極致，穩健才是長期獲利的基礎。

前進分析（Walk Forward Analysis）是今天最實用的工具。傳統回測只用一段歷史數據驗證，但前進分析模擬了真實的交易決策過程——你永遠只能用「到目前為止」的數據來決定策略，然後用「未來」的數據來驗證。這才是真實的交易環境。

另外，FinLab文章中提到的過度擬合機率量化方法也很震撼——74%的情況下，最佳化後的參數在樣本外比中位數還差。這說明大多數人以為自己在「優化」，其實是在「過度擬合」。這個數字應該貼在每個交易者的螢幕上，每次想調參數的時候就看一下。

最後，回測績效衰退30-50%這個經驗法則很實用。如果回測年化20%，實盤預期10-14%就該滿意了。設定合理預期，才不會在實盤表現不如回測時懷疑策略、頻繁更換——而頻繁更換策略本身，就是另一種形式的過度擬合。
