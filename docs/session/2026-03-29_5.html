核心知識

交易系統基本概念

是什麼：一套規則和程序的組合，決定什麼時候買、什麼時候賣，基於市場分析、歷史數據和技術指標制定的策略。

為什麼需要交易系統：

  • 人是感性的，容易受情緒影響
  • 系統幫助客觀評價投資標的合理價位
  • 避免追高殺低的壞習慣
  • 控制投資風險，適當分散配置
  • 回饋投資績效，穩固投資信念

怎麼做：
1. 先檢視自己的投資性格（風險承受度、時間週期）
2. 建立規則化的進出場條件
3. 嚴格執行，不受情緒干擾

交易系統六大組成要素

是什麼：完整交易系統需要涵蓋六大核心要素，缺一不可。

怎麼做：

1. 市場選擇（買什麼）

  • 選擇你熟悉的市場（台股、美股、期貨、選擇權）
  • 建立適合自己投資屬性的資源池
  • 定期汰換資源池個股

實例：

  • 情境：波段交易者
  • 操作：選擇台股50成分股作為核心池，每月檢視一次
  • 適用時機：有時間研究基本面者

2. 進場訊號（何時買）

  • 明確定義買入條件
  • 可以是技術訊號、基本面訊號或籌碼訊號

實例：

  • 情境：均線突破策略
  • 操作：20日均線黃金交叉60日均線時進場
  • 台積電案例：2024年1月週K圖月季線黃金交叉，進場買入

3. 出場訊號（何時賣）

  • 可以是獲利出場或停損出場
  • 必須事先設定，不能臨場決定

實例：

  • 情境：均線策略出場
  • 操作：20日均線死亡交叉60日均線時出場
  • 適用時機：波段持有，不適合當沖

4. 停損設定（怎麼停）

  • 保護資本的最重要一環
  • 設定在進場當下，不是虧損後才想

實例：

  • 情境：突破策略停損
  • 操作：買入後停損設在突破K棒的低點
  • 風險控制：單筆交易風險不超過總資金的2%

5. 部位管理（買多少）

  • 決定每筆交易投入多少資金
  • 避免單一交易造成重大傷害

實例：

  • 情境：總資金100萬
  • 操作：單筆交易風險2%，停損5%，則部位 = 2% ÷ 5% = 40萬
  • 進階：根據波動度調整部位大小

6. 心態與紀律（怎麼執行）

  • 嚴格遵守自己設定的規則
  • 持續學習，不斷改進

實例：

  • 情境：連續虧損三次
  • 操作：暫停交易一週，檢討策略是否失效
  • 適用時機：任何交易系統都需要搭配心態管理

交易規則設計實戰

是什麼：清晰且易於執行的具體規則，指導何時進出場。

盤整區間突破策略範例：

定義盤整區間：股價在相對狹窄範圍內波動一段時間，買賣雙方力量大致平衡。

進場規則：

  • 買入：價格突破盤整區間上限（買方力量占優）
  • 賣出：價格跌破盤整區間下限（賣方力量較強）

進階優化（回調確認法）：
1. 等待突破後回調
2. 確認原盤整區間上緣成為支撐
3. 在支撐附近買入

怎麼做：
1. 識別盤整區間：至少10個交易日，波動範圍小於10%
2. 等待突破：價格收盤突破區間上緣
3. 回調確認：價格拉回測試上緣不破
4. 進場買入：在支撐確認後進場
5. 設定停損：設在突破K棒低點

實例：

  • 情境：某股票在100-105元盤整兩週
  • 操作：突破105元後，回調至105元附近獲支撐，進場
  • 停損：設在102元（突破K棒低點）
  • 適用時機：波段操作，不適合極短線

優點：

  • 過濾假突破訊號
  • 提供更好的進場價格
  • 停損點更接近進場點，提高賺賠比

回測：驗證系統有效性

是什麼：用歷史數據檢驗交易策略是否有效。

兩種方式：

1. 人工回測

  • 查看歷史K線圖，模擬交易系統操作
  • 記錄超過100次交易的勝率、最大回撤、賺賠比
  • 缺點：耗時、容易受主觀判斷影響
  • 適用：複雜策略如SMC

2. 程式化回測

  • 將策略寫成程式碼
  • 自動在大量歷史數據上運行
  • 優點：快速、精確、可調整參數
  • 適用：規則明確的策略

怎麼做：
1. 定義進出場規則（可程式化）
2. 準備歷史數據（至少3年以上）
3. 執行回測，記錄每筆交易
4. 計算關鍵指標：勝率、賺賠比、最大回撤、夏普值
5. 評估是否值得實戰

實例：

  • 情境：均線交叉策略回測
  • 操作：使用台股10年日K數據，測試20/60日均線交叉
  • 可能結果：勝率45%，賺賠比1.8，最大回撤25%

注意：過去績效不代表未來，但回測是評估策略的必要起點。

系統交易 vs 主觀交易

系統交易（Systematic Trading）：

  • 完全依靠演算法或規則
  • 所有買賣訊號由程式或明確條件觸發
  • 交易者不需臨場判斷
  • 優點：紀律性強、避免情緒干擾
  • 缺點：特殊行情下可能僵化

主觀交易（Discretionary Trading）：

  • 依靠交易者經驗、判斷力與市場感覺
  • 靈活應對突發狀況
  • 優點：彈性大，可臨機應變
  • 缺點：容易受情緒影響，紀律難維持

比較表：

面向主觀交易系統交易量化交易
決策方式經驗、直覺、盤感明確規則數學模型、程式
規則化程度低中高極高
可否回測幾乎無法可以大規模回測
情緒影響高低幾乎零
技術需求低中高
適合的人盤感強、反應快喜歡紀律喜歡數據、程式

關鍵區別：

  • 系統交易重點在「紀律規則化」
  • 量化交易 = 系統交易 + 數學建模/程式化

實務建議：很多專業交易者會混合使用，以系統交易做核心，再搭配主觀判斷調整部位。

交易系統優化方法

是什麼：根據回測和實際交易結果，持續調整系統以適應市場變化。

調整時機：

  • 連續出現虧損
  • 表現明顯低於預期
  • 市場條件發生顯著變化
  • 發現更有效的技術指標

優化方向：
1. 更改入場條件（更嚴格或更寬鬆）
2. 調整出場策略（停利停損）
3. 引入新的過濾條件
4. 調整部位管理規則

注意陷阱：

  • 避免過度優化（Overfitting）
  • 參數不要調到完美符合歷史數據
  • 樣本外測試驗證

怎麼做：
1. 記錄每筆交易的理由和結果
2. 月度檢討：哪些成功、哪些失敗
3. 識別模式：什麼情況下策略失效
4. 調整規則：只改變一個變數
5. 重新回測驗證

成功交易者的心態與習慣

持續學習：

  • 市場是動態環境，不斷變化
  • 更新知識庫和技能
  • 學習新策略、新工具
  • 深入市場心理學

自律與耐心：

  • 設立明確規則並嚴格遵守
  • 不讓情緒影響交易決策
  • 等待完全符合策略的機會
  • 耐心等待進場點和結果

實例：

  • 情境：看到強勢股大漲想追
  • 自律操作：檢查是否符合進場條件，不符合就等待
  • 不自律操作：追高買入，結果被套

心得

這回合學了「交易系統設計」，這是一個整合性的主題。之前學了很多技術分析、籌碼面、風險管理，但這些都只是零件；交易系統就是要把這些零件組裝成一輛能開的車。

最關鍵的體悟是：沒有交易系統，再好的技術分析都沒用。你可以會看K線、會判斷籌碼、懂風險管理，但如果進場靠感覺、出場看心情、停損捨不得，那最後還是賠錢。交易系統就是把所有知識「規則化」，讓執行變得像刷牙一樣自動化。

幾個重點：
1. 六要素缺一不可：市場選擇、進場訊號、出場訊號、停損設定、部位管理、心態紀律。少一樣就不是完整系統。
2. 用盤整突破舉例很清楚：盤整區間突破後的回調確認法，既過濾假突破，又提供更好進場點。這個策略本身就很完整了。
3. 回測是驗證的關鍵：不要憑感覺認為策略有效，要實際用歷史數據驗證。100次以上才有參考價值。
4. 系統 vs 主觀不是對立：很多專業交易者是混合使用。核心用系統，關鍵時刻用主觀判斷調整。
5. 系統是活的：市場會變，系統也要優化。但優化不是每虧一次就改規則，而是要有系統性的檢討機制。

最後一句話總結：交易系統就是把「我知道」變成「我做到」的橋樑。知道均線黃金交叉是買點沒用，要寫成規則、設定停損、管理部位、嚴格執行，才是交易系統。
