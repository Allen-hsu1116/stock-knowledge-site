MDD回撤管理×降速協定×再平衡策略

一、MDD最大回撤深度解析

### 是什麼 MDD（Maximum Drawdown）是在一段觀察期間內，資產從歷史最高點跌到後續最低點的最大跌幅百分比。它不是平均下跌，不是單日最大虧損，而是「投資人最痛的那一段」。

計算方法

回撤(DD) = 1 − (當前淨值 ÷ 歷史最高淨值)
MDD = MAX(所有回撤)

實例：

  • 淨值走勢：100 → 120 → 115 → 130 → 90 → 95 → 140
  • 120→115：回撤 = (120-115)/120 = 4.17%
  • 130→90：回撤 = (130-90)/130 = 30.77% ← 最大
  • MDD = 30.77%

MDD三大誤區

誤區1：歷史MDD不代表未來MDD

  • 歷史最大回撤只是「過去發生過的最慘」，未來可能更慘
  • 2022年前長期公債MDD只有-23.1%，2022年升息後飆到-45.3%
  • 任何策略執行夠久，都會遇到比歷史更大的MDD

誤區2：停損不一定能改善MDD

  • MDD常常是連續多筆虧損造成的，不是一筆交易
  • 停損設越緊越容易被觸發，連續小虧損累積也可能產生大回撤
  • 管理MDD的重點在控制部位大小，不是設停損

誤區3：拿MDD做槓桿最佳化

  • 用歷史MDD反推最大槓桿是危險的
  • 未來MDD可能比歷史更大，槓桿最佳化越精確，意外時傷害越大
  • 正確做法：把歷史MDD再高估1.5-2倍，當作壓力測試的基準

適用時機：建立任何投資組合前，先看各資產的歷史MDD，評估自己能否承受

二、MDD的三個維度：深度、期間、恢復

不只看深度，還要看期間

回撤期間（Drawdown Duration）：從創高點開始下跌，到重新創新高為止所經歷的時間。

為什麼期間很重要：

  • 30%的回撤3週就回復 → 短痛，可以忍
  • 30%的回撤卡了3年 → 長期折磨，心理崩潰
  • 兩者MDD相同，但投資人的體感完全不同

恢復難度公式：

恢復所需漲幅 = MDD ÷ (1 - MDD)
MDD恢復所需漲幅難度
-10%+11.1%容易
-20%+25.0%中等
-30%+42.9%困難
-40%+66.7%極困難
-50%+100%需要翻倍
-60%+150%幾乎不可能

怎麼做：
1. 設定你能承受的最大MDD（建議15-20%）
2. 根據MDD反推部位大小和槓桿上限
3. 預設恢復時間（MDD 20%通常需要6-18個月恢復）
4. 確保這段恢復期間你有足夠的生活現金

Calmar Ratio：MDD的品質衡量

Calmar Ratio = 年化報酬率 ÷ MDD
Calmar Ratio品質
< 0.5差，承擔太多風險換太少報酬
0.5-1.0可接受
1.0-2.0良好
> 2.0優秀

實例：

  • 策略A：年化15%，MDD 30%，Calmar = 0.5
  • 策略B：年化12%，MDD 10%，Calmar = 1.2
  • 結論：策略B風險調整後報酬更優，單位痛苦換到的報酬更多

三、回撤降速協定（Drawdown Throttling Protocol）

### 是什麼 當帳戶出現回撤時，自動降低交易部位和風險的系統化規則。就像車子遇到危險路段自動降速，不是靠駕駛的意志力，而是靠系統強制執行。

為什麼需要降速協定

  • 連虧時心理壓力最大，最容易做出最差的決策（報復性交易、加大部位）
  • 人性的弱點：虧損時想翻本，贏利時想加碼——兩個都是錯的
  • 降速協定把「何時減速」的決定交給規則，不是交給情緒

三色燈降速系統

綠燈（正常交易）：

  • 回撤 0-10%
  • 單筆風險：2%（或你設定的正常比例）
  • 正常部位大小
  • 正常交易頻率

黃燈（降速交易）：

  • 回撤 10-20%
  • 單筆風險：降至1%（正常的一半）
  • 部位大小減半
  • 只做最有把握的交易，過濾條件加倍嚴格

紅燈（暫停交易）：

  • 回撤超過20%
  • 停止所有新交易
  • 只管理現有部位（移動停損、分批出場）
  • 完整覆盤：策略失效？市場類型改變？執行偏差？
  • 訂定恢復條件：資金曲線回升5%後才重新開始

實例：

  • 帳戶100萬，正常單筆風險2%（2萬）
  • 連虧後帳戶降到88萬（回撤12%），進入黃燈
  • 單筆風險降至1%（8,800元），部位減半
  • 帳戶回升到90萬以上，但未超過歷史高點 → 維持黃燈
  • 帳戶回升到95萬以上 → 恢復綠燈，風險回到2%
  • 如果帳戶跌破80萬（回撤20%）→ 紅燈，停止交易

降速的數學邏輯

為什麼降速是對的：

  • 回撤10%後，恢復需要11.1%漲幅
  • 如果此時維持原部位，再跌10%就變成回撤19%（需要24.7%恢復）
  • 降速後，即使繼續虧損，幅度也更小
  • 數學證明：在回撤期降低風險，長期複合報酬率更高

反直覺但正確：降速不是「放棄」，而是「保護本金以便更快恢復」。回撤20%需要25%恢復，但回撤40%需要67%恢復——保護本金比追求報酬更重要。

四、再平衡策略（Rebalancing）

### 是什麼 再平衡是將投資組合中各資產的比例，定期或定條件調整回原始目標配置的過程。隨著市場波動，各資產比例會偏離設定，再平衡就是把比例拉回來。

為什麼需要再平衡

  • 股票大漲後，股債比可能從70:30變成85:15，風險曝露大幅增加
  • 不再平衡 = 讓市場決定你的風險配置，而不是你自己決定
  • 再平衡的核心是「賣高買低」——這是反人性的，但數學上是對的

四種再平衡方法

方法1：定期再平衡（Calendar Rebalancing）

怎麼做：
1. 設定固定時間間隔（每季、每半年、每年）
2. 到時間就檢視各資產比例
3. 將偏離目標的資產調整回目標比例

實例：

  • 原始配置：股票70%、債券30%
  • 一年後：股票漲到80%、債券20%
  • 再平衡：賣出10%的股票，買入10%的債券，回到70:30

優點：簡單、不需要盯盤、交易成本可控
缺點：可能錯過大幅偏離時的及時調整

研究證據：回測1987-2021年，每月、每季、每年再平衡的年化報酬和波動差異不大（<0.5%），但交易成本差很多。結論：每半年或每年再平衡即可。

方法2：臨界再平衡（Threshold Rebalancing）

怎麼做：
1. 設定偏離門檻（如5%或10%）
2. 當任何資產偏離目標比例超過門檻時，執行再平衡
3. 不用等固定時間，偏離就調整

三種調整方式：

  • 調回目標：偏離超過門檻，直接調回目標比例（最激進）
  • 調回區間邊緣：偏離超過門檻，只調到再平衡區的邊緣（最保守）
  • 調到可容忍區：偏離超過門檻，調到可容忍區內（折衷）

實例（目標70%股票，再平衡區±5%，可容忍區±3%）：

  • 股票漲到78%（偏離8%，超過5%門檻）
  • 調回目標：賣到70%
  • 調回區間邊緣：賣到75%
  • 調到可容忍區：賣到73%

優點：及時捕捉大幅偏離，在極端行情時更有保護效果
缺點：需要更頻繁地監控，可能增加交易次數

方法3：動態再平衡（Tactical Rebalancing）

怎麼做：
1. 根據市場狀況動態調整目標比例
2. 多頭市場：股票比例提高（如70:30→80:20）
3. 空頭市場：債券比例提高（如70:30→50:50）
4. 調整幅度有上限，不是全進全出

實例：

  • 正常配置：股票70%、債券30%
  • VIX<15（市場平穩）：股票75%、債券25%
  • VIX>25（市場恐慌）：股票50%、債券50%
  • 觸發條件明確，不靠感覺

優點：結合擇時和再平衡，有機會提升報酬
缺點：需要判斷市場狀況，判斷錯誤可能比不調整更差

方法4：現金流再平衡（Cash Flow Rebalancing）

怎麼做：
1. 有新資金投入時，買入比例偏低的資產
2. 需要提取資金時，賣出比例偏高的資產
3. 不需要額外交易，在正常現金流中完成再平衡

實例：

  • 目標配置：股票70%、債券30%
  • 目前：股票75%、債券25%
  • 每月定額5,000元投入
  • 再平衡：5,000元全部買入債券，不買股票
  • 直到比例回到70:30後，再按70:30分配

優點：零額外交易成本，最適合定期定額投資人
缺點：只適用於有持續現金流入的情況，大偏離時可能不夠

再平衡的實務建議

什麼頻率最好？

  • Vanguard研究：每年和每季再平衡的報酬差異不到0.5%
  • 但越頻繁交易成本越高
  • 建議：每半年或每年再平衡一次，搭配5-10%偏離門檻的混合式方法

混合式再平衡（推薦）：
1. 每半年定期檢視
2. 平時如果偏離超過10%就立即再平衡
3. 有新資金時優先買入偏低資產

再平衡的心理挑戰：

  • 再平衡要你賣掉漲最多的資產，買入跌最多的資產
  • 這完全反直覺——人性是追漲殺跌
  • 2008年金融海嘯時，再平衡要你賣債券買股票——你敢嗎？
  • 但數據證明：在極端市場，再平衡的效果最大

五、MDD管理+降速協定+再平衡的整合框架

整合SOP

平時（綠燈區，回撤0-10%）：
1. 正常交易，單筆風險2%
2. 每半年執行一次資產再平衡
3. 每月檢視MDD和回撤期間
4. 新資金按目標配置投入

警戒（黃燈區，回撤10-20%）：
1. 單筆風險降至1%，部位減半
2. 觸發臨界再平衡：偏離超過5%就調整
3. 增加避險部位（Put、反向ETF）
4. 只做高勝率交易，過濾條件加倍
5. 每週覆盤（而不是每月）

危機（紅燈區，回撤超過20%）：
1. 停止所有新交易
2. 全面再平衡：賣高買低，回到防禦配置
3. 啟動尾部保護（VIX相關、Put）
4. 完整策略覆盤
5. 資金曲線回升5%後才恢復交易（從黃燈開始，不直接跳綠燈）

Calmar Ratio監控

怎麼做：
1. 每月計算投資組合的Calmar Ratio
2. Calmar > 1.0 → 綠燈，正常操作
3. Calmar 0.5-1.0 → 黃燈，降速
4. Calmar < 0.5 → 紅燈，暫停並覆盤

實例：

  • 組合年化報酬18%，MDD 15%，Calmar = 18/15 = 1.2 → 綠燈
  • 報酬降至12%，MDD擴大到25%，Calmar = 12/25 = 0.48 → 紅燈
  • 判斷：策略可能失效，需要暫停並覆盤

六、追蹤回撤 vs 固定回撤

追蹤回撤（Trailing Drawdown）

  • 從歷史最高點計算回撤
  • 每次創新高，回撤歸零
  • 最常見的MDD計算方式

固定回撤（Fixed Drawdown）

  • 從年初或某個固定起點計算回撤
  • 不會因為創新高就歸零
  • 更保守的風險衡量方式

實例：

  • 年初100萬，最高漲到120萬，現在110萬
  • 追蹤回撤 = (120-110)/120 = 8.3%
  • 固定回撤 = (100-110)/100 = -10%（從年初算是+10%）
  • 追蹤回撤更適合衡量「你距離高峰跌了多少」
  • 固定回撤更適合衡量「你今年整體表現如何」

心得

MDD、降速協定、再平衡——這三個主題串起來，就是一個完整的「回撤管理系統」。

MDD告訴你「最壞會多慘」，降速協定告訴你「慘的時候怎麼辦」，再平衡告訴你「平時怎麼預防」。

最有感的洞見是「停損不一定能改善MDD」。以前以為設好停損就萬事OK，但MDD常常是連續小虧損造成的，不是一筆大虧損。連虧10筆各2%，加起來就是18%的回撤——這不是停損能解決的，這是部位管理和系統性風險管理的問題。

降速協定的三色燈系統特別實用。綠燈正常、黃燈減速、紅燈暫停，這跟開車一樣——你不會在山路急彎還踩油門。交易也一樣，回撤越大，速度越要放慢。最反直覺的是：降速反而能提升長期報酬，因為保護本金的速度比恢復本金的速度快得多。回撤20%需要25%恢復，但如果降速後只回撤12%，只需要13.6%恢復——差距是10個月的複利。

再平衡最難的是心理。2008年叫你賣債券買股票，2020年3月叫你賣避險資產買暴跌的股票——這需要「反人性」的紀律。但數據很清楚：在極端市場再平衡的效果最大。Vanguard的研究說每半年和每年再平衡差異不大，但混合式方法（定期+偏離門檻）在極端年份表現最好。

最後，整合框架的關鍵是：不是三個獨立工具，而是三個互相配合的防禦層。MDD是偵測器（告訴你風險多大），降速協定是煞車（告訴你何時減速），再平衡是方向盤（告訴你如何回到正軌）。三個一起用，才是完整的回撤管理。

Kill Switch×帳戶保護×交易系統災難復原

一、Kill Switch（緊急停止開關）

### 是什麼 Kill Switch是交易系統的「緊急停止開關」——當系統偵測到異常（部位超限、虧損過快、訂單異常暴增），自動或手動切斷所有交易程式的下單能力，瞬間凍結帳戶，防止災難擴大。就像電路保險絲一樣，電流過大就斷電保命。

Kill Switch的兩種層級

1. 帳戶層級 Kill Switch（個人交易者適用）

是什麼：監控你自己的交易帳戶，異常時切斷你自己的下單能力。

觸發條件：

  • 帳戶權益低於設定閾值（如從100萬跌破90萬）
  • 單日虧損超過設定上限（如5%）
  • 部位數量超過設定上限（如同時持有超過10個部位）
  • 單筆訂單量超過正常範圍（如一次下100張但平均只下5張）
  • 下單頻率異常（如1分鐘內下了50筆但平均1天只下3筆）

怎麼做：
1. 在交易軟體或下單機中設定帳戶層級的Kill Switch參數
2. 觸發後：所有未成交委託全部取消（Cancel All）
3. 不允許新開倉（只允許平倉）
4. 發送緊急通知（Line/Telegram/簡訊）
5. 必須手動確認帳戶狀態正常後才能重新啟動

實例：

  • 情境：程式交易運行中，網路突然異常導致重複下單
  • Kill Switch偵測：1分鐘內下單數量超過上限（設定為5筆/分鐘）
  • 動作：自動取消所有掛單、禁止新開倉、發送Telegram通知
  • 結果：避免像Knight Capital事件一樣45分鐘虧損4.6億美元

2. 市場層級 Kill Switch / 熔斷機制（交易所適用）

是什麼：交易所層級的緊急停止機制，當整個市場出現極端波動時，暫停所有交易。

美股熔斷機制三級制：

  • 第一級：S&P 500較前日收盤下跌7% → 暫停交易15分鐘
  • 第二級：下跌13% → 再次暫停交易15分鐘
  • 第三級：下跌20% → 當日提前收盤，交易完全停止

台股指數穩定措施：

  • 個股瞬間價格穩定措施：試算成交價與前次成交價波動超過3.5% → 進入2分鐘集合競價
  • 期貨動態價格穩定措施：撮合價格超過上下限 → 暫停撮合

對交易者的意義：

  • 熔斷啟動 = 市場出現系統性恐慌
  • 熔斷期間：確認部位和訂單、評估風險是否過大
  • 如果風險過大，熔斷結束後的優先動作是降低風險而非加碼
  • 熔斷結束後可能出現流動性真空（滑價極大），非必要不交易

Kill Switch的爭議

支持方：

  • 可以在毫秒內阻止災難擴大
  • 不依賴人類反應速度（Knight Capital事件中，工程師花了45分鐘才找到問題根源）
  • 自動化保護比手動干預更可靠

反對方：

  • 如果Kill Switch在錯誤時機觸發，可能反而造成市場不穩定
  • 只有一個變數的Kill Switch不夠（不能用單一閾值判斷複雜情境）
  • 跨市場或跨產品策略，只砍一腳可能造成更嚴重的風險不對稱
  • 過度依賴自動化會讓人類監控鬆懈

實務建議：

  • 不要只靠自動Kill Switch，保留手動緊急停止的能力
  • 多層級觸發：先警告→再限制→最後Kill
  • Kill Switch觸發後，不要自動重啟，必須人工確認
  • 設定Kill Switch參數時要預留安全邊際，避免正常波動觸發

二、幽靈單（Phantom Orders）

### 是什麼 幽靈單是指「策略庫存與帳戶實際庫存不一致」的狀況——你的交易策略以為持有2口多單，但實際帳戶可能持有0口或4口。程式交易最怕的惡夢，因為後續所有加減碼決策都基於錯誤的部位資訊。

幽靈單產生原因

1. 網路異常

  • 下單訊號送出但未收到回報 → 策略以為沒成交，實際已成交
  • 網路斷線重連後，策略狀態未同步

2. 券商主機異常

  • 券商系統延遲或掛線，委託狀態不明
  • 交易所撮合成功但回報遲延

3. 程式Bug

  • 策略邏輯錯誤導致重複下單
  • 多Data在同一圖表時觸發異常（MultiCharts常見問題）
  • 非同步處理的競態條件（Race Condition）

4. 人工干預

  • 手動在券商平台下了單或平了倉，但策略不知道
  • 複數策略同時運行在同一帳戶，互相干擾

幽靈單的災難性後果

實例：

  • 情境：你的策略認為目前空手（0口），但實際帳戶持有2口多單
  • 策略發出「買進2口」訊號 → 實際帳戶變成4口多單
  • 策略再發出「買進2口」訊號 → 實際帳戶變成6口多單
  • 當市場反轉時，策略以為只虧2口的錢，實際虧6口的錢
  • 槓桿放大後，可能一次就把帳戶打穿

如何防止幽靈單

1. 庫存確認機制（Position Reconciliation）

怎麼做：
1. 每次下單前，先比對策略庫存與帳戶實際庫存
2. 如果不一致，以帳戶實際庫存為準
3. 修正策略內部狀態後才允許繼續下單
4. 每日開盤前、收盤後強制做一次完整對帳

實例：

  • 策略認為持有台指期2口多單
  • 向券商查詢實際部位：3口多單
  • 自動修正策略內部狀態為3口
  • 發送通知：「部位不一致，已自動修正」
  • 如果差異超過設定閾值（如>1口），暫停策略並發送警報

2. 即時通知系統

怎麼做：
1. 每一筆成交即時推播到Line/Telegram
2. 部位異動即時通知
3. 異常狀況（下單失敗、部位不一致）立即警報
4. 不要等收盤才發現問題

3. 單一策略單一帳戶

怎麼做：
1. 每個策略使用獨立的子帳戶
2. 避免多個策略互相干擾部位計算
3. 如果券商不支援子帳戶，至少使用獨立的交易API連線

4. 心跳偵測（Heartbeat Monitoring）

怎麼做：
1. 策略每30秒發送一次心跳訊號
2. 監控系統如果超過60秒未收到心跳 → 發送警報
3. 心跳遺失可能代表策略當機、網路斷線、電腦休眠
4. 心跳遺失超過5分鐘 → 觸發緊急平倉或Kill Switch

三、交易系統災難復原（Disaster Recovery）

### 是什麼 交易系統災難復原是指當交易系統遭遇重大故障（當機、斷線、數據損壞、硬體故障）時，如何快速恢復交易能力並控制損失。不只是「修好系統」，而是「在最短時間內讓帳戶回到安全狀態」。

災難復原的四個目標

目標全稱定義交易系統適用
RPORecovery Point Objective可接受的最大數據損失時間策略狀態+部位數據的備份頻率
RTORecovery Time Objective系統恢復正常運作的目標時間從斷線到重新上線的最長容忍時間
MTPDMaximum Tolerable Period of Disruption業務可承受的最大中斷時間帳戶無法交易的最長容忍時間
MBCOMinimum Business Continuity Objective最低限度的業務連續目標災難時至少要能平倉、查部位

災難復原三層架構

第一層：預防（Prevention）

怎麼做：
1. 雙網路備援：主線中華電信+備線台灣大哥大，自動切換
2. UPS不斷電系統：交易主機至少30分鐘備用電力
3. 雙機熱備：主機+備機同時運行，主機掛了備機秒接
4. 定期備份：每日備份策略狀態、部位數據、交易日誌
5. 監控告警：CPU/記憶體/網路/硬碟使用率超過80%即告警

實例：

  • 交易主機CPU風扇壞掉導致過熱當機
  • 監控系統偵統偵測到CPU溫度異常（超過90度）
  • 自動觸發：將策略遷移到備機、通知交易者
  • 備機接手運行，交易不中斷

第二層：應急（Emergency Response）

怎麼做：
1. 緊急平倉SOP：

  • 優先平掉所有裸露部位（無避險的單邊倉位）

2. 人工接管程序：

  • 程式無法交易時，立刻切換到手機APP或電話下單

3. 通報流程：

  • 發現異常 → 1分鐘內評估嚴重程度

實例：

  • 情境：交易程式當機，帳戶持有3口台指期多單+2口選擇權賣權
  • 步驟1：用手機APP登入券商查看實際部位（確認有無幽靈單）
  • 步驟2：如果部位正確，評估是否需要立即平倉
  • 步驟3：如果市場正在劇烈波動，優先平期貨部位（風險最大）
  • 步驟4：修正程式問題後，重新上線前先做完整對帳

第三層：恢復（Recovery）

怎麼做：
1. 數據恢復：從最近的備份點恢復策略狀態和部位數據
2. 部位同步：強制將策略內部狀態同步為帳戶實際部位
3. 漸進式重啟：

  • 先用觀察模式運行（不下單，只記錄策略會做什麼）

4. 事故覆盤：

  • 記錄故障原因、持續時間、影響金額

常見災難情境與應對

情境1：網路斷線

怎麼做：
1. 立刻用手機4G/5G熱點連線
2. 登入券商APP確認部位
3. 如果有未設停損的部位，手動設定停損
4. 網路恢復後重新啟動交易程式
5. 預防：設定雙網路自動切換

情境2：程式無限下單（失控）

怎麼做：
1. 立刻手動啟動Kill Switch（取消所有掛單+禁止新單）
2. 查看實際部位，手動平掉所有不預期的部位
3. 檢查程式碼找出Bug
4. 修正後在模擬環境測試至少1小時
5. 預防：設定每分鐘最大下單數量限制

情境3：交易所系統異常

怎麼做：
1. 不要在異常期間追價下單（流動性可能為零）
2. 等待交易所恢復正常後再評估
3. 如果持有部位，評估是否需要在恢復後立刻平倉
4. 預防：追蹤交易所系統狀態公告，提前避開高風險時段

情境4：硬體故障（電腦當機）

怎麼做：
1. 切換到備機或筆電
2. 確認部位後決定是否需要平倉
3. 修復主機後重新部署策略
4. 預防：交易主機定期維護、雙機熱備

四、帳戶保護五道防線

第一道：事前限制（Pre-Trade Controls）

怎麼做：
1. 最大單筆委託量限制（如單次最多5口期貨）
2. 最大持倉限制（如台指期最多10口）
3. 每日最大下單次數限制（如當沖最多20筆）
4. 禁止交易的標的清單（流動性不足的冷門股）
5. 價格偏離限制（委託價偏離現價超過2%拒絕下單）

第二道：即時監控（Real-Time Monitoring）

怎麼做：
1. 每秒更新帳戶權益和部位
2. 監控關鍵指標：未實現損益、保證金使用率、風險值
3. 異常行為偵測：下單頻率、成交速率、部位變化速率
4. 即時Dashboard：一目了然的帳戶健康狀態

第三道：警報通知（Alert & Notification）

怎麼做：
1. 三級警報制：

  • 黃色警報：接近限制（如虧損達3%時通知）

2. 多管道通知：Telegram + Email + 簡訊
3. 警報不確認不自動解除

第四道：自動熔斷（Auto Circuit Breaker）

怎麼做：
1. 帳戶級熔斷：

  • 單日虧損5% → 禁止新開倉

2. 策略級熔斷：

  • 策略連續3筆虧損 → 暫停策略10分鐘

3. 系統級熔斷：

  • 下單速率超過5倍正常值 → 凍結下單1分鐘

第五道：災難復原（Disaster Recovery）

前面第三大節已詳述，此處強調：

  • 災難復原是最後一道防線，前面四道都失效時才需要
  • 平時就要演練：每季做一次災難模擬，確保流程可行
  • 目標：從災難發生到帳戶安全，不超過15分鐘

五、程式交易風控檢查清單

每日開盤前（5分鐘）

  • [ ] 確認交易主機正常運行
  • [ ] 確認網路連線穩定
  • [ ] 確認券商API連線正常
  • [ ] 核對策略庫存與帳戶實際部位一致
  • [ ] 確認Kill Switch參數正確
  • [ ] 確認停損單已掛好

交易中（持續）

  • [ ] 監控Dashboard正常顯示
  • [ ] 即時通知系統運作正常
  • [ ] 帳戶權益在合理範圍
  • [ ] 無幽靈單警報
  • [ ] 下單頻率在正常範圍

每日收盤後（10分鐘）

  • [ ] 完整對帳：策略庫存 vs 實際庫存
  • [ ] 確認所有部位都有停損保護
  • [ ] 備份策略狀態和交易日誌
  • [ ] 檢查系統日誌有無異常
  • [ ] 更新監控Dashboard

每月（1小時）

  • [ ] 測試Kill Switch是否能正常觸發
  • [ ] 測試備機是否能順利接手
  • [ ] 測試網路切換是否正常
  • [ ] 檢查所有風控參數是否需要調整
  • [ ] 模擬一次災難復原流程

心得

這個主題是所有風險管理中最「不性感」但最可能救你一命的。之前學的停損、部位控制、心理管理，都是「正常交易時」的風控。Kill Switch和災難復原處理的是「不正常時」的風控——程式失控、網路斷線、幽靈單暴增，這些事情發生的機率很低，但一旦發生，沒有準備的人可能一次就被消滅。

Knight Capital的故事是最佳教材：2012年8月1日，因為程式部署錯誤，一台沒有被正確更新的伺服器開始瘋狂下單，45分鐘內執行了400萬筆交易，虧損4.6億美元，公司從此消失。如果當時有Kill Switch，在1分鐘內就能止損。

另一個深刻體悟是：Kill Switch不是萬靈丹。NYIF的文章指出一個關鍵問題——如果你有跨市場價差策略（如買台指期+賣新加坡摩台），Kill Switch只砍一腳會造成更嚴重的風險不對稱。所以Kill Switch的設計必須是「策略感知」的，不是簡單的一刀切。

最後，幽靈單這個問題在散戶程式交易中太常見了。很多人用MultiCharts或TradeStation，網路一不穩就出現部位不同步，然後策略基於錯誤資訊繼續加碼，越偏越遠。庫存確認機制不是「nice to have」，是「must have」。每一筆下單前，都必須確認策略庫存和實際庫存一致，這是最基本的防線。

交易隱藏成本×滑價×流動性風險×市場衝擊

一、交易成本的完整版圖：看得見的 vs 看不見的

### 是什麼 多數人只看到手續費和交易稅，但真正的交易成本遠不止這些。完整的交易成本公式是： 
真實交易成本 = 手續費 + 交易稅 + 滑價 + 市場衝擊 + 機會成本

台股交易成本明細

看得見的成本：

  • 買進手續費：0.1425%（網路下單通常6折 = 0.0855%）
  • 賣出手續費：0.1425%（6折 = 0.0855%）
  • 證交稅：0.3%（一般交易）、0.15%（當沖）
  • 合計可見成本：一般交易約 0.47%（6折）、當沖約 0.32%

看不見的成本：

  • 滑價（Slippage）：預期成交價與實際成交價的差異
  • 市場衝擊（Market Impact）：大單造成價格偏離
  • 流動性成本：低流動性標的買賣價差大
  • 機會成本：限價單沒成交的損失

實例：

  • 情境：當沖台指期，一天進出5次
  • 可見成本：手續費+交易稅，每次約1點，5次共10點（買賣各5次）
  • 滑價成本：順勢追價每次約2點，5次共10點
  • 停損成本：假設3次停損各8點 = 24點
  • 總成本：10 + 10 + 24 = 44點，只剩2次獲利機會，每次至少要賺22點才能損益兩平
  • 而台指期日均波動超過80點的機率只有33%，要一天內出現2次39點以上的波段更是難上加難

二、滑價（Slippage）：最被低估的隱形成本

### 是什麼 滑價是指交易時，實際成交價格與預期價格之間的差異。它不是券商收的費用，而是市場給你的「意外收費」。

正滑價 vs 負滑價

類型說明發生情境
負滑價實際成交價比預期更差（買貴/賣便宜）順勢追價、市價單、高波動時
正滑價實際成交價比預期更好（買便宜/賣貴）逆勢掛單、限價單、低波動時

滑價何時最嚴重？

1. 順勢突破進場

  • 突破時所有人都在追，你想買在9500結果買到9502
  • 順勢買進和順勢賣出各發生一次滑價
  • 突破策略的滑價是逆勢策略的數倍

2. 重大消息公布

  • FOMC利率決議、非農就業數據等
  • 點差瞬間擴大10倍（0.5 pips → 5 pips）
  • 滑價可能高達12-20 pips
  • 總入場成本：點差5 pips + 滑價12 pips = 17 pips

3. 開盤和收盤

  • 開盤前5分鐘和收盤前1-2分鐘
  • 成交量集中在短時間，價格跳動劇烈
  • 系統化交易訊號在這些時段觸發，滑價最大

4. 低流動性標的

  • 日均成交量小於1000張的股票
  • 買賣價差大，大單會直接吃掉好幾檔報價
  • 想賣的時候找不到買家

滑價對不同策略的影響

策略類型滑價影響原因
當沖⚠️ 極大進出頻繁，每次滑價累積驚人
短線波段中等進出次數少，但順勢追價有滑價
中長線小持倉天數長，滑價佔比極小
逆勢策略小甚至正滑價掛限價單等回檔，可能成交在更好價格
順勢策略⚠️ 大追價進場，每次都付滑價

怎麼做：減少滑價的5個方法

1. 限價單優先

  • 限價單：指定價格成交，無滑價，但可能不成交
  • 市價單：確保成交，但有滑價
  • 實務解法：先用限價單，限價單沒成交再轉市價單
  • 短線交易者必須用限價單，否則滑價可能超過手續費

2. 避開重大消息時段

  • FOMC、非農、財報公布前後30分鐘不下單
  • 開盤前5分鐘和收盤前2分鐘避免市價單
  • 專業交易者從不在數據公布的第一秒入場，等流動性恢復再進場

3. 選擇高流動性標的

  • 股票：日均成交量至少2000張以上
  • 期貨：選近月合約，遠月流動性差
  • 持有部位不超過日均成交量的1%

4. 分批下單

  • 大單拆成小單，減少市場衝擊
  • 每筆不超過當前掛單量的10-20%
  • 算法交易常用VWAP（成交量加權均價）策略分批執行

5. 回測必須加入滑價

  • 回測時每筆交易額外扣0.05%-0.1%的滑價成本
  • 順勢策略滑價設大一些（0.1-0.3%）
  • 逆勢策略滑價設小一些（0.02-0.05%）
  • 如果回測加了滑價還能賺錢，策略才有實盤價值

三、流動性風險（Liquidity Risk）：想賣賣不掉的風險

### 是什麼 流動性風險是指想買買不到、想賣賣不掉的風險。不是「價格會跌」的風險，而是「價格可能根本不是你想的那個價」的風險。

流動性的三個維度

1. 寬度（Width）：買賣價差

  • 價差越小，流動性越好
  • 台積電：買賣價差可能只有0.1%（1毛）
  • 冷門股：買賣價差可能高達2-5%

2. 深度（Depth）：訂單簿厚度

  • 五檔報價的掛單量
  • 掛單越多，大單越不容易造成價格偏離
  • 台積電：每檔可能有數百張掛單
  • 冷門股：每檔可能只有個位數掛單

3. 彈性（Resiliency）：價格恢復速度

  • 大單衝擊後，價格多快恢復正常
  • 高流動性標的：幾秒到幾分鐘
  • 低流動性標的：可能數小時甚至數天

流動性風險的四個陷阱

陷阱1：想賣時賣不掉

  • 冷門股日均成交100張，你持有50張
  • 賣出時佔日成交量50%，必須大幅降價才能成交
  • 實際賣出價可能比看到的「成交價」低3-5%

陷阱2：流動性會消失

  • 正常時期流動性好的標的，危機時可能完全沒流動性
  • 2008年金融海嘯：很多「投資級」公司債完全找不到買家
  • 2020年3月：連美國公債都出現流動性枯竭

陷阱3：ETF的流動性陷阱

  • ETF本身的成交量不等於成分股的流動性
  • 小型ETF在恐慌時可能出現大幅折價（NAV與市價偏離）
  • 2020年3月：高收益債ETF折價超過5%

陷阱4：對手方風險

  • 流動性差的資產，交易對手少
  • 如果交易對手違約，你可能求償無門
  • 未上市股票、私募股權的流動性風險最大

怎麼做：流動性風險管理

1. 部位控制

  • 持有部位不超過日均成交量的1%
  • 例：某股票日均成交5000張，持有不超過50張
  • 極端保守：不超過0.5%

2. 分散流動性

  • 不要把所有資金投入低流動性標的
  • 建議：至少70%資金在高流動性資產（大型股、ETF、公債）
  • 最多30%可以放在中低流動性標的

3. 現金緩衝

  • 永遠保持20-30%現金部位
  • 危機時現金是唯一的流動性來源
  • 別忘了：巴菲特在2008年手握大量現金，才能在別人恐慌時買進

4. 看成分股流動性而非ETF本身

  • ETF的成交量只是表面
  • 真正要關注的是ETF持有成分股的流動性
  • 成分股流動性差 → ETF在恐慌時會大幅折價

5. 避免投資冷門標的

  • 新手應該避開：冷門股、未上市股票、小眾債券、收藏品
  • 除非你有資訊優勢和專業知識
  • 市場先生的建議：「新手不應該碰流動性差的商品，因為你沒有資訊優勢，超額報酬輪不到你，只有被宰的份」

四、市場衝擊（Market Impact）：大單如何推動價格

### 是什麼 市場衝擊是指你的訂單大到足以影響市場價格。當你買入大量股票時，你的買單本身就會推高價格；當你賣出大量股票時，你的賣單本身就會壓低價格。

市場衝擊的計算邏輯

市場衝擊 ≈ (你的訂單量 / 市場平均成交量) × 價格波動度

實例：

  • 情境：你想買入某股票500張，該股日均成交量2000張
  • 你的訂單佔日均成交量的25%
  • 這筆大單可能將價格推高0.5-1%
  • 如果你本來預期在100元買入，實際可能買到100.5-101元
  • 額外成本：0.5-1%的市場衝擊

不同資金規模的市場衝擊

資金規模適合的標的市場衝擊
100萬以下大型股、ETF幾乎為零
100-500萬大型股為主微小
500-2000萬需注意流動性0.1-0.3%
2000萬以上必須考慮衝擊0.3-1%或更高
機構法人分批執行使用VWAP/TWAP算法

怎麼做：降低市場衝擊的4個方法

1. 分批執行

  • 大單拆成多筆小單
  • 例：要買500張，拆成10筆50張，分30分鐘執行
  • 或使用金字塔加碼法，分3-4次逐步建立部位

2. 避開集中時段

  • 開盤和收盤成交量最大但波動也最大
  • 上午10:00-11:30和下午1:30-2:30是相對穩定的時段
  • 大單在這些時段執行，市場衝擊較小

3. 使用演算法交易

  • VWAP（Volume Weighted Average Price）：按成交量比例分批下單
  • TWAP（Time Weighted Average Price）：按時間均勻分批下單
  • 冰山單（Iceberg Order）：只顯示部分訂單量，其餘隱藏

4. 選擇流動性充足的標的

  • 大型股（台積電、聯發科等）日均成交量數萬張
  • 500張的單只佔日成交量1-2%，市場衝擊幾乎為零
  • 如果必須投資中小型股，控制部位在日成交量的1%以內

當沖的真實成本有多恐怖？

實例：

  • 情境：100萬本金，當沖一天進出5次
  • 每次可見成本：100萬 × 0.321% = 3,210元
  • 每次滑價+衝擊：100萬 × 0.2% = 2,000元
  • 每次總成本：5,210元
  • 5次總成本：26,050元
  • 佔本金2.6%，一天就燒掉2.6%！

更恐怖的計算：

  • 一月交易22天，每天5次
  • 月成本：26,050 × 22 = 573,100元
  • 佔本金57.3%！
  • 就算當沖勝率50%，光交易成本就吃掉超過一半本金
  • 這就是為什麼統計上散戶當沖從未轉正——成本結構本身就是災難

順勢策略當沖的滑價災難

市場先生的計算案例：

  • 順勢策略每天交易5次
  • 假設每次交易成本+滑價只有1點（極度保守）
  • 買賣各5次 = 10點成本
  • 勝率40%，5次中虧3次，停損各8點 = 24點虧損
  • 總成本：10 + 24 = 34點
  • 剩下2次獲利機會，每次至少要賺17點才能損益兩平
  • 順勢策略頭尾各10點賺不到，還有2點滑價
  • 一段波動需要：10+10+1+1+17 = 39點
  • 一天需要發生2次39點波動才能損益兩平
  • 台指期日均波動超過80點的機率只有33%

結論：順勢策略+當沖+高頻交易 = 幾乎必賠的組合

  • 逆勢策略：每筆扣0.02-0.05%

心得

交易成本這個主題看似枯燥，但可能是散戶最需要理解卻最常忽略的。

滑價這個概念特別有衝擊力——順勢策略的滑價成本是逆勢策略的數倍，這直接解釋了為什麼當沖散戶長期幾乎不可能賺錢。市場先生的那個計算案例太清楚了：一天進出5次，光成本就要34點，剩下2次獲利機會每次要賺17點以上才能打平，而台指期一天要出現2次39點以上的波段，機率只有33%。這不是策略好壞的問題，是成本結構本身就不允許你贏。

另一個重要體悟是「回測必須加入滑價」。之前學回測驗證的時候，知道要加交易成本，但滑價這塊常常被忽略。如果回測年化20%，加了0.1%的滑價成本可能就變成年化15%，加了0.3%可能只剩10%。這剛好呼應之前學過的「回測績效衰退30-50%」的經驗法則——滑價和市場衝擊就是衰退的重要來源之一。

流動性風險也是投資人容易忽略的。正常時期看起來流動性充足的標的，危機時可能完全找不到買家。市場先生那句話很好：「流動性風險不是價格會跌的風險，而是價格可能根本不是你想的那個價的風險。」2008年連投資級公司債都找不到買家，這才是流動性風險最可怕的地方——它會在你最需要流動性的時候消失。

最後，1%法則非常實用：持有部位不超過日均成交量的1%。這個簡單規則同時管理了滑價、市場衝擊和流動性風險三個問題。以後選股時，除了看基本面和技術面，也要把日均成交量當成一個硬性過濾條件。

一、保證金交易與斷頭機制

### 是什麼 保證金交易（融資、期貨、選擇權）使用槓桿放大部位，當虧損使保證金低於規定門檻時，券商會發出追繳通知，要求補足保證金；若未在期限內補足，就會被「斷頭」（強制平倉），所有部位被強制賣出。

台灣期貨斷頭規則

兩種斷頭觸發機制：

1. 盤中斷頭（風險指標 < 25%）

  • 風險指標 = 權益數 ÷ 原始保證金

2. 盤後追繳（未在期限內補繳）

  • 收盤後權益數低於維持保證金 → 發出追繳通知

怎麼做（避免斷頭）：
1. 保證金放超過原始保證金2-3倍，永遠不要只放最低門檻
2. 設定停損：在風險指標到達50%之前就自行減碼
3. 連續虧損時主動減少部位，不要等到被追繳才處理
4. 絕對不要在收到追繳通知後才想辦法——那時候已經太遲了

實例：

  • 情境：持有1口台指期多單，原始保證金20萬，維持保證金15萬
  • 帳戶權益數只有10萬 → 風險指標 = 10/20 = 50%
  • 權益數降到5萬 → 風險指標 = 5/20 = 25% → 立即斷頭
  • 正確做法：權益數降到15萬（風險指標75%）時就自行減碼或停損

台灣股票融資斷頭規則

融資維持率三條線：

融資維持率狀態處理方式
> 166%安全正常交易
130%-166%警戒區暫時解除追繳，但持續監控
< 130%斷頭警戒T日通知，T+2日前補足至166%，否則T+3日強制賣出

怎麼做（融資安全操作）：
1. 融資維持率永遠維持在200%以上，留足安全邊際
2. 融資比例不超過總資金的30%
3. 設定「自我斷頭線」：維持率低於150%就自行減碼，不等到130%
4. 永遠不要用融資做長期投資——融資是短線工具，時間成本會吃掉你

實例：

  • 情境：用融資買進100萬的股票，自備款40萬，融資60萬
  • 融資維持率 = 股票市值 / 融資金額
  • 初始維持率 = 100/60 = 166.7%（剛好在安全線上）
  • 股票跌10% → 市值90萬 → 維持率 = 90/60 = 150%（警戒區）
  • 股票跌20% → 市值80萬 → 維持率 = 80/60 = 133.3%（逼近斷頭）
  • 股票跌22.5% → 市值77.5萬 → 維持率 = 77.5/60 = 129.2%（斷頭！）

超額損失（Over Loss）

是什麼：當市場劇烈波動（跳空、連續跌停），強制平倉的成交價可能遠差於斷頭時的理論價，造成損失超過保證金帳戶餘額。

怎麼做：
1. 了解超額損失的真實風險：斷頭不是最壞情況，超額損失才是
2. 2018年「0206期貨大屠殺」：散戶損失2.3億，就是超額損失的案例
3. 2021年Archegos爆倉事件：5倍槓桿，一天虧掉200億美元
4. 避免超額損失的唯一方法：不要讓保證金水位低到需要被斷頭

適用時機：任何使用保證金交易的人（融資、期貨、選擇權賣方）都必須理解斷頭機制

二、槓桿風險管理

槓桿的雙面刃

是什麼：槓桿放大報酬也放大虧損。10倍槓桿意味著10%的市場波動就是100%的帳戶波動。

不同工具的槓桿倍數：

工具槓桿倍數斷頭風險適合程度
現股1倍無新手首選
融資2-2.5倍中等有經驗後再考慮
期貨（大台）8-10倍高至少2年交易經驗
期貨（小台）8-10倍高至少1年交易經驗
選擇權買方變動（可達20倍以上）低（最多賠權利金）了解希臘字母後再碰
選擇權賣方變動極高需要大量資金和風控經驗
當沖無槓桿（但可融券）低適合短線交易者

槓桿安全公式

怎麼做：
1. 實際槓桿 = 總部位 / 自有資金
2. 安全槓桿 = 1 / 最大可接受回撤
3. 如果安全槓桿 < 實際槓桿，就過度槓桿了

實例：

  • 最大可接受回撤：20%
  • 安全槓桿 = 1/0.2 = 5倍
  • 如果你的總部位是自有資金的8倍 → 過度槓桿
  • 調整：減少部位或增加自有資金，使槓桿降到5倍以下

槓桿使用的五條鐵律

1. 新手不用槓桿：至少1年穩定獲利後才考慮
2. 永遠不要用最大槓桿：只用安全槓桿的50-70%
3. 連虧時降槓桿：連虧3筆 → 槓桿減半
4. 槓桿與波動度反相關：高波動標的用低槓桿，低波動標的可用稍高槓桿
5. 斷頭線遠離安全線：自我斷頭線設在券商斷頭線的2倍以上距離

三、流動性風險

### 是什麼 流動性風險是指「想賣的時候賣不掉，或者必須以顯著低於期望的價格才能賣掉」的風險。這是所有交易者都會面對但最容易忽略的風險。

流動性風險的三種型態

型態1：市場流動性風險

  • 買賣價差過大，成交困難
  • 小型股、冷門股、未上市股票最常見
  • 特徵：日均成交量低、買賣五檔價差大

型態2：融資流動性風險

  • 需要資金時無法及時變現
  • 長期投資鎖定期（定存、保險、不動產）
  • 特徵：提前解約有罰金、折價嚴重

型態3：強制賣出流動性風險

  • 被追繳保證金或斷頭時，必須在不利價格強制賣出
  • 這是最致命的流動性風險——在最差的時機被迫交易
  • 特徵：恐慌性拋售、跌停鎖死

流動性風險的隱蔽性

怎麼做：
1. 流動性在正常時期不是問題，但在危機時變成致命問題
2. 相關性飆升時，所有資產同時下跌，你需要賣東西籌現金
3. 此時只有高流動性資產能賣出，但它們也在下跌
4. 低流動性資產根本賣不掉，或者必須折價30-50%才能成交

實例：

  • 情境A（正常時期）：某小型股日均成交量500張，你想買賣10張，幾秒成交
  • 情境B（恐慌時期）：同檔股票，一天成交量只有50張，你想賣100張
  • 結果：必須在跌停價掛單才可能成交，或者連續3天跌停賣不掉
  • 這就是流動性風險——正常時看不出來，危機時才發現問題

怎麼做（降低流動性風險）：
1. 持有部位不超過日均成交量的1%（安全線）
2. 避免在開盤前5分鐘和收盤前5分鐘交易（流動性最不穩定）
3. 使用限價單而非市價單（避免滑價）
4. 大部位分批進出（不要一次買賣超過日均量的5%）
5. 持有足夠現金緩衝（至少20-30%），避免被迫在不利時機賣出

四、滑價（Slippage）與交易成本

### 是什麼 滑價是指下單價格與實際成交價格之間的差異。這是交易成本中最容易被忽略、也最難控制的部分。

滑價的四種來源

1. 市場衝擊（Market Impact）

  • 大單進出會推動價格，買進時把價格推高，賣出時把價格壓低
  • 佔部位越大、成交量越小的標的，衝擊成本越高

2. 價差成本（Bid-Ask Spread）

  • 買價和賣價之間的差距就是你的隱形成本
  • 小型股、冷門權證的價差可能高達1-3%

3. 延遲滑價（Latency Slippage）

  • 從你看到報價到下單成交之間的時間差
  • 高頻交易者和一般散戶最大的不公平來源
  • 重大消息公布時，延遲可能高達數秒

4. 跳空滑價（Gap Slippage）

  • 停損單設定在特定價位，但開盤跳空導致成交在更差的價格
  • 這是停損最大的隱藏風險

實例：

  • 情境A：停損設在100元，但隔天開盤跳空到95元
  • 結果：停損單在95元成交，滑價5%（5元）
  • 教訓：停損只是盡力而為，不是保證成交在設定價

滑價與交易成本的完整計算

怎麼做：
1. 交易成本 = 手續費 + 交易稅 + 滑價 + 衝擊成本
2. 台股買賣一次的固定成本約0.3-0.5%（手續費0.1425%×2 + 證交稅0.3%）
3. 滑價約0.1-0.5%（視流動性而定）
4. 總交易成本約0.4-1.0%

對策略的影響：

  • 假設策略年周轉率10倍（一年買賣10次）
  • 每次交易成本0.5%
  • 年化交易成本 = 10 × 0.5% = 5%
  • 如果策略年化報酬只有8%，交易成本就吃掉62.5%
  • 結論：高周轉率策略對交易成本極度敏感

降低滑價的七個方法

1. 避開冷門股：只交易日均成交量超過2000張的標的
2. 使用限價單：永遠不要用市價單，除非你是當沖需要即時成交
3. 分批進出：大部位分成3-5筆，每隔5-10分鐘下一筆
4. 避開消息面：財報公布前後、重大政策公布時，滑價會暴增
5. 選擇流動性好的時段：09:00-09:30和13:00-13:30是流動性最好的時段
6. 設定合理價格：限價單掛在買賣價差的中間，不要掛太離譜的價格
7. 停損設在支撐下方：不要設在整數關卡（如100元），因為很多人停損設在那裡，容易造成滑價

五、保證金追繳與斷頭的完整防禦體系

五道防線

第一道：槓桿控制（事前）

  • 永遠不要用最大槓桿
  • 安全槓桿 = 1 / 最大可接受回撤
  • 實際槓桿只用安全槓桿的50-70%

第二道：自我斷頭線（預警）

  • 設定自己的斷頭線，遠高於券商的斷頭線
  • 融資維持率200%以下就減碼，不等到166%
  • 期貨風險指標低於80%就減碼，不等到25%

第三道：停損單（即時）

  • 進場同時設好停損，不依賴手動停損
  • 停損距離根據ATR設定，不是隨意百分比
  • 停損是保護本金的保險，不是認輸

第四道：部位控制（系統）

  • 單筆風險不超過總資金的1-2%
  • 同時持有部位不超過5個
  • 相關性高的標的不超過3個

第五道：現金緩衝（最後防線）

  • 永遠保留20-30%現金
  • 現金不用付保證金、不會被斷頭、不用擔心流動性
  • 在危機時，現金是唯一能加碼的子彈

斷頭後的處理SOP

1. 停止交易：斷頭後當天不再開新倉
2. 冷靜分析：為什麼被斷頭？是槓桿過高、停損太寬、還是沒設停損？
3. 檢討策略：策略本身是否有問題？還是只是部位管理不當？
4. 調整參數：降低槓桿、縮小部位、增加保證金
5. 小部位重新開始：用1/4部位重新測試策略，確認能穩定獲利後才恢復正常部位

心得

保證金和斷頭這個主題，是風險管理中最「痛」的部分——因為它不是理論上的可能，而是真實會發生的災難。2018年0206大屠殺、Archegos爆倉，這些都不是教科書上的假設情境，而是血淋淋的教訓。

最讓我印象深刻的是「自我斷頭線」的概念。券商的斷頭線（融資維持率130%、期貨風險指標25%）是給你「最後的警告」，但你永遠不應該讓自己走到那一步。正確的做法是在券商斷頭線的兩倍以上就自行減碼。這跟之前學的「回撤降速協定」完全一致——綠燈（安全）、黃燈（警戒）、紅燈（減碼），而不是等到斷頭才處理。

流動性風險和滑價成本這兩個隱形殺手，以前我關注得不夠。特別是滑價——停損設在100元，開盤跳空到95元，你的停損就是一張廢紙。這意味著回測中的停損假設永遠比實際樂觀，回測績效衰退30-50%的其中一個原因就是滑價。以後在回測時，一定要把0.3-0.5%的滑價成本計入。

最後，保證金交易的五道防線是之前所有風險管理知識的集大成：槓桿控制是凱利公式的應用、自我斷頭線是MDD控制的延伸、停損單是停損五法的具體執行、部位控制是Van Tharp部位管理的實踐、現金緩衝是槓鈴策略安全端的核心。風險管理不是一個概念，是一套系統——每一道防線都是獨立的保險，五道防線同時失靈的機率指數級下降。
