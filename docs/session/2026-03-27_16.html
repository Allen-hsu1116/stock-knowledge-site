核心知識

壓力測試基本概念

是什麼：壓力測試是一種風險管理工具，用於評估投資組合或交易策略在「極端但可能發生」的市場條件下的表現。與傳統風險指標（如 VaR）依賴歷史數據和常態市場變化不同，壓力測試專門評估從未發生過但理論上可能發生的不利情境。

為什麼重要：

  • 發現正常條件下不明顯的隱藏風險
  • 模擬黑天鵝事件、流動性危機、市場崩盤
  • 驗證風險指標、部位規模、分散策略的韌性
  • 法規要求（銀行需符合 Basel III、DFAST）

怎麼做：
1. 定義測試範圍和目標
2. 設計壓力情境
3. 評估對投資組合的影響
4. 解釋結果並制定改善策略
5. 定期更新

壓力測試四大類型

1. 敏感性分析（Sensitivity Analysis）

是什麼：評估投資組合對單一風險因子小幅變動的反應，例如利率、匯率、股價。

怎麼做：
1. 選擇要測試的風險因子（如利率+1%、股價-10%）
2. 假設其他條件不變
3. 計算投資組合價值變化
4. 找出對績效影響最大的因子

實例：

  • 情境：持有台積電 100 張，成本 600 元
  • 測試：股價下跌 5%、10%、15%、20%
  • 結果：-5% → 損失 30 萬；-10% → 損失 60 萬；-20% → 損失 120 萬
  • 判斷：能否承受這些損失？是否需要減碼或對沖？

適用時機：找出部位最敏感的風險因子、微調風險參數。

2. 情境分析（Scenario Analysis）

是什麼：評估多個風險因子同時變動對投資組合的綜合影響，比敏感性分析更貼近現實。

怎麼做：
1. 選擇情境類型（歷史事件或假設情境）
2. 設定各風險因子的變動幅度
3. 計算投資組合整體影響
4. 分析哪些情境最危險

情境範例：

情境類型利率股價匯率原油
2008 金融海嘯+2%-40%美元+15%-50%
2020 疫情崩盤-1%-35%美元+5%-60%
通膨高點+3%-20%美元-5%+30%
地緣政治危機-0.5%-25%美元+10%+40%

實例：

  • 情境：投資組合 60% 股票、30% 債券、10% 現金
  • 測試：2008 金融海嘯重演
  • 結果：股票跌 40% → -24%、債券跌 5% → -1.5%、合計 -25.5%
  • 判斷：與 VaR 估計的 -15% 相比，壓力情境可能再多虧 10%

適用時機：全面評估投資組合在危機中的韌性。

3. 反向壓力測試（Reverse Stress Testing）

是什麼：從「災難結果」反推「可能原因」。定義一個嚴重的不利結果（如虧損 50%），然後找出什麼情境會導致這個結果。

怎麼做：
1. 定義不可接受的結果（如：帳戶歸零、虧損 50%、被迫斷頭）
2. 反推哪些情境會導致此結果
3. 評估這些情境發生的機率
4. 制定預防措施

實例：

  • 目標結果：帳戶虧損 50%
  • 反推情境：
  • 預防措施：限制槓桿倍數、分散持股、設定熔斷機制

適用時機：找出可能摧毀帳戶的情境，是風險管理的「底線思維」。

4. 歷史壓力測試（Historical Stress Testing）

是什麼：使用過去真實發生的市場事件來測試當前投資組合的表現。

怎麼做：
1. 選擇歷史事件（2008 金融海嘯、2010 閃電崩盤、2020 疫情崩盤）
2. 取出該時期的市場數據（股價、利率、波動率、流動性）
3. 將當前投資組合代入該環境
4. 計算損益

實例：

  • 測試：2010 年 5 月 6 日閃電崩盤（道瓊 5 分鐘跌 1000 點）
  • 情境：持有台指期 2 口，多單
  • 結果：5 分鐘內可能虧損超過保證金 50%
  • 判斷：是否需要更大的保證金緩衝？是否要設定熔斷停損？

適用時機：評估真實發生過的危機對當前部位的衝擊。

蒙地卡羅模擬（Monte Carlo Simulation）

是什麼：透過隨機模擬產生數千甚至數萬種可能的未來情境，計算投資組合的報酬分布。

怎麼做：
1. 設定投資組合的基本參數（平均報酬、波動率、相關性）
2. 隨機產生大量價格路徑（至少 1000 次）
3. 計算每一條路徑的最終損益
4. 統計報酬分布，計算 VaR、預期虧損等

實例：

  • 情境：策略回測顯示最大回撤 10%
  • 蒙地卡羅模擬結果：
  • 判斷：實際交易可能面臨比回測更差的表現，需準備更多資金緩衝

Build Alpha 研究發現：

  • 回測回撤 $1,663.90
  • 蒙地卡羅模擬可能最大回撤：$5,195.17
  • 結論：回測可能嚴重低估風險

適用時機：偵測「幸運回測」、評估風險分布全貌、避免過度自信。

壓力測試執行六步驟

Step 1：定義範圍與目標

是什麼：確定要測試什麼風險、時間範圍、具體目標。

怎麼做：
1. 列出主要風險因子（利率、匯率、股價、商品價格、流動性）
2. 設定時間範圍（1 天、1 週、1 個月、1 季）
3. 確定目標（確保流動性？維持資本適足？避免斷頭？）

實例：

  • 目標：確保在股市崩盤時不會被斷頭
  • 範圍：融資維持率低於 130% 的情境
  • 時間：1 週內可能發生的最大跌幅

Step 2：設計壓力情境

怎麼做：

  • 使用歷史事件（金融海嘯、疫情崩盤）
  • 設計假設情境（利率+3%、股市-30%、油價+50%）
  • 參考法規情境（銀行壓力測標準）

情境設計範例：

情境利率股市匯率波動率
輕度+1%-15%±3%+10%
中度+2%-25%±5%+20%
重度+3%-40%±10%+50%

Step 3：評估影響

怎麼做：
1. 計算每個情境下的資產價值變化
2. 考慮相關性變化（危機時相關性會上升）
3. 評估流動性影響（賣不掉的風險）
4. 計算保證金/融資維持率變化

實例：

  • 情境：股市-30%、相關性從 0.5 升至 0.9
  • 投資組合：5 檔股票，原來相關性低
  • 結果：分散效果消失，整體跌 28%（比預期更慘）

Step 4：解釋結果

怎麼做：
1. 找出最危險的情境
2. 識別脆弱環節（哪個部位最危險？）
3. 評估是否符合風險承受度

關鍵指標：

  • 最大虧損金額
  • 融資/保證金維持率變化
  • 流動性危機可能性
  • 損失恢復時間

Step 5：制定改善策略

改善方向：

  • 調整部位：減少高風險曝露
  • 增加對沖：買入保護性選擇權、期貨對沖
  • 提高流動性：保留更多現金
  • 設定熔斷：自動停損機制

實例：

  • 問題：重度情境下融資維持率跌破 130%
  • 解決：減碼 20%、或提高現金部位、或設定提早停損

Step 6：定期檢視與更新

怎麼做：

  • 每季至少執行一次
  • 重大市場事件後立即更新
  • 新增新風險因子（如加密貨幣、新商品）

個人交易者壓力測試實務

帳戶壓力測試檢查清單

基本檢查：

  • [ ] 如果大盤一週跌 20%，我會虧多少？
  • [ ] 如果最低保證金要求提高 50%，我會被斷頭嗎？
  • [ ] 如果最大連續虧損發生，資金還能撐多久？
  • [ ] 如果流動性枯竭（跌停鎖死），我能出場嗎？

槓桿檢查：

  • [ ] 如果價格反向 10%，部位還安全嗎？
  • [ ] 如果保證金要求翻倍，我還能維持部位嗎？
  • [ ] 如果 overnight gap 跳空 15%，會爆倉嗎？

心理檢查：

  • [ ] 如果這個月虧 30%，我還能冷靜交易嗎？
  • [ ] 如果連續虧 10 筆，我還有信心嗎？
  • [ ] 我有設定「停止交易」的虧損上限嗎？

壓力測試常見陷阱

過度依賴歷史數據

  • 問題：未來危機可能與過去完全不同
  • 解決：加入假設情境、反向壓力測試

忽略相關性變化

  • 問題：危機時資產相關性會急劇上升
  • 解決：在壓力情境中假設高相關性

低估流動性風險

  • 問題：壓力情境下可能賣不掉
  • 解決：加入流動性折價、價差擴大假設

模型風險

  • 問題：模型假設可能錯誤
  • 解決：使用多種模型交叉驗證

執行頻率過低

  • 問題：市場變化快速，舊測試可能過時
  • 解決：每季執行、重大事件後更新

心得

壓力測試是風險管理的「最後一道防線」——不是預測會發生什麼，而是問自己「如果最壞的情況發生，我還能活著嗎？」

這次的學習讓我意識到幾個重要概念：

1. 回測會騙人：Build Alpha 的研究顯示，回測回撤可能是 $1,663.90，但蒙地卡羅模擬可能達到 $5,195.17。這意味著我們看到的「最大回撤」可能嚴重低估真實風險。

2. 反向壓力測試是底線思維：與其問「可能虧多少」，不如問「虧多少會讓我破產/斷頭/退場」，然後反推什麼情況會導致這個結果。這是更實際的風險管理方式。

3. 個人交易者更需要壓力測試：機構有風控部門，個人交易者往往只看回測績效，忽略了極端情況。一個簡單的問題：「如果大盤跌停 + 跳空 + 流動性枯竭，我的部位會怎樣？」可能救了一個帳戶。

4. 壓力測試不是一次性的：市場會變，策略會老化，壓力測試要定期更新。每季至少一次，重大事件後立即更新。

實務應用：

  • 每季對我的交易帳戶做一次壓力測試
  • 定義個人「不可接受結果」（如：帳戶砍半、被迫斷頭）
  • 找出觸發這些結果的情境
  • 確保有足夠的安全邊際（現金、停損、熔斷）

一句話總結：「壓力測試不是預測未來，而是為最壞的情況做準備。」

核心知識

回測過度擬合（Overfitting）

是什麼：策略被過度調整以完美契合歷史數據，像把考試答案背下來而不是理解概念，遇到新題目就崩潰。

怎麼辨識：
1. 回測報酬率異常高（年化 > 20%、Sharpe > 3 要警覺）
2. 參數過多、規則太複雜
3. 績效依賴少數幾筆「幸運」交易
4. 歷史績效完美，實盤立即虧損

實例：

  • 情境：某策略回測 Sharpe Ratio = 5.2，年化報酬 45%
  • 判斷：這種數據在真實市場極罕見，高度懷疑過度擬合
  • 對策：用樣本外數據驗證、簡化參數、測試不同市場

適用時機：每次回測都要檢查這些警訊

回測七大罪

1. 存活偏誤（Survivorship Bias）

是什麼：只用還存在的股票回測，忽略已經下市的失敗公司。

怎麼做：
1. 使用包含下市股票的完整資料庫
2. 台股可用 TEJ 企業資料庫
3. 美股要注意 CRSP 或 Bloomberg 只有存活股票的問題

實例：

  • 情境：回測 2010-2020 年「買入所有上市公司」策略
  • 錯誤：只用現存股票，忽略了期間下市的 300 家公司
  • 結果：績效被高估，實際風險被低估

適用時機：長期回測（>5年）、價值投資策略、小型股策略

2. 前視偏誤（Look-Ahead Bias）

是什麼：在交易決策時使用了當時還不知道的資訊。像拿明天的報紙下注。

常見錯誤：
1. 用收盤價做當日盤中交易決策
2. 用財報公布後的數據，但設定在公布日前交易
3. 程式碼不小心引用到未來數據

怎麼防範：
1. 使用「時間點」資料（Point-in-Time Data）
2. 確保每筆交易只用「當時已知」的數據
3. 用事件驅動回測系統，按時間順序處理數據

實例：

  • 錯誤：用「當日最高價突破」做當日開盤買進訊號
  • 正確：用「昨日最高價突破」做今日開盤買進訊號

警訊：Sharpe Ratio > 1.5 或年化報酬 > 12%，要檢查是否有前視偏誤

3. 故事偏誤（Storytelling Bias）

是什麼：先發現有效模式，事後才編造經濟邏輯解釋。這是數據挖掘的陷阱。

怎麼做：
1. 先提出假設：為什麼這個策略應該有效？
2. 再驗證數據：用回測檢驗假設
3. 記錄理由：在測試前寫下策略邏輯

實例：

  • 錯誤：測試發現「每週二買進，週三賣出」好像賺錢，於是編造「週二法人進場」的理由
  • 正確：先有「週二法人佈局效應」假設，再驗證是否真有此效應

4. 過度擬合（Overfitting）

是什麼：測試太多參數組合，找到「剛好 fit 歷史數據」的垃圾策略。四個參數可以擬合大象，五個參數可以讓大象搖鼻子。

怎麼防範：
1. 簡化策略：參數越少越好
2. 追蹤測試次數：記錄總共測試了幾組參數
3. 使用 Deflated Sharpe Ratio：調整多重測試偏誤
4. 樣本外測試：80% 數據訓練，20% 數據驗證

實例：

  • 情境：測試了 10,000 種參數組合，找到一組 Sharpe = 4.0
  • 判斷：這幾乎必然是過度擬合，純粹是機率問題
  • 對策：減少參數、報告測試次數、使用統計調整

5. 忽略交易成本

是什麼：假設無摩擦市場，忽略手續費、滑價、價差衝擊。

成本明細：

成本類型說明典型影響
佣金券商手續費0.1425% (台股)
交易稅證交稅0.1% (台股)
滑價實際成交與預期價差0.01-0.1%
市場衝擊大單影響價格0.1-1%

實例：

  • 情境：高頻策略回測年化 15%，但每日交易 10 次
  • 計算：每次交易成本 0.15%，每日 1.5%，一年 250 天 = 375%
  • 結果：策略實際是嚴重虧損

怎麼做：
1. 每筆交易加入 0.1-0.3% 成本
2. 高頻策略要用 tick 級數據估算滑價
3. 大部位要考慮市場衝擊

6. 櫻桃採摘（Cherry-Picked Periods）

是什麼：只選對策略有利的時段回測，避開金融海嘯、疫情崩盤等不利時期。

怎麼做：
1. 回測要涵蓋完整景氣循環（至少 10 年）
2. 一定要包含一次重大崩盤（2008、2020）
3. 測試多頭、空頭、盤整三種市場

實例：

  • 情境：只回測 2012-2014（美股多頭）
  • 問題：策略在 2008、2020 會怎樣？不知道
  • 正確：回測 2006-2016，包含多空循環

7. 不切實際的放空假設

是什麼：假設放空和做多一樣容易，忽略借券成本、券源限制、軋空風險。

現實限制：

  • 借券成本：熱門券 5-30% 年息
  • 券源限制：無券可借、隨時被召回
  • 軋空風險：無限漲幅，強制回補

怎麼做：
1. 借券成本加入回測
2. 假設放空部位有限制
3. 加入軋空情境壓力測試

過度擬合檢測指標

Profit Factor 正常範圍：1.5 - 2.0

  • > 2.5 要警覺
  • > 3.0 高度懷疑過度擬合

Sharpe Ratio 正常範圍：< 3

  • > 3 極罕見，要檢查
  • > 5 幾乎必然有問題

Sortino Ratio 正常範圍：< 3

  • 邏輯同 Sharpe

Walk-Forward 分析法

是什麼：滾動式驗證策略，模擬真實的「訓練-驗證」循環。

怎麼做：
1. 將數據分成多個期間（如每年一個期間）
2. 用前一期間優化參數
3. 用本期間驗證績效
4. 滾動到下一期間重複

實例：

期間1（2010-2011）：訓練
期間2（2012）：驗證
期間2（2011-2012）：訓練
期間3（2013）：驗證
...以此類推

優點：比單次樣本外測試更嚴格，更接近實際交易

樣本外測試（Out-of-Sample Testing）

怎麼做：
1. 將數據切成 80% 訓練組 + 20% 測試組
2. 用訓練組設計策略、調整參數
3. 完全不碰測試組
4. 最後用測試組驗證一次，只看結果，不調參數

實例：

  • 情境：有 10 年數據（2010-2019）
  • 做法：用 2010-2017 設計策略，2018-2019 驗證
  • 關鍵：2018-2019 的數據在設計過程中完全不能看

模擬交易（Paper Trading）終極驗證

是什麼：用即時數據但不實際交易，驗證策略在真實市場的表現。

怎麼做：
1. 策略設計完成後，先紙上交易 1-3 個月
2. 記錄所有訊號、進出場價格、滑價
3. 比較實際 vs 預期績效
4. 確認穩定再投入真實資金

適用時機：

  • 新策略上線前
  • 大幅修改參數後
  • 市場結構改變後

多市場驗證

是什麼：同一策略在不同市場/標的測試，確認不是只對單一資產有效。

怎麼做：
1. 用台股設計的策略，也測試美股、港股
2. 用大型股設計的策略，也測試中小型股
3. 用多頭時期設計的策略，也測試空頭時期

實例：

  • 情境：均線突破策略在台積電回測 Sharpe = 1.8
  • 驗證：同樣策略在聯發科、鴻海 Sharpe 分別為 1.2、0.9
  • 判斷：策略在台積電可能過度擬合

多空市場測試

市場因子在不同環境的績效：

因子多頭市場空頭市場
價值中等強正
動能低強正
規模（SMB）正負
獲利能力平均強正

關鍵：價值、獲利能力、動能在空頭市場比多頭市場更有利

心得

這回合學的是回測陷阱——老實說，這是所有量化交易者最容易踩雷的地方。

最大的學習：
1. 過度擬合是「看起來很美」的陷阱：回測績效太好通常是壞事，不是好事。Sharpe > 3 幾乎必然有問題。

2. 你永遠不知道你不知道什麼：存活偏誤、前視偏誤這些東西，你不去特意檢查就不會發現。要有意識地去挑戰自己的假設。

3. 成本是策略殺手：很多策略回測看起來賺錢，加上真實成本就變虧損。高頻策略尤其致命。

4. 最誠實的驗證是實盤：再怎麼嚴謹的回測都比不上真實市場。Paper trading 是必要的最後一關。

5. 簡單比複雜好：參數越少，策略越robust。四個參數就能擬合大象，別再加了。

回測的目的是「淘汰爛策略」，不是「證明好策略」。心態要對。

核心知識

R-Multiple 基礎概念

是什麼：R = 初始風險（每筆交易願意承擔的最大虧損金額）。R-Multiple = 實際損益除以初始風險，用來標準化衡量交易績效。

怎麼算：

  • 風險 $100，賺 $300 → +3R
  • 風險 $100，賠 $100 → -1R
  • 風險 $100，賺 $50 → +0.5R
  • 風險 $100，賠 $50 → -0.5R

實例：

  • 情境：買進台積電 $600，停損設 $570（風險 = $30）
  • 目標 $660（潛在報酬 = $60）
  • 潛在 R-Multiple = $60 / $30 = 2R
  • 若股價真的到 $660，你賺了 2R

為什麼重要：
1. 標準化不同部位大小的交易結果，方便比較和分析
2. 專注於風險管理，而非金錢數字
3. 讓你能跨日期、跨標的、跨策略做績效比較

期望值計算（R版）

公式：

期望值 = (勝率 × 平均獲勝R) - (敗率 × 平均虧損R)

實例：

  • 策略 A：勝率 70%，平均獲勝 1R，平均虧損 1R
  • 策略 B：勝率 40%，平均獲勝 3R，平均虧損 1R

關鍵洞察：策略 B 勝率較低但期望值較高！高勝率 ≠ 高獲利。

適用時機：

  • 評估交易策略的長期盈利能力
  • 比較不同交易系統的績效
  • 決定是否調整停損停利設定

損益平衡勝率對照表

概念：各種 R:R 需要的最低勝率

R:R損益平衡勝率
1:150%
2:133%
3:125%
4:120%
5:117%

實例：

  • 3:1 R:R 策略：只要 25% 勝率就能打平
  • 意味著你可以錯 3 次只對 1 次，長期下來仍然獲利
  • 但這不表示你應該追求低勝率，而是讓你有「犯錯空間」

R-Multiple 策略選擇

怎麼選：依據勝率、市場波動度、時間框架選擇適合的 R-Multiple 目標。

1x R-Multiple（損益平衡取向）

  • 適用：高頻交易、剝頭皮策略
  • 勝率需求：> 50%
  • 優點：進場機會多、心理壓力較小
  • 缺點：需要高勝率才能獲利

1.5x R-Multiple（適中獲利取向）

  • 適用：精準度高的日內交易
  • 勝率需求：50-55%
  • 優點：風險報酬平衡、機會仍多
  • 缺點：需要一定技術分析能力

2x R-Multiple（標準目標）

  • 適用：主流策略、適合大多數交易者
  • 勝率需求：> 40%
  • 優點：容錯率高、適合趨勢與區間策略
  • 缺點：需要耐心等待好的進場點

3x R-Multiple（高報酬緩衝）

  • 適用：波動市場、波段交易
  • 勝率需求：25-30%
  • 優點：容錯率極高、適合捕捉大行情
  • 缺點：交易頻率降低、需要高信心

4x+ R-Multiple（最大獲利取向）

  • 適用：精選波段、部位交易
  • 勝率需求：20-25% 或更低
  • 優點：容錯率最高、捕捉大行情
  • 缺點：需要極大耐心和信心

實例：

  • 情境：台積電突破 $600 整理區
  • 停損設 $570（風險 $30）
  • 用 3x 目標 = $690（報酬 $90）
  • 若認為 $690 太遠、不現實
  • 改用結構目標：下一檔壓力在 $650
  • 實際 R:R = $50/$30 ≈ 1.67x
  • 評估：風險報酬比不錯，可以進場

R-Multiple 追蹤實務

要追蹤什麼：
1. 每筆交易的初始 R（風險金額）
2. 實際 R-Multiple（正或負）
3. 平均獲勝 R 值
4. 平均虧損 R 值
5. 總期望值（R/筆）

分析方法：

  • 平均獲勝 R 太低？→ 可能在過早停利
  • 平均虧損 R 太高？→ 可能在移動停損或滑價
  • 哪些 R:R 區間績效最好？→ 優化策略方向

實例追蹤表：

交易風險(R)結果R-Multiple
1$500賺$1,500+3R
2$500賠$500-1R
3$500賺$1,000+2R
4$500賠$500-1R
5$500賺$250+0.5R
  • 平均獲勝 R = (3+2+0.5)/3 = 1.83R
  • 平均虧損 R = -1R
  • 勝率 = 60%
  • 期望值 = (0.60 × 1.83) - (0.40 × 1) = 0.70R/筆

常見 R-Multiple 錯誤

錯誤 1：追逐勝率

  • 過早停利「鎖定獲利」→ 把 3R 變成 0.5R
  • 結果：高勝率但低期望值

錯誤 2：忽略風險

  • 沒有停損 = 無法定義 R
  • 一筆大虧損可能抵銷多筆獲利

錯誤 3：隨意移動目標

  • 情緒化更改停利點
  • 恐懼下過早出場
  • 期待虧損會回本

錯誤 4：只看金額不看 R

  • 「我賺了 $500」→ 沒意義
  • 「我賺了 2R」→ 有意義，因為標準化可比較

設定實際的 R-Multiple 目標

考慮因素：

因素 1：市場結構

  • 下一檔支撐/壓力在哪？
  • 有足夠空間達到目標嗎？
  • 不要設定超出合理範圍的目標

因素 2：波動度

  • 高波動 = 需要更寬的停損
  • 目標應該隨波動度縮放
  • 使用 ATR 來校準

因素 3：時間框架

  • 長時間框架 = 可能的移動幅度更大
  • 短時間框架 = 目標要更緊
  • R:R 要與時間框架匹配

因素 4：歷史分析

  • 你的策略歷史上達成的 R:R 是多少？
  • 這種 setup 實際可能達到多少 R？
  • 不要假設不切實際的 R:R

R-Multiple 心態

用 R 思考，不用金錢：

  • 說「我賺了 2R」而不是「我賺了 $500」
  • 說「我賠了 1R」而不是「我賠了 $250」
  • 標準化後才能比較不同部位

專注期望值，不是勝率：

  • 連續虧損不代表系統壞了
  • 期望值為正，結果終將來臨
  • 相信數學，而非短期結果

接受虧損是經營成本：

  • -1R 是預期且已計畫的
  • 這是「參與市場的費用」
  • 獲利會補償這些成本

合理期望值範圍

不切實際的期望：

  • 「我只做 5:1 的交易」→ 這種機會很少
  • 「我從不虧損超過 0.5R」→ 滑價會發生
  • 「我平均獲勝是 4R」→ 用數據驗證

實際的期望：

  • 平均獲勝 R：1.5-2.5R 就很不錯
  • 平均虧損 R：-0.8 到 -1.2R 是正常的
  • 期望值：0.2-0.5R/筆 就很穩健

心得：

R-Multiple 是我最喜歡的概念之一 — 簡單但被嚴重低估。

用 R 當單位徹底改變看待交易的方式。以前我會盯著「賺多少錢」或「賠多少錢」，但這些數字沒辦法跨時間、跨標的比較。用 R 之後，每筆交易變成一個標準化的數字，可以累積分析、找出模式。

最震撼的是那個數學：40% 勝率加 3R 目標，竟然比 70% 勝率加 1R 目標還賺更多。這完全顛覆「要贏多輸少才會賺錢」的直覺。事實上，你要的是「期望值為正」，不是「勝率很高」。

兩個核心洞察：
1. 容錯空間比勝率重要 — 高 R:R 讓你能連錯幾次仍不傷筋骨
2. 追蹤 R，不是追蹤金額 — 「我平均獲勝 2R」比「我平均賺 $400」有意義得多

實戰上，我現在會：

  • 進場前先定好「這筆交易的 R 是多少」
  • 用結構（支撐壓力）來設定停損停利，再算 R:R
  • R:R 低於 2 的 setup，除非勝率特高，否則跳過
  • 每週計算平均獲勝 R、平均虧損 R、期望值

R-Multiple 把「風險」變成一個可量、可比較、可管理的單位。這才是風險管理的正確姿勢。

是什麼：衡量標的資產價格變動1單位時，選擇權權利金變動多少單位的指標。

公式：

數值範圍：

類型Delta範圍意義
買權 (Call)0 ~ +1正相關，股漲權漲
賣權 (Put)-1 ~ 0負相關，股跌權漲

期貨空單-1完全負相關

價位狀態對照：

  • 價平 (ATM)：Delta ≈ ±0.5
  • 價內 (ITM)：Delta 接近 ±1（深度價內更接近）
  • 價外 (OTM)：Delta 低於 ±0.5（深度價外接近 0）

實例：

  • TSLA賣權 Delta = -0.75，股價下跌$1，權利金上漲約$0.75

是什麼：Delta的絕對值可以粗略估計選擇權成為價內的機率。

怎麼用：
1. 查看選擇權的Delta值
2. Delta絕對值 = 成為價內的機率
3. 用於評估賣方勝率

實例：

  • 履約價20500的買權，Delta = 0.2
  • 代表目前指數有20%機率漲破20500
  • 賣出此買權，勝率約80%

雙賣勝率計算：

賣出 Delta 0.2 的 Call + 賣出 Delta -0.2 的 Put
勝率 = 100% - 20% - 20% = 60%

Delta中性避險策略

是什麼：透過調整部位，讓整體Delta總和接近0，消除方向性風險。

核心目標：

  • 不預測市場方向
  • 將風險轉向時間價值（Theta）或波動率（Vega）
  • 保護既有部位

動態避險實戰步驟

怎麼做：

步驟一：計算部位總Delta

總Delta = Delta值 × 合約張數 × 乘數(100)

步驟二：判斷避險方向

  • 總Delta > 0（偏多）→ 需要負Delta工具避險（買Put或賣Call）
  • 總Delta < 0（偏空）→ 需要正Delta工具避險（買股票或買Call）

步驟三：計算避險數量

避險數量 = |總Delta| ÷ 避險工具的Delta

步驟四：建倉後持續監控

  • Delta會隨股價變動而改變
  • 需要定期再平衡（每日或每週）

實戰案例一：期貨多單避險

情境：

  • 持有1口小台多單（Delta = +1）
  • 擔心短期下跌但不想平倉

操作：

買進 2口價平賣權（每口Delta = -0.5）
總Delta = 1 + (−0.5 × 2) = 0

結果：Delta歸零，部位不偏多不偏空，等待危機過去再平倉賣權。

替代方案：

  • 只買1口價外賣權（成本較低）
  • 若指數跌破該履約價，賣權獲利抵銷期貨虧損

實戰案例二：賣權部位避險

情境：

  • 持有1張ABC公司賣權（Delta = -0.60，乘數100）
  • 擔心股價上漲造成虧損

計算：

總Delta = -0.60 × 1 × 100 = -60

操作：

買進 60股 ABC股票（每股正Delta = +1）
總Delta = -60 + 60 = 0（中性）

成本計算：

  • ABC股價 $25
  • 避險成本 = $25 × 60股 = $1,500

Delta中性的優點

優點說明
降低方向風險價格小漲小跌影響有限

規避極端行情對抗黑天鵝事件的緩衝

Gamma風險詳解

是什麼：Gamma衡量Delta的變化率，即「Delta的Delta」。

影響：

  • Gamma越高，Delta變動越劇烈
  • 震盪市場中，Delta中性會快速失效
  • 需要更頻繁的再平衡

實務建議：

  • 價平選擇權Gamma最大，要特別注意
  • 接近到期日Gamma會增加
  • 高波動環境下減少Delta中性操作

實戰SOP

建立Delta中性部位：

1. 計算現有部位的總Delta
2. 決定避險工具（股票、期貨、選擇權）
3. 計算需要的避險數量
4. 建立反向部位
5. 確認總Delta接近0

維持Delta中性：

1. 每日收盤檢查Delta值
2. 設定再平衡門檻（如Delta偏離±0.1）
3. 評估調整成本是否划算
4. 記錄每次調整的原因和結果

退出Delta中性：

1. 當風險事件過去
2. 或原本部位已達止損/止利條件
3. 平掉避險部位，恢復原始方向

心得

Delta中性避險是進階的風險管理技巧，核心概念很簡單：讓正負Delta互相抵消，消除方向風險。但實戰中最大的挑戰是「動態再平衡」——不是做一次就沒事，而是要持續監控調整。

這策略最適合的情境：
1. 持有長期部位但短期有風險事件（財報、選舉、政策）
2. 不想平倉但想降低風險
3. 想賺時間價值或波動率而非方向

最大的坑在於成本和Gamma風險。每次再平衡都有成本，高Gamma時更是要不停調整，搞不好調整成本比避險效益還高。

實戰建議：

  • 設定明確的再平衡門檻，不要有感覺就調整
  • 避開高Gamma時段（價平、接近到期）
  • 算清楚總成本（包括手續費、滑價、時間價值損耗）
  • 把避險當保險費，不是獲利手段
