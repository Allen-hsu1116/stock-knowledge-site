一、ATR 指標基礎概念

1. ATR 是什麼？

  • 全名：Average True Range（平均真實波幅）
  • 發明者：J. Welles Wilder Jr.（RSI、ADX 的發明者）
  • 功能：衡量市場波動率，不是預測方向
  • 核心理念：是一把「動態的尺」，測量特定時間週期內價格的平均波動幅度
  • 解讀：

2. True Range（真實波幅）計算公式

  • TR 取以下三者的最大值：

1. |當期最高價 - 當期最低價|
2. |當期最高價 - 前期收盤價|
3. |當期最低價 - 前期收盤價|

  • 為什麼要這樣算？：傳統算法忽略跳空缺口，TR 將跳空也納入考量
  • ATR：過去 N 期 TR 的移動平均（預設 14 期）

3. ATR 參數設定

  • 預設值：14 週期（適用大多數情況）
  • 短線交易：縮短至 5-7 週期，對近期波動更敏感
  • 長線投資：放寬至 20-25 週期，平滑短期雜訊

二、ATR 停損策略（動態風險管理）

1. ATR 固定倍數停損法

  • 具體操作：停損點設在進場價 ± N × ATR
  • 步驟：

1. 確認進場訊號（均線交叉、趨勢線突破等）
2. 讀取進場時的 ATR 數值（如 ATR(14) = 70 pips）
3. 計算停損距離 = ATR × 倍數（常用 1.5~3 倍）
4. 做多停損價 = 進場價 - 停損距離
5. 做空停損價 = 進場價 + 停損距離

  • 實例：

2. 為什麼用 ATR 倍數而非固定點數？

  • 高波動市場：固定 50 pips 停損容易被「洗盤」出場
  • 低波動市場：固定 50 pips 停損又過寬，風險過大
  • ATR 解決方案：讓停損「動態適應」當前市場波動

3. ATR 停損倍數選擇建議

  • 1.5 倍 ATR：適合短線、當沖，停損較緊
  • 2 倍 ATR：最常用，平衡風險與被洗出的機率
  • 3 倍 ATR：適合波段、長線，給足空間

三、ATR 追蹤停損法（Trailing Stop）

1. 追蹤停損原理

  • 目的：保護已有利潤，同時抓住整個趨勢
  • 做多的追蹤停損：停損點 = 當前價格高點 - N × ATR
  • 做空的追蹤停損：停損點 = 當前價格低點 + N × ATR
  • 關鍵規則：停損點只能往有利方向移動，絕不後退

2. 具體操作步驟

  • 進場後：設定初始停損 = 進場價 - 2 × ATR
  • 價格上漲創新高：停損點上移至「新高點 - 2 × ATR」
  • 價格下跌未破停損：停損不動，繼續持有
  • 價格跌破停損：出場，鎖定利潤

3. 實例演示

  • 100 元買進，ATR = 3 元
  • 初始停損 = 100 - 2×3 = 94 元
  • 漲到 110 元：停損上移至 110 - 6 = 104 元
  • 漲到 120 元：停損上移至 120 - 6 = 114 元
  • 回跌到 114 元：觸及停損，出場，鎖定 14% 利潤

四、ATR 資金管理（風險一致化）

1. 核心觀念

  • 不同市場、不同時期的波動度不同
  • 用固定金額下單會導致：高波動時風險大、低波動時風險小
  • 解決方案：用 ATR 調整倉位，讓每筆交易風險一致

2. 倉位大小計算公式

倉位大小 = (帳戶總資金 × 風險百分比) / (ATR 值 × 停損倍數 × 每點價值)

3. 實例計算

  • 帳戶資金：100 萬元
  • 單筆風險：2%（2 萬元）
  • 目前 ATR：50 元
  • 停損倍數：2 倍 ATR
  • 每點價值：1 元
  • 倉位大小 = 2 萬 / (50 × 2 × 1) = 200 股

4. 風險一致化的效果

  • 高波動市場（ATR 大）：自動減少倉位
  • 低波動市場（ATR 小）：自動增加倉位
  • 每筆交易的最大虧損金額都控制在預設範圍（如帳戶的 1-2%）

五、ATR 搭配其他指標的進階策略

1. ATR + ADX 趨勢強度確認

  • ADX（平均趨向指標）：衡量趨勢強度（非方向）
  • ADX < 25：無趨勢或趨勢弱，盤整
  • ADX > 25 且向上突破：趨勢正在形成
  • 策略：

1. ADX 從低點向上突破 25
2. 確認價格方向（均線之上做多，之下做空）
3. 使用 ATR 設定停損（強趨勢下 ATR 較大，需放寬停損）

**2. ATR + 布林通道突破}

  • 布林通道帶寬收窄：稱為「擠壓」，預示大行情
  • ATR 處於相對低位：市場平靜，可能是暴風雨前的寧靜
  • 策略：

1. 布林通道帶寬收窄 + ATR 低
2. 等待價格強勢突破布林上下軌
3. ATR 開始上揚確認突破有效性
4. 進場，用 ATR 設定停損

七、如何規劃避險策略（三大重點）

1. 衡量自己的避險能力

  • 關鍵：像主動 vs 被動投資一樣，有能力才主動，沒能力就被動

2. 判斷執行避險的時機

  • 該避險的時機：進場理由「可能改變」

八、避險的反向思考：承受風險

1. 投資本質是承受風險換取報酬

  • 定存、短期公債：幾乎不承受波動風險，報酬低
  • 股票：短期波動大，長期報酬較高
  • 高報酬來自「不確定性」

2. 應該問的問題

  • 錯誤問法：「怎麼避開所有風險？」
  • 正確問法：「我願意承受怎樣的風險？」

3. 核心結論

  • 完全避開風險 = 錯失市場報酬（機會成本）
  • 好的投資組合 = 能承受各種意外風險
  • 讓組合能承受風險，比每次意外都避險更重要

優點：

ATR 策略優點：

  • 客觀性：量化的波動數據，避免主觀猜測
  • 適應性：自動適應市場波動，高波動停損寬、低波動停損緊
  • 通用性：適用各種金融商品、各種時間週期
  • 資金管理：可精確計算倉位大小，確保風險一致
  • 動態追蹤：追蹤停損既能保護利潤又能抓住趨勢

避險策略優點：

  • 事前成本低：事先規劃比臨時應變便宜
  • 保護極端風險：避免黑天鵝事件造成毀滅性傷害
  • 維持紀律：預設避險方案避免情緒化決策
  • 控制傷害範圍：將損失控制在可接受的範圍內

缺點/限制：

ATR 策略缺點：

  • 滯後性：基於歷史數據，對突發極端行情反應不足
  • 無方向性：只告訴波動大小，不判斷漲跌方向
  • 倍數主觀：1.5 倍 vs 2 倍 vs 3 倍需自行回測
  • 極端行情風險：ATR 值過大時，停損點可能過寬

避險策略缺點：

  • 機會成本：避險期間錯過的報酬可能很高
  • 可能追高殺低：事後避險容易賣低買高
  • 打破紀律風險：沒規劃的避險 = 破壞原有策略
  • 長期報酬代價：可能每年損失 2-3% 報酬

1. 買進後記錄最高價
2. 停損價 = 最高價 × (1 - 停損百分比)
3. 股價創新高 → 用新高點重新計算停損價
4. 股價回落觸及停損價 → 賣出

3. 支撐線停損法

  • 具體操作：跌破技術分析的支撐線時賣出
  • 步驟：

1. 畫出股價趨勢圖
2. 找出支撐線（價格多次觸及反彈的低點連線）
3. 股價跌破支撐線且沒有反彈 → 停損出場

  • 關鍵判斷：

-「碰到支撐線反彈」= 支撐有效，續持有
-「跌破支撐線不回頭」= 支撐失守，出場

4. 基本面停損法（買進理由消失）

  • 具體操作：不看股價數字，而是檢視「買進理由」是否還成立
  • 步驟：

1. 記錄買進這支股票的理由（如：看好營收成長、政策利多、新產品）
2. 定期檢視公司財報、新聞、產業變化
3. 買進理由消失了（公司轉壞、產業變差）→ 賣出

二、停損點設置的優缺點分析

優點：

  • 減少情緒波動：心裡對最壞狀況有準備，不會因下跌緊張不安
  • 控制風險：停損點是你自己設定的，可根據財務狀況決定
  • 避免重大虧損：價格到停損點就賣出，不用擔心虧損越來越大

缺點：

  • 增加交易成本：停損點設很緊，頻繁交易產生手續費
  • 錯失上漲機會：停損後股價上漲，捨不得買回就錯過
  • 忽略公司體質：只看價格數字波動，忘記檢視公司真實價值

重要觀念：

  • 長期投資人「不設停損」不會怎樣
  • 設停損短期保護資產，長期可能錯過獲利
  • 關鍵是先問自己「為什麼要投資這檔股票」
  • 想清楚買進理由，比設停損更重要

三、凱利公式（Kelly Criterion）- 科學化倉位管理

1. 凱利公式是什麼？

  • 定義：在給定勝率與盈虧比下，計算出讓長期資本成長速度最快的投資比例
  • 目的：平衡「增長」與「生存」
  • 核心信念：獲利是勝率、盈虧比與倉位大小的乘積
  • 關鍵洞察：即便策略勝率 90%，下注規模超過凱利公式邊界，長期「歸零機率」趨近 100%

2. 系統運作邏輯

倉位優化（Position Optimization）：

  • 根據回測所得的期望值與當前策略信心度
  • 自動計算建議投入比例
  • 公式輸出的是「應該投入多少資金」

回撤控制（MDD Management）：

  • 凱利公式的目標：將最大回撤（MDD）控制在 10% 以內
  • 即使連續虧損，整體權益曲線仍具恢復力

盈虧比綁定：

  • 系統要求預期盈虧比須大於 2.0
  • 盈虧比 >= 2 才能用較低勝率獲利
  • 確保長線獲利效率

3. 實戰案例分析

成功用法 - 在優勢區間放大收益：

  • 場景：系統偵測到「主升攻擊」機制，8 維度共識達成強共振（8/8）
  • 觀察：勝率與預期盈虧比同時處於高檔
  • 凱利公式計算：給出較高的建議倉位
  • 結論：在統計優勢時刻重拳出擊，低風險前提下實現跳躍式增長

失敗用法 - 過度押注導致毀滅風險：

  • 場景：投資者因連續獲利產生過度自信，無視凱利上限，全倉交易
  • 觀察：市場突發黑天鵝，極短線劇烈震盪觸發強制平倉
  • 結論：典型「過度博弈」，忽略倉位風險，長期必輸

4. 關鍵實戰建議

  • 用半凱利（Half-Kelly）：全額按凱利公式下注波動太大，建議只投入計算結果的「一半」
  • 盈虧比 > 2：低於 2 的策略要極高勝率才能獲利
  • 不要過度最佳化：參數調太完美 = 對歷史數據過度擬合
  • 記入交易成本：手續費、滑價都會侵蝕獲利

四、資產配置與分散投資（風險管理核心）

1. 什麼是資產配置？

  • 定義：將資金分配至不同資產類別（股票、債券、現金等）
  • 目的：降低單一資產波動對整體投資組合的影響
  • 核心精神：不是預測市場，而是建立能穿越景氣循環的投資架構

2. 三種配置類型

類型股票比例債券比例適合對象
保守型低（20-30%）高（70-80%）風險趨避者、退休族
平衡型中（50%）中（50%）穩健投資者
積極型高（70-80%）低（20-30%）年輕人、高風險承受者

3. 現代投資組合理論（MPT）

核心概念：

  • 在給定風險水準下，透過組合不同相關性資產
  • 建構「有效邊界」（Efficient Frontier）
  • 使投資人獲得相對最佳的風險報酬組合

關鍵洞察：

  • 資產之間的「相關性」是關鍵
  • 相關性越低，分散效果越好
  • 整體組合風險 < 單一資產風險加總

4. 分散投資三個維度

資產類別：

  • 股票 vs 債券
  • 美股 vs 台股 vs 新興市場
  • 公債 vs 公司債

產業分散：

  • 不要重押單一產業

地區分散：

  • 新興市場（中國、印度、東南亞）
  • 不同地區經濟週期不同步

5. 策略型 vs 戰術型配置

策略型資產配置：

  • 根據長期目標設定固定比例
  • 定期再平衡（如每季、每年）
  • 一般投資人較適合

戰術型資產配置：

  • 依市場短期變化調整比例
  • 屬於主動策略
  • 需要市場判斷能力

6. 再平衡的重要性

為什麼要再平衡：

  • 市場波動會讓比例偏離目標
  • 例如：目標股債 60/40，股市大漲變成 75/25
  • 需要賣股買債「再平衡」回 60/40

再平衡作用：

  • 維持原本風險水準
  • 避免單一資產比重過高
  • 建立紀律性操作機制
  • 研究顯示：再平衡有助風險控制，非提升報酬

7. 新手常見錯誤

  • 過度集中投資（單一股票或產業）
  • 忽略再平衡
  • 情緒化調整配置
  • 將短期市場波動誤認為策略失敗

五、期望值計算（交易員的「廟算」）

1. 什麼是廟算？

  • 典故：《孫子兵法》「多算勝，少算不勝」
  • 古代：將領出征前在宗廟推演兵力、糧草、天時地利
  • 現代交易：進場前的數據回測與期望值計算
  • 目的：在開戰第一秒就決定勝負

2. 核心公式：期望值

期望值 = (勝率 × 平均獲利) - (敗率 × 平均虧損)

範例計算：

  • 勝率 50%
  • 平均獲利 10,000 元
  • 平均虧損 5,000 元
  • 期望值 = (50% × 10,000) - (50% × 5,000) = 2,500 元

解讀：

  • 正期望值 → 系統長期有利可圖
  • 負期望值 → 長期必輸
  • 正期望值 + 樣本數足夠（>100 次）→ 有進場底氣

3. 廟算的兩個核心

數據回測（現代版沙盤推演）：

  • 有交易想法後，丟進歷史數據測試
  • 過濾掉「看起來會賺」的垃圾靈感
  • 很多想法在腦中完美，回測後 MDD 高得嚇人，勝率連 40% 都不到

計入交易成本：

  • 手續費、滑價會侵蝕獲利
  • 頻繁進出導致成本大幅侵蝕
  • 沒把摩擦成本算進去 = 勝算是幻覺

4. 廟算的致命盲點

過度最佳化（Curve Fitting）：

  • 為追求完美績效曲線而過度調整參數
  • 等於「竄改歷史考題」
  • 必須用走勢最佳化（WFO）確保策略在未知數據中依然有效

忽視最大回撤：

  • 期望值正不代表中途不會虧損
  • 要確認「連續虧損後，系統是否能靠小輸大贏回歸正值」

5. 交易者的日常分配

  • 80% 時間：在安靜環境裡計算與研究
  • 20% 時間：冷靜地執行計算後的結果
  • 不是臨場發揮，而是事前準備

二、五種倉位計算策略



1. 固定百分比法（Fixed % Risk）



    

  • 具體操作：每次交易僅使用帳戶總額的固定比例（如 1-2%）作為風險額
    • 

  • 步驟：
    • 




1. 計算單筆風險金額 = 帳戶總額 × 風險比例（如 100 萬 × 2% = 2 萬）

2. 計算每股風險 = 進場價 - 停損價

3. 下單股數 = 單筆風險金額 / 每股風險

  • 優點：風險與帳戶規模同步成長，易於紀律執行
    • 

  • 缺點：無法考量當前市場波動性
    • 

  • 適用：長期穩健增長型交易者
    • 




2. 固定金額法（Fixed $ Risk）



    

  • 具體操作：每次交易固定金額作為風險額
    • 

  • 步驟：
    • 




1. 設定固定風險金額（如每筆 5,000 元）

2. 計算每股風險

3. 下單股數 = 固定風險金額 / 每股風險



    

  • 優點：計算簡單明瞭
    • 

  • 缺點：帳戶大幅波動後風險分配不靈活
    • 

  • 適用：小型帳戶、程式化交易策略
    • 




3. ATR 波動率法



    

  • 具體操作：根據 ATR 動態調整倉位大小
    • 

  • 步驟：
    • 




1. 讀取當前 ATR(14) 數值

2. 計算停損距離 = ATR × 倍數（1.5-3 倍）

3. 倉位大小 = (帳戶資金 × 風險%) / (ATR × 停損倍數 × 每點價值)



    

  • 優點：自動考量市場波動性，停損更貼近行情
    • 

  • 缺點：必須正確設定 ATR 週期，高波動時過度縮小倉位可能錯失機會
    • 

  • 適用：追求與市場波動同步的波段交易
    • 




4. Kelly 公式



    

  • 具體操作：根據勝率與盈虧比計算最佳下注比例
    • 

  • 公式：f* = (p × b - q) / b
    • 

  • 實務調整：使用「半凱利」（Half-Kelly）更安全
    • 

  • 優點：理論上可最大化資本成長
    • 

  • 缺點：需精準估算勝率與盈虧比，易過度激進
    • 

  • 適用：資料充分、勝率與盈虧比穩定的策略
    • 




5. 金字塔加碼法（Pyramiding）



    

  • 具體操作：趨勢明確時分批加倉，利潤放大
    • 

  • 步驟：
    • 




1. 趨勢確認後先建立初始倉位

2. 獲利後回調時加碼（不是虧損時加碼）

3. 每次加碼金額遞減（如第一筆 10 萬，第二筆 8 萬，第三筆 6 萬）

4. 停損移至成本價保護本金



    

  • 優點：趨勢明確時可擴大利潤，避免一次重倉
    • 

  • 缺點：趨勢反轉時最後一次加碼風險最大
    • 

  • 適用：趨勢跟隨策略、波段交易
    •

    •

    •

2. 為什麼風險報酬比重要？



    

  • 控管單筆虧損：設定停損上限，防止一次失誤造成毀滅性打擊
    • 

  • 確保長期獲利：勝率 50% 配合 1:2 風險報酬比，長期仍可獲利
    • 

  • 培養紀律：進場前先設停損停利，避免情緒化交易
    • 

  • 評估交易機會：快速評估一筆交易值不值得做
    •

  • 步驟 1：設定進場價、停損價、停利價
    • 

  • 步驟 2：計算可能虧損 = |進場價 - 停損價|
    • 

  • 步驟 3：計算預期獲利 = |停利價 - 進場價|
    • 

  • 步驟 4：風險報酬比 = 預期獲利 ÷ 可能虧損
    • 

  • 步驟 5：判斷是否符合最低標準（建議至少 1:1.5 或 1:2）

3. 關鍵洞察



    

  • 高風險報酬比 + 低勝率：也能獲利
    • 

  • 低風險報酬比 + 高勝率：需要極高勝率
    • 

  • 最差情況：低勝率 + 低風險報酬比 = 長期負期望 = 爆倉
    • 




4. 勝率與風險報酬比的平衡



    

  • 高勝率策略：停利目標近，停損較寬，勝率可達 80% 以上
    • 

  • 高報酬策略：停利目標遠，停損較緊，勝率可能只有 30-40%
    • 

  • 最佳化關鍵：找到適合自己的勝率/報酬比組合
    • 







五、倉位管理常見錯誤



1. 過度槓桿與重倉



    

  • 滿倉交易：一旦逆勢造成嚴重回撤
    • 

  • 高槓桿放大損失：50 倍槓桿 2% 波動就會爆倉
    • 




2. 忽略停損與風險



    

  • 沒設停損：帳戶暴露在無限虧損風險
    • 

  • 隨意移動停損：破壞風險管理紀律
    • 




3. 情緒化倉位調整



    

  • 虧損時加碼「補倉」：越賠越重倉
    • 

  • 獲利時追漲：越賺越追高
    • 

  • 「越賠越重倉、越賺越追高」= 爆倉公式
    • 




4. 忽略投資組合分散



    

  • 集中單一標的或單一市場
    • 

  • 相關資產風險疊加
    • 




5. 過度自信與高追預期



    

  • 小賺就大幅加碼
    • 

  • 遇到逆轉損失更多
    •

2. 停損距離視為「1 單位風險」

2. 使用技術分析找高報酬/低風險區域



    

  • 支撐阻力：在關鍵支撐區域進場，停損設在支撐下方不遠處
    • 

  • 趨勢線突破：突破關鍵點進場，假突破小賠，真突破大賺
    • 

  • 形態學：三角形突破、頭肩底等，風險小、獲利空間大
    • 




3. 追蹤停損保護利潤



    

  • 目的：達到目標後用追蹤停損爭取更大報酬
    • 

  • 操作：價格上漲創高後，停損跟著上移
    • 

  • 效果：將 1:2 的風險報酬比變成 1:3 或更高
    •

  • 在 X 月 X 號，如果價格沒到 X 元，就賣出
    • 

  • 例：買進後 5 個交易日，若未突破前高，賣出
    • 




空間停損點：



    

  • 百分比停損法：虧損 7% 停損
    • 

  • 均線停損法：跌破 5 日均線停損
    • 

  • 支撐停損法：跌破關鍵支撐停損
    • 




最大可能虧損：



    

  • 通常設定比停損更高（因為大盤可能突然暴跌）
    • 

  • 例：停損設 7%，但心理準備最大虧損 10%
    • 




何時看盤決定賣出？



    

  • 開盤 9:00-9:30？
    • 

  • 尾盤 13:00-13:30？
    • 

  • 盤中逢高賣出？
    • 

  • 提前決定，盤中執行
    • 




停利點設定：



    

  • 股價帳面獲利超過 X 元 → 停利
    • 

  • 股價到 X 價位 → 停利
    • 

  • 分批停利：第一目標達成賣一半，第二目標達成全出
    • 




加碼點設定：



    

  • 股價突破 X 價格 → 再買一張
    • 

  • 帳面獲利 X 元 → 再買一張
    • 

  • 金字塔加碼：每次加碼金額遞減
    • 




極端狀況應對（保險）：



    

  • 大盤狂跌千點，個股瞬間穿過停損 → 該賣嗎？
    • 

  • 賣不掉跌破停損 → 該怎麼辦？

  • 永遠為任何狀況做好準備，否則交易就沒有永遠

三、期貨對沖策略



1. 期貨對沖原理



    

  • 持有現貨 + 放空等量期貨 = 鎖定價格
    • 

  • 價格下跌時，現貨虧損，期貨獲利
    • 

  • 價格上漲時，現貨獲利，期貨虧損
    • 

  • 最終結果：價格變動影響相互抵消
    • 




2. 股票期貨對沖實例



情境：持有台積電 100 張，擔心短期下跌

操作：放空台積電期貨 100 張

結果：



    

  • 股價下跌 5%：現貨虧損 5%，期貨獲利 5% → 損益兩平
    • 

  • 股價上漲 5%：現貨獲利 5%，期貨虧損 5% → 損益兩平
    • 




3. 期貨對沖的注意事項



    

  • 基差風險：期貨價格與現貨價格可能有價差
    • 

  • 流動性風險：小台到期貨可能流動性不足
    • 

  • 保證金追繳：期貨虧損需要追繳保證金
    • 

  • 到期日風險：期貨有到期日，需要換倉

核心內容：



一、VaR 風險價值的定義



1. VaR 是什麼？



    

  • 全名：Value at Risk（風險價值）
    • 

  • 定義：在特定信心水準下，某投資組合在特定時間內最大可能的虧損金額
    • 

  • 核心概念：「在最糟糕的情況下，我們最多會虧多少？」
    • 




2. VaR 的三個核心參數



參數說明常見設定
**信心水準**預測的可信程度95%、99%
**時間範圍**預測的期間1 天、10 天、1 個月
**基礎貨幣**計算損失的幣別本國貨幣



3. VaR 的解讀方式



範例：



    

  • VaR = 100 萬元（95%信心水準，1 天）
    • 

  • 意義：有 95% 的信心，明天最大虧損不超過 100 萬
    • 

  • 反之：有 5% 的機率，明天虧損可能超過 100 萬
    • 




4. VaR 的計算方法



方法一：歷史模擬法



    

  • 收集歷史價格變動資料
    • 

  • 計算每個歷史情境的損益
    • 

  • 找出對應信心水準的分位數
    • 

  • 優點：簡單、不需要假設分配
    • 

  • 缺點：依賴歷史資料、無法預測極端事件
    • 




方法二：變異數-共變異數法



    

  • 假設報酬率服從常態分配
    • 

  • 計算平均報酬和標準差
    • 

  • 使用常態分配分位數推算 VaR
    • 

  • 優點：計算快速
    • 

  • 缺點：假設常態分配，忽略肥尾風險
    • 




方法三：蒙地卡羅模擬



    

  • 設定資產分配參數
    • 

  • 模擬大量可能的價格路徑
    • 

  • 計算每個路徑的損益
    • 

  • 找出對應信心水準的分位數
    • 

  • 優點：可以處理複雜分配
    • 

  • 缺點：計算量大、依賴參數假設
    • 







二、VaR 的優點與限制



優點：



    

  • 易懂：單一數字表示風險，方便溝通
    • 

  • 可比較：不同資產、策略可以用 VaR 比較風險
    • 

  • 整合性：可以整合不同風險來源
    • 

  • 標準化：銀行業廣泛使用，有標準規範
    • 




限制：



    

  • 不告訴極端損失：VaR 只說「最大可能」虧損，不說「如果更糟會怎樣」
    • 

  • 分配假設：假設常態分配可能低估極端事件
    • 

  • 歷史依賴：歷史資料無法預測新的風險
    • 

  • 相關性變化：市場極端時相關性會改變
    • 







三、條件風險價值 CVaR



1. CVaR 是什麼？



    

  • 全名：Conditional VaR（條件風險價值）
    • 

  • 又稱：Expected Shortfall（預期短缺）
    • 

  • 定義：當損失超過 VaR 時，平均會損失多少
    • 




2. CVaR vs VaR



    

  • VaR 告訴你「最大可能」虧損
    • 

  • CVaR 告訴你「如果情況更糟」的平均虧損
    • 

  • CVaR 比 VaR 更保守，更能反映極端風險
    • 




3. 為什麼需要 CVaR？



    

  • VaR 假設損失是常態分配
    • 

  • 但極端事件可能讓損失遠超 VaR
    • 

  • CVaR 考慮了 VaR 之後的風險
    • 







四、其他風險量化指標



1. 標準差（Standard Deviation）



    

  • 衡量資產價格的波動程度
    • 

  • 標準差越大，波動越大
    • 

  • 缺點：上行波動和下行波動一視同仁
    • 




2. Beta 值



    

  • 衡量資產相對於大盤的波動
    • 

  • Beta > 1：資產比大盤波動大
    • 

  • Beta < 1：資產比大盤波動小
    • 

  • 缺點：只考慮相對波動，忽略絕對風險
    • 




3. Alpha 值



    

  • 衡量資產相對於大盤的超額報酬
    • 

  • Alpha > 0：資產表現優於大盤
    • 

  • Alpha < 0：資產表現劣於大盤
    • 




4. 夏普比率（Sharpe Ratio）



    

  • 公式：(報酬率 - 無風險利率) / 標準差
    • 

  • 衡量每單位風險的報酬
    • 

  • 夏普比率越高，風險調整後報酬越好
    • 




5. 索提諾比率（Sortino Ratio）



    

  • 公式：(報酬率 - 無風險利率) / 下行標準差
    • 

  • 只考慮下行風險，更符合投資人關注虧損的心理
    •

一、MDD 最大回撤的定義



1. MDD 是什麼？



    

  • 全名：Maximum Drawdown（最大回撤）
    • 

  • 定義：在特定時間內，投資組合從歷史最高點跌至最低點的最大跌幅
    • 

  • 意義：直觀反映策略在最糟糕情況下可能承受的損失程度
    • 




2. MDD 計算公式



步驟一：計算回撤（Drawdown）



回撤 = (投資組合最高價值 - 最高後的最低價值) / 投資組合最高價值 × 100%



步驟二：找出最大回撤



MDD = MAX(特定時間內所有回撤)



3. 實際計算範例



情境：



    

  • 起始資金：5 萬美元
    • 

  • 漲到高點：10 萬美元
    • 

  • 跌到谷底：7 萬美元
    • 

  • 回升高點：12 萬美元
    • 

  • 再跌到：9 萬美元
    • 




計算：



    

  • 第一次回撤：(10 - 7) / 10 = 30%
    • 

  • 第二次回撤：(12 - 9) / 12 = 25%
    • 

  • MDD = MAX(30%, 25%) = 30%

核心內容：



一、什麼是交易系統？



1. 定義



    

  • 交易系統：一套明確定義的規則，告訴交易者何時進場、何時出場、何時停損停利
    • 

  • 核心目的：消除主觀判斷，讓交易可以重複執行
    • 

  • 關鍵特徵：可定義、可回測、可執行
    • 




2. 交易系統的組成要素



要素說明是否必須
**進場規則**何時買進必須
**出場規則**何時賣出必須
**停損規則**虧損多少出場必須
**停利規則**獲利多少出場建議
**倉位管理**每次投入多少資金必須
**風險管理**單筆最大虧損、總體 MDD 上限必須



3. 為什麼需要交易系統？



    

  • 消除情緒：依照規則執行，不受恐懼貪婪影響
    • 

  • 可重複執行：每次條件相同，操作相同
    • 

  • 可回測驗證：可以用歷史數據驗證有效性
    • 

  • 可持續改進：知道哪裡出問題才能修正
    • 







二、系統交易 vs 主觀交易



比較面向系統交易主觀交易
**決策依據**明確規則經驗判斷
**情緒影響**低高
**執行一致性**高低
**回測驗證**可以困難
**適應性**低，需定期更新高，可即時調整
**學習門檻**較低較高
**時間投入**初期高，後期低持續高
**適合對象**新手、有紀律者經驗豐富、直覺強者



1. 系統交易的優點



    

  • 100% 照規則執行，不受情緒影響
    • 

  • 可以回測驗證策略有效性
    • 

  • 紀律容易維持
    • 

  • 適合新手快速建立框架
    • 




2. 系統交易的缺點



    

  • 缺乏彈性，無法應對極端情況
    • 

  • 市場變化時需要更新規則
    • 

  • 可能被有心人利用規則
    • 




3. 主觀交易的優點



    

  • 可以即時應對市場變化
    • 

  • 利用自己的經驗和直覺
    • 

  • 靈活度高
    • 




4. 主觀交易的缺點



    

  • 情緒影響大，難以保持紀律
    • 

  • 經驗難以傳承
    • 

  • 無法回測驗證
    •

    •

    • 

  • 長期投資：持有數月到數年
    • 




步驟二：定義進場規則



    

  • 技術指標：如均線交叉、MACD、RSI
    • 

  • 價格型態：如突破、拉回買進
    • 

  • 基本面條件：如財報數據、產業趨勢
    •

    • 




步驟三：定義出場規則



    

  • 停損：技術支撐跌破、固定比例虧損
    • 

  • 停利：技術壓力突破、固定比例獲利
    • 

  • 移動停利：根據價格移動調整
    • 

  • 時間停損：持有 N 天未達目標自動出場
    • 




步驟四：定義倉位管理



    

  • 單筆風險：不超過帳戶 1-2%
    •

    • 

  • 分批進場：根據訊號強度調整倉位
    • 




步驟五：回測驗證

    •

    •

    •

    •

    •

    •

    •

    • 

  • 最大回撤在可接受範圍
    • 




原則四：適合自己



    

  • 考慮時間可以投入多少
    •

    • 

  • 考慮個性特質

  • 特徵：大漲大跌，波動劇烈
    • 

  • 含義：風險過高，可能爆倉
    • 

  • 問題：心臟受不了，容易在高點低點做錯決策

  • 特徵：上上下下，沒有明顯方向
    • 

  • 含義：策略期望值為零或接近零
    • 

  • 問題：白做工，交易成本還賠錢
    • 







二、資金曲線管理的核心原則



原則一：穩定勝過暴利



    

  • 年報酬 20% 優於大賺大賠
    • 

  • 長期複利的力量
    • 

  • 10 年 20% 報酬 = 6.19 倍
    • 




原則二：控制回撤



    

  • 回撤 20% 需要 25% 漲幅恢復
    • 

  • 回撤 50% 需要 100% 漲幅恢復
    • 

  • 控制回撤比追求報酬更重要
    • 




原則三：資金來源管理



    

  • 自有資金：壓力可控
    • 

  • 親友資金：壓力來自他人期望
    • 

  • 借貸資金：壓力最大，利息成本高
    • 




原則四：資金效率三要素



    

  • 獲利能力：策略的期望值
    • 

  • 風險控制：最大回撤、波動度
    • 

  • 時間效率：持有時間、機會成本
    •

    •

核心內容：



一、VaR（風險價值）基礎概念



1. VaR 是什麼？



    

  • 全名：Value at Risk（風險價值）
    • 

  • 定義：資產組合在持有期間內，在給定的信賴區間內，由於市場價格變動所導致的最大預期損失的數值
    • 

  • 核心功能：將所有風險彙整成一個數值，讓管理階層能簡單明瞭地知道資產組合可能面臨之最大損失
    • 

  • 直觀解讀：「有 95% 的信心在未來 N 天內，最大損失不會超過 V 元」
    • 




2. VaR 的應用場景



    

  • 金融機構資本適足率計算
    • 

  • 企業內部風險控管
    • 

  • 資產配置的參考依據
    • 

  • 投資組合風險量化
    • 




3. VaR 的歷史背景



    

  • 1980年代末至1990年代初，美國銀行業遭受商業風險困擾
    • 

  • JP 摩根公司發明 VaR 方法
    • 

  • 1996年巴塞爾修正協定將 VaR 作為市場風險衡量指標
    • 

  • 芝加哥商品交易所於1988年推出 SPAN 系統，成為計算投資組合保證金的全球行業標準
    • 




4. VaR 與傳統風險測度的差異



    

  • 傳統風險測度：以標準差為代表，關注整體波動
    • 

  • VaR 優勢：著重資產下行風險，特別是極端情況下的損失規模
    • 







二、VaR 的三種計算方法



方法一：變異數-共變異數法（Delta Normal）



原理：



    

  • 假設風險因子為常態分配
    • 

  • 報酬率之對數為常態分配
    • 

  • 常態分配的線性組合仍為常態分配
    • 




計算步驟：

1. 計算股價報酬率（連續型報酬）

2. 計算投資組合變異數矩陣

3. 套用公式計算 VaR



股票投資組合 VaR 公式：



VaR = Z × σ × √T × V



    

  • Z：信賴水準對應的 Z 值（95% = 1.645，99% = 2.326）
    • 

  • σ：投資組合標準差
    • 

  • T：持有期間（天）
    • 

  • V：投資組合價值
    • 




優點：



    

  • 計算效率高
    • 

  • 風險因子數量多且相互獨立時適用
    • 

  • 易於處理增額風險值（Marginal VaR）
    • 




缺點：



    

  • 無法妥善處理厚尾問題（極端事件）
    • 

  • 需要估計波動度及報酬相關係數
    • 

  • 不適合處理選擇權等非線性資產
    • 







方法二：歷史模擬法



原理：



    

  • 利用實際歷史資料求算資產組合風險值
    • 

  • 假設未來資產報酬走勢與過去大致相同
    • 

  • 無需對報酬分配做任何假設
    • 




計算步驟：

1. 收集持有一段時間的歷史價格資料（如過去100天）

2. 計算每種資產的損益報酬率

3. 利用歷史報酬率模擬未來可能報酬

4. 將投資組合可能報酬由小至大排序

5. 依信賴水準找出對應百分位數



實例：



    

  • 100 筆損益資料
    • 

  • 信賴水準 95%
    • 

  • 第 5 筆資料（100 × 5%）即為風險值
    • 




優點：



    

  • 不需對風險因子作任何假設
    • 

  • 不需估計波動度與相關係數
    • 

  • 可考量厚尾分配及極端事件
    • 




缺點：



    

  • 完全仰賴特定歷史資料集
    • 

  • 極端事件可能扭曲估計結果
    • 

  • 未考量市場結構改變
    • 

  • 資料期過短可能導致偏差
    • 







方法三：蒙地卡羅模擬法



原理：



    

  • 假設資產價格變化符合特定隨機過程
    • 

  • 股價常用幾何布朗運動（Geometric Brownian Motion）
    • 

  • 重複模擬產生足夠情境
    • 




隨機過程公式：



dS/S = μdt + σdW



    

  • S：股價
    • 

  • μ：預期報酬率
    • 

  • σ：波動度
    • 

  • dW：布朗運動
    • 




計算步驟：

1. 建立價格走勢隨機過程，估計參數（平均數、波動度、相關係數）

2. 從標準常態分配隨機抽取 ε，產生下一期股價

3. 重複 N 次（如 1000 次），產生投資組合報酬分配

4. 依信賴水準找出對應百分位數



優點：



    

  • 可以包含任何風險因子分配
    • 

  • 將任何複雜投資組合模組化
    • 

  • 可執行壓力測試與敏感度分析
    • 

  • 模型設定具彈性
    • 




缺點：



    

  • 隨機過程設定錯誤會產生模型風險
    • 

  • 電腦資源需求高
    • 

  • 極端值未能包含於分配中
    •

四、VaR 的限制與補強



VaR 的限制：

1. 未考量超門限損失：VaR 只告訴你最大預期損失，不告訴你超過這個損失會怎樣

2. 不滿足次可加性：不是一致風險測度，可能出現投資組合 VaR 大於個別資產 VaR 之和

3. 模型風險：參數估計誤差會影響結果

4. 常態分配假設：實際市場報酬有厚尾現象



補強方法：



1. 壓力測試（Stress Testing）



    

  • 分析市場出現異於常態的極端情況
    • 

  • 模擬歷史重大事件（如金融海嘯）的衝擊
    • 

  • 評估投資組合在極端壓力下的表現
    • 




2. 情境分析（Scenario Analysis）



    

  • 設計特定市場情境
    • 

  • 評估不同壓力情境下的潛在損失
    • 




3. 回溯測試（Backtesting）



    

  • 驗證 VaR 模型的準確性
    • 

  • 比較每日真實報酬與 VaR 預測
    • 

  • 調整模型參數
    • 







五、最大回撤（MDD）定義與計算



1. MDD 是什麼？



    

  • 全名：Maximum Drawdown（最大回撤）
    • 

  • 定義：在特定期間內，資產淨值從歷史最高點跌至最低點的最大跌幅
    • 

  • 直觀理解：投資過程中可能出現的最大資金虧損比例
    • 




2. MDD 計算公式



    

  • 步驟一：計算所有回檔
    • 

  • 步驟二：找出所有回檔中最大的數值
    • 




3. 實例說明



    

  • 帳戶起始資金：5 萬美元
    • 

  • 漲到高點：10 萬美元
    • 

  • 跌至谷底：7 萬美元
    • 

  • 再漲至新高：12 萬美元
    • 

  • 再跌至：9 萬美元
    • 




計算：



    

  • 第一次回撤：(10 - 7) ÷ 10 = 30%
    • 

  • 第二次回撤：(12 - 9) ÷ 12 = 25%
    • 

  • MDD = MAX(30%, 25%) = 30%
    •

九、使用 MDD 的四個重要注意事項



1. MDD 與未來無關



    

  • MDD 只能根據歷史得出過去最大回撤
    • 

  • 未來 MDD 可能比歷史更大
    • 

  • 不能用歷史 MDD 推估未來
    • 

  • 例：2022年前長期公債 MDD 僅 -23.1%，2022年突破至 -45.3%
    • 




2. MDD 可能來自極端市場狀況



    

  • 某些 MDD 不是常態，而是特定條件
    • 

  • 例：黃金 1980-2000 年的 MDD 來自金本位結束 + 石油危機
    • 

  • 這些事件在未來很難複製
    • 




3. 管理 MDD 的關鍵是控制部位大小



    

  • 影響 MDD 的要素：持有時間、交易頻率、部位大小、市場波動
    • 

  • 無法控制所有要素
    • 

  • 有效方法：控制部位大小
    • 

  • 透過資產配置控制風險
    • 




4. MDD 不一定是單筆交易造成



    

  • 迷思：設定停損就能改善 MDD
    • 

  • 事實：MDD 常是連續虧損造成，非單筆交易
    • 

  • 停損設得越緊，越容易觸發，虧損越容易發生
    • 

  • 任何策略只要執行夠久，都有機會遇到更大的 MDD
    •

資金來源三類型：



類型說明風險

資金配置策略：



    

  • 初期靜態配置：資金較少時，預留閒置資金後全數投入

  • 動態配置：資金變多後分散至貨幣市場、債券、股市
    • 




範例配置：



    

  • 10%：當沖日結損益
    • 

  • 20%：短期投資（5 分 K）
    • 

  • 30%：中期投資（小時/日 K）
    • 

  • 40%：長期投資（基本面分析）

資金效率三要素：



要素定義關鍵
**充分性**資金使用程度避免閒置過多或槓桿過高
**有效性**投入是否有效益高獲利機率、高獲利率
**流動性**資金週轉效率週轉越快，資金累積越快






二、系統性 vs 非系統性風險



系統性風險（市場風險）：



    

  • 定義：整個市場都會出現的風險
    • 

  • 特徵：無法透過分散投資消除
    • 

  • 來源：金融風暴、經濟崩盤、天災、戰爭、政治
    • 

  • 應對：分散資產（股票+債券+黃金+REITs 等）
    • 




非系統性風險（個股風險）：



    

  • 定義：單一股票/企業特有的風險
    • 

  • 特徵：可透過分散投資消除
    • 

  • 來源：財報不佳、高層舞弊、競爭對手出現
    • 

  • 應對：投資多檔股票
    • 




Beta 值（衡量系統性風險）：



    

  • β > 1：波動比市場大（高風險高報酬）
    • 

  • β < 1：波動比市場小（保守穩健）
    • 

  • β = 1：與市場同步
    • 




Alpha 值（衡量非系統性風險）：



    

  • α 高：扣除系統性風險後可獲得超額報酬
    • 

  • 與公司基本面、營運狀況、管理能力相關
    • 







三、尾端風險（Tail Risk）



定義：



    

  • 發生機率極低，但一旦發生衝擊巨大的極端事件
    • 

  • 投資收益偏離均值超過 3 個標準偏差
    • 

  • 「黑天鵝中的黑天鵝」
    • 




風險測度方法：

1. VaR（風險價值）：給定信賴區間內最大預期損失

2. ES（期望損失）：超過 VaR 後的預期損失

3. TCE（尾部條件期望）：考慮尾部分佈的風險



應對策略：



    

  • 保留現金（逢低買進）
    • 

  • 分散資產配置
    • 

  • 尾部避險策略（保護性認沽期權）
    • 

  • 降低槓桿
    •

斷頭定義：



    

  • 保證金帳戶低於維持保證金

三、交易策略：三大量化要素

要素說明計算方式

期望值公式：

一、投資組合 VaR 計算實務



VaR 計算三種方法比較



方法分配假設非線性資產厚尾問題優點缺點
**變異數-共變異數法**常態分配不適合無法處理計算快、易解釋不能處理非線性
**歷史模擬法**各種分配可處理可處理無須假設分配需大量歷史資料
**蒙地卡羅模擬法**各種分配可處理可處理彈性高、可壓力測試計算複雜、模型風險



投資組合 VaR 計算步驟



步驟一：計算個別標的 DEAR（每日盈餘風險）



DEAR = |-α| × σ × 價值
α = 信賴水準對應 z 值（99% → 2.33）
σ = 報酬率標準差



步驟二：計算標的間相關係數



    

  • 使用報酬率資料，而非價格資料
    • 

  • 避免假性迴歸問題
    • 




步驟三：計算投資組合 VaR



組合 VaR = √(Σ個別 VaR² + 2×Σ(相關係數× VaR_i × VaR_j))



實例：四檔股票投資組合（儒鴻、台積電、國泰金、長榮）



    

  • 99% 絕對 VaR = $50,647.78
    • 

  • 99% 相對 VaR = $52,205.86
    • 

  • 意義：單一交易日最大損失有 99% 機率不超過 5.2 萬元
    • 




絕對 VaR vs 相對 VaR



類型公式說明
**絕對 VaR**(\-α\× σ – 均值) × 價值相對於 0 的風險值
**相對 VaR**\−α\× σ × 價值相對於報酬率均值的風險值



變異數-共變異數法的限制



1. 無法衡量非線性資產



    

  • 相關係數假設線性關係
    • 




2. 忽略厚尾問題



    

  • 假設常態分配
    • 




解決方法：



    

  • 使用歷史模擬法或蒙地卡羅模擬
    • 

  • 採用 CVaR（條件風險價值）補充
    • 







二、CVaR（條件風險價值）進階概念



CVaR 定義



    

  • CVaR = Conditional Value at Risk = 期望尾端損失（Expected Shortfall）
    • 

  • 核心問題：當損失超過 VaR 時，預期會虧多少？
    • 

  • CVaR 計算 VaR 點以下的平均損失
    • 




VaR vs CVaR 比較



特性VaRCVaR
**定義**特定信賴水準下的最大損失超過 VaR 後的平均損失
**關注點**情況會變多糟糕？最糟糕發生時預期虧多少？
**次可加性**❌ 不滿足✅ 滿足
**尾部資訊**不考量充分考量
**一致性風險測度**否是



為什麼 CVaR 更好？



VaR 的缺陷：



    

  • 不考慮超過閾值後的損失嚴重度
    • 

  • 不滿足次可加性（組合 VaR 可能大於個別 VaR 之和）
    • 

  • 無法正確反映分散風險的好處
    • 




CVaR 的優勢：



    

  • 考慮尾端損失的嚴重度
    • 

  • 滿足次可加性，符合一致性風險測度
    • 

  • 對估計誤差更敏感
    • 

  • 更適合風險管理和資本配置
    • 




CVaR 公式概念



直觀理解：



    

  • CVaR 把 VaR 以下的損失全部加權平均
    • 

  • 正態分配下：(CVaR - VaR) / VaR ≈ 15%（99% 信賴水準）
    • 

  • 峰度越高，差距越大
    • 




計算方法：



    

  • 把尾端分成 n 等分
    • 

  • 計算對應 n-1 分的 VaR
    • 

  • 求期望值

四、實戰應用步驟



投資組合 VaR 計算流程



1. 蒐集資料：各標的歷史價格、日報酬率、標準差和相關係數

2. 設定參數：信賴水準（95%、99%）、持有期間（1天、10天）、計算方法

3. 計算個別 DEAR：DEAR_i = 2.33 × σ_i × 價值_i

4. 計算組合 VaR：組合 VaR = √(Σ DEAR_i² + 2×Σ(ρ_ij × DEAR_i × DEAR_j))

5. 檢核厚尾問題：繪製報酬率直方圖、檢視高峰度資產



風險限額設定



1. 設定 VaR 限額：單一部位 VaR 限制、總組合 VaR 限制、日 VaR 不超過資金 X%

2. 停損觸發機制：VaR 估算虧損達到設定門檻 → 自動減倉或平倉






五、關鍵心得



1. VaR 是起點不是終點：VaR 方便但有其限制，應搭配 CVaR、壓力測試使用

2. 厚尾很重要：台股常有肥尾現象，變異數-共變異數法可能低估風險

3. 組合風險計算需要相關係數：分散效果看相關性，不要只看個別風險

4. CVaR 比 VaR 更適合風險配置：滿足一致性風險測度，是更好的風險指標

5. 定期回測驗證模型：歷史資料可能失效，定期檢視模型準確度
