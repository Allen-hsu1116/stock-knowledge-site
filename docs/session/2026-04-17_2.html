倖存者偏差（Survivorship Bias）

是什麼：只看活下來的、成功的案例，忽略死掉的、失敗的案例，導致對整體情況的誤判。二戰時 Wald 教授發現——軍方想加強返航飛機中彈最多的部位，但他指出：該加強的是那些「沒回來的」飛機被擊中的部位。

怎麼做：
1. 自我覺察：當你聽到「少年股神」「航海王」的故事時，先問：那些跟著做但失敗的人呢？他們發生什麼事？
2. 研究失敗案例：想贏，先學怎麼不輸。失敗案例裡有比成功案例更多的實用教訓。成功看起來都「水到渠成」，但失敗才知道哪裡會死人
3. 找否證資訊：刻意找跟自己立場相反的資料，不要只看支持自己觀點的數據（這也是確認偏誤的剋星）
4. 站在對手角度想：你在這裡買進，為什麼有人願意在這裡賣出？如果答案是想不清楚，你可能就是那個被收割的

實例：

  • 2021年航海王、鋼鐵人熱潮：大家只看到曬對帳單的贏家，沒看到無數追高套牢的散戶
  • TEJ回測實證：生技股2020-2022均線策略，包含下市公司的ROI是-14%，剔除下市公司後變成較好的數字。光是把失敗者拿掉，績效就變好了——這就是倖存者偏差在回測中的陷阱
  • 創業數據：第一年倒閉率90%，五年內99%陣亡。但你只看到那1%的成功故事，就以為「我也可以」

適用時機：每次你覺得「這策略很穩」「這方法一定行」的時候。尤其在回測選股時，如果只用現在還活著的股票回測，績效會被高估。要納入已下市公司的數據才算完整。

倖存者偏差對交易決策的三大影響

是什麼：倖存者偏差不是抽象概念，它具體影響你的策略研究、心理狀態、風險管理三個層面。

怎麼做：
1. 策略研究影響：

  • 只看過去有效的策略 → 忽略失效的策略 → 市場變了還在用老招

2. 心理狀態影響：

  • 只看成功交易 → 高估自己能力 → 過度自信 → 使用超過負荷的槓桿

3. 風險管理影響：

  • 只看存活交易者的策略 → 低估失敗機率 → 部位過大

實例：

  • 回測生技股均線策略：包含9家下市公司時ROI=-14%，剔除後數字好看很多。那9家就是「沒回來的飛機」——忽略它們，你的模型就是在騙自己
  • 基金績效排名：只看目前還存在的基金，忽略已清算的爛基金，排名永遠比實際好看

適用時機：做任何回測、研究任何策略、評估任何「成功方法」之前，先問：「那些失敗的在哪裡？」

漲停跌停流動性風險（Liquidity Risk at Limit Price）

是什麼：漲停鎖死時你買不到，跌停鎖死時你賣不掉。這就是流動性風險——你明明願意交易，但市場上沒有對手方。散戶最容易忽略的風險不是價格波動，而是「想跑跑不掉」。

怎麼做：
1. 漲停買進的現實：

  • 漲停時所有買單掛同一價格，成交全靠「時間優先」

2. 跌停賣出的困境：

  • 所有人都想逃，沒有人要接，你掛跌停賣單排在幾萬張後面

3. 追漲停三大陷阱：

  • 隔日開高走低：今天鎖死吸引散戶追價，隔天主力倒貨，追高的直接套牢

4. 判斷漲停品質：

  • 一字漲停鎖死到底 > 盤中多次打開又鎖上

實例：

  • 某股票昨日收盤100元，今日漲停價110元。散戶9:00掛漲停買單，但前面排了5萬張，全天只成交2000張——根本買不到
  • 更慘的情況：運氣好買到了，隔天開高走低殺到95元，你虧了14%（110→95），而且跌停鎖死想停損都賣不掉
  • 新股前5天沒有漲跌幅限制，一天可以漲跌超過10%，流動性風險更大

適用時機：每次你想「追漲停」或「跌停逃命」時。記住：漲停時你是買方的最末端，跌停時你是賣方的最末端。流動性風險比價格風險更致命，因為你連出場的機會都沒有。

心得

今天學的三個主題，串起來就是一句話：你看不見的風險，才是最致命的。

倖存者偏差是認知層面的盲點。你以為某個策略有效，可能是因為你只看到了成功的案例。TEJ的回測實證很紮實——把下市公司拿掉，績效就變好看。這不是數字遊戲，這是你腦子在騙你。每次做回測、看績效、聽故事，都要問：「那些失敗的在哪？」這個問題可能比任何技術指標都重要。

倖存者偏差對交易的影響最讓我警惕的是「當所有人都在用成功策略，策略本身就失效」。市場是零和賽局，一個方法如果被太多人知道了，它就不再是優勢，而是陷阱。所以光看成功案例不夠，還要理解為什麼失敗的案例會失敗——那些失敗裡有比成功更多的情報。

漲停跌停流動性風險是實戰層面的「看不見的風險」。散戶最常犯的錯不是看錯方向，而是看對方向卻跑不掉。漲停追進去，隔天跌停跌停鎖死，虧20%連停損的機會都沒有。這不是價格風險，是流動性風險——你願意賣，但沒有人願意買。解法不是「不要追漲停」這麼簡單，而是要在進場前就想好：如果隔天跌停鎖死，我受得了嗎？如果受不了，就不要追。

三個主題的共同核心：風險管理不是管理你知道的風險，而是管理你看不到的風險。 倖存者偏差讓你看不到失敗，流動性風險讓你出不了場。兩者都是「看不見的」，但兩者都會殺死你。

VaR 風險值（Value at Risk）實戰全解析

一、VaR 是什麼？

一句話：在正常市場條件下，某個投資組合在特定期間內、特定信心水準下，預期的最大可能損失金額。

白話文：VaR 回答的問題是 — 「我有 95% 的把握，明天最多只會虧 X 元。」

公式表示：
> P(ΔP ≤ VaR) = 1 - α

其中：

  • ΔP = 損失金額
  • VaR = 在信心水準 α 下的最大可能損失
  • α = 信心水準（常見 95%、99%）

實例：

  • 你持有 100 萬的股票組合
  • 日 VaR（95%信心水準）= 5 萬元
  • 意思：明天有 95% 的機率，虧損不會超過 5 萬元
  • 也就是說，20 個交易日中大約只有 1 天會虧超過 5 萬

二、VaR 三大參數

1. 持有期間（Holding Period）

決定你要算「多久時間內」的風險。

持有期間適用情境
1 天當沖、短線交易、流動性高的部位
10 天巴塞爾協定要求、中線部位
1 個月波段操作、投資組合
1 年長期投資、退休基金

怎麼選：

  • 交易越頻繁 → 持有期間越短
  • 巴塞爾委員會要求銀行用 10 個營業日
  • 散戶短線交易建議用 1 天

2. 信心水準（Confidence Level）

信心水準越高 → VaR 數值越大 → 保守程度越高。

信心水準使用者含義
95%J.P. Morgan、美洲銀行20 天中有 1 天會超過
95.4%花旗銀行2σ 常態分配
97.5%大通曼哈頓銀行較保守
99%巴塞爾委員會、Bankers Trust100 天中有 1 天會超過

散戶實戰建議：

  • 日常風控用 95%（平衡靈敏度和保守度）
  • 極端風險評估用 99%（抓最壞情況）

3. 觀察期間（Observation Period）

計算歷史波動度所用的數據期間。

  • 太短 → 樣本不足，估計不穩
  • 太長 → 市場結構已改變，歷史數據失真
  • 巴塞爾委員會要求 1 年
  • 散戶建議用 250 個交易日（約 1 年）

三、VaR 三種計算方法

方法一：變異數-共變異數法（Delta-Normal 法）

核心邏輯：假設報酬率服從常態分配，用平均數和標準差直接計算。

計算步驟：
1. 收集歷史日報酬率
2. 計算平均報酬率 μ 和標準差 σ
3. VaR = 投資金額 × (μ - z × σ) × √持有天數

  • 95% 信心水準：z = 1.645

實例：

  • 持有台積電股票 50 萬
  • 過去 250 天日報酬率標準差 σ = 2%
  • 日均報酬率 μ = 0.05%
  • 95% VaR = 500,000 × (0.05% - 1.645 × 2%) = 500,000 × (-3.24%) = -16,200 元
  • 也就是說，95% 的機率明天最多虧 16,200 元

優點：

  • 計算快速簡單
  • 可處理多個風險因子
  • 容易做增額風險值分析

缺點：

  • ❌ 無法處理「厚尾」問題（極端事件發生頻率比常態分配預測的高）
  • ❌ 需要估計波動度和相關係數
  • ❌ 對選擇權等非線性商品不適用

方法二：歷史模擬法

核心邏輯：用歷史真實數據排序，直接看第 5% 或第 1% 分位數的損失。

計算步驟：
1. 收集過去 N 天的日報酬率（建議 250 天以上）
2. 將報酬率從小到大排序
3. 取第 5% 分位數（95% 信心水準）或第 1% 分位數（99% 信心水準）
4. VaR = 投資金額 × 該分位數的報酬率

實例：

  • 持有 100 萬投資組合
  • 過去 250 天報酬率排序後，第 5% 分位數（第 12.5 筆）的報酬率 = -3.5%
  • 95% 日 VaR = 1,000,000 × 3.5% = 35,000 元

優點：

  • ✅ 不需要假設分配型態（非參數法）
  • ✅ 自動捕捉厚尾和偏態
  • ✅ 不需要估計參數

缺點：

  • ❌ 完全依賴歷史數據，極端事件可能扭曲結果
  • ❌ 無法反映市場結構變化
  • ❌ 數據期間太短會有偏差

方法三：蒙地卡羅模擬法

核心邏輯：假設股價服從某種隨機過程（如幾何布朗運動），用電腦模擬數千到數萬次可能的路徑，再統計損失分配。

計算步驟：
1. 設定隨機過程模型（如 GBM：dS = μSdt + σSdW）
2. 估計參數 μ（飄移率）和 σ（波動率）
3. 模擬 10,000+ 次價格路徑
4. 計算每條路徑的損益
5. 排序後取對應分位數

實例：

  • 持有期貨部位 200 萬
  • 模擬 10,000 次明天的可能損益
  • 95% 分位數對應虧損 = 8 萬
  • 99% 分位數對應虧損 = 15 萬
  • 95% 日 VaR = 8 萬，99% 日 VaR = 15 萬

優點：

  • ✅ 可處理任何分配和複雜組合
  • ✅ 彈性最高，可加入各種假設
  • ✅ 適合選擇權等非線性商品

缺點：

  • ❌ 計算資源需求大
  • ❌ 極端值未必包含在模擬中
  • ❌ 模型假設錯誤會導致結果偏差

五、VaR 實戰應用場景

場景一：單一持股風險評估

情境：你持有 80 萬的台積電，想知道一天的風險。

用變異數法：

  • 台積電日均波動度 σ ≈ 2.5%（近一年數據）
  • 95% 日 VaR = 800,000 × 1.645 × 2.5% = 32,900 元
  • 解讀：明天有 95% 的機率虧損不超過 32,900 元

決策：

  • 如果 32,900 超過你能承受的單日最大虧損 → 部位太大，減碼
  • 如果你的 1% 法則是總資金 100 萬最多虧 1 萬 → VaR 32,900 遠超 1% → 必須減碼到約 24 萬

場景二：投資組合風險控管

情境：你持有三檔股票，總市值 150 萬。

步驟：
1. 計算各股權重：w₁, w₂, w₃
2. 計算各股標準差：σ₁, σ₂, σ₃
3. 計算相關係數矩陣：ρ₁₂, ρ₁₃, ρ₂₃
4. 組合標準差 σp = √(w'Σw)
5. VaR = 組合市值 × 1.645 × σp

重點：分散投資降低的是相關性風險，相關係數越低，組合 VaR 越小。

場景三：設定停損參考

做法：

  • 日 VaR（95%）= 帳面最大可接受日虧損
  • 將 VaR 作為停損的量化依據，而不是憑感覺設停損
  • 例：VaR = 32,900 → 停損設在 -3.5%（略寬於 VaR）

六、VaR 的限制與補強

VaR 三大限制

1. 不告訴你超過 VaR 會虧多少

  • VaR 只說「95% 機率不超過 X」

2. 厚尾低估風險

  • 金融市場報酬率不是常態分配，極端事件比常態分配預測的更頻繁

3. 不適合做跨期間加總

  • 日 VaR × √N ≠ N 日 VaR（因為波動度聚集效應）

CVaR（條件風險值 / Expected Shortfall）

定義：超過 VaR 的平均損失。

公式：CVaR = E[Loss | Loss > VaR]

實例：

  • 95% VaR = 32,900 元
  • 超過 VaR 的那 5% 情境中，平均虧損 = 65,000 元
  • CVaR = 65,000 元

為什麼重要：

  • VaR 告訴你「通常最多虧多少」
  • CVaR 告訴你「真的出事時平均會虧多少」
  • 兩者搭配使用才是完整的風險評估

壓力測試與情境分析

VaR 只看正常情況，壓力測試看極端情況：

方法情境目的
歷史壓力測試2008 金融海嘯、2020 COVID看類似歷史事件下會虧多少
假設情境分析利率+3%、台股跌20%看特定假設下的損失
反向壓力測試往回推算什麼情境會讓你破產找出最脆弱的地方

七、散戶 VaR 實戰 SOP

最簡版本：歷史模擬法（Excel 可做）

1. 下載過去 250 天收盤價
2. 計算日報酬率 =（今日收盤 - 昨日收盤）/ 昨日收盤
3. 排序所有日報酬率（從小到大）
4. 取第 12 或 13 個值（250 × 5% ≈ 12.5）→ 這就是 95% VaR 對應的報酬率
5. VaR 金額 = 持股市值 × |該報酬率|

進階版本：變異數法（搭配 ATR）

1. 用 ATR（14）當作日波動度的代理指標
2. VaR ≈ 持股市值 × 1.645 × (ATR / 收盤價)
3. 快速估算，不用算歷史數據

每日風控 SOP

1. 開盤前：算出今日 VaR，確認最大可接受虧損
2. 盤中：如果虧損接近 VaR → 準備執行停損
3. 收盤後：更新波動度估計，重新計算 VaR
4. 每週：回溯測試，比較實際虧損和 VaR 預測是否吻合

心得

VaR 是金融機構的風險管理標準工具，但散戶幾乎沒人在用，這就是專業和業餘的差距。最簡單的歷史模擬法，用 Excel 就能算出來，告訴你「明天有 95% 的機率最多虧多少」，這比任何憑感覺設的停損都科學。

三種方法中，歷史模擬法最適合散戶：不需要假設分配、不用寫程式、Excel 排序就能做。缺點是它只看過去，不能預測前所未有的黑天鵝，所以一定要搭配 CVaR 和壓力測試。

最實用的收穫：把 VaR 當作停損的量化依據。不要憑感覺設 -5% 或 -7%，而是用數據告訴你：在 95% 信心水準下，這檔股票一天的正常波動範圍是多少。超過這個範圍，才需要警覺。這就是科學化風險管理的起點。

VaR 回測驗證（Backtesting VaR）實戰全解析

一、VaR 回測是什麼？

一句話：把 VaR 模型預測的損失跟實際損失做比較，驗證模型到底準不準。VaR 算出來的數字不是拿來看的，是要拿來驗的。

白話文：你算出「95% 機率最多虧 5 萬」，那到底是不是真的只有 5% 的日子虧超過 5 萬？回測就是在回答這個問題。

核心概念——例外（Exception）：

  • 每當實際虧損超過 VaR 預測值，就記錄一次「例外」
  • 95% 信心水準下，例外應該佔 5%
  • 99% 信心水準下，例外應該佔 1%
  • 例外太多 = 模型低估風險（危險！）
  • 例外太少 = 模型過度保守（浪費資金效率）

實例：

  • 用 99% 信心水準的 VaR，回測 250 天
  • 理論上例外次數 = 250 × 1% = 2.5 次
  • 如果實際出現了 8 次例外 → 模型嚴重低估風險
  • 如果實際出現了 0 次例外 → 模型可能太保守

二、例外次數的統計判斷

二項分佈模型

例外次數遵循二項分佈 B(n, p)，其中 n 是樣本天數，p 是例外機率（= 1 - 信心水準）。

公式：
> P(X = x) = C(n, x) × p^x × (1-p)^(n-x)

期望值：E(X) = n × p
變異數：Var(X) = p × (1-p) × n

實例：

  • 250 天回測，99% 信心水準（p = 1%）
  • 期望例外次數 = 250 × 1% = 2.5 次
  • P(X = 0) = C(250,0) × 0.01^0 × 0.99^250 = 8.1%
  • P(X = 5) = C(250,5) × 0.01^5 × 0.99^245 = 6.7%
  • 也就是說，即使模型完全正確，0 次例外也有 8.1% 的機率發生——這不是「太好」，是統計波動

怎麼做：
1. 收集回測期間的 VaR 預測值和實際損益
2. 計算例外次數（實際虧損 > VaR 的天數）
3. 用 z 統計量判斷模型是否校準正確
4. z = (x - pT) / √(p(1-p)T)，其中 x 是例外次數，T 是樣本天數

z 統計量判斷：

  • |z| < 1.96 → 95% 信心下不拒絕模型（模型可接受）
  • |z| ≥ 1.96 → 95% 信心下拒絕模型（模型有問題）
  • |z| ≥ 2.576 → 99% 信心下拒絕模型（模型嚴重有問題）

實例：

  • 252 天回測，95% 信心水準 VaR
  • 期望例外 = 252 × 5% = 12.6 次
  • 實際例外 25 次
  • z = (25 - 12.6) / √(0.05 × 0.95 × 252) = 12.4 / √11.97 = 3.58
  • z = 3.58 > 2.576 → 即使在 99% 信心下也拒絕模型
  • 結論：這個 VaR 模型嚴重低估風險

三、巴塞爾紅綠燈測試（Traffic Light Test）

是什麼：巴塞爾銀行監理委員會設計的三區域判斷系統，用來決定 VaR 模型是否可接受。基於 99% 信心水準的日 VaR，回測期間 250 天。

三個區域：

🟢 綠燈區（Green Zone）：

  • 例外次數 0-4 次
  • 模型可接受，不加資本乘數
  • 資本乘數 = +0（維持最低 3 倍）

🟡 黃燈區（Yellow Zone）：

  • 例外次數 5-9 次
  • 模型有問題但未嚴重失效
  • 資本乘數 +0.4 到 +0.85（遞增）
  • 需要調查原因並改善

🔴 紅燈區（Red Zone）：

  • 例外次數 ≥ 10 次
  • 模型嚴重失效
  • 資本乘數 +1.0（總共 4 倍）
  • 必須立即修正模型

怎麼做：
1. 使用 99% 信心水準的日 VaR
2. 回測過去 250 個交易日
3. 計算例外次數
4. 對照紅綠燈區域判斷
5. 黃燈或紅燈 → 調查原因，可能需要更換模型或調整參數

實例：

  • 你的 VaR 模型（99% 信心），回測 250 天
  • 例外 3 次 → 🟢 綠燈，模型 OK
  • 例外 7 次 → 🟡 黃燈，資本乘數 +0.65，需要調查
  • 例外 12 次 → 🔴 紅燈，模型嚴重失敗，必須修正

適用時機：這是銀行業的標準，散戶不需要算資本乘數，但可以借鑑這個框架——用例外次數判斷你的 VaR 模型到底靠不靠譜。如果你用 95% VaR，那 250 天回測中例外應該在 12-13 次左右。超過 20 次就是紅燈了。

四、Kupiec POF 檢定（比例失敗檢定）

是什麼：Kupiec（1995）提出的概似比檢定，正式檢驗例外的觀察比例是否與 VaR 信心水準隱含的比例一致。比單純計算 z 值更嚴謹。

核心邏輯：

  • 虛無假設 H₀：模型正確（例外比例 = p）
  • 對立假設 H₁：模型不正確（例外比例 ≠ p）
  • 如果概似比超過臨界值 → 拒絕 H₀ → 模型有問題

POF 統計量公式：
> LR_POF = -2 × ln[(1-p)^(N-x) × p^x / (1-x/N)^(N-x) × (x/N)^x]

其中：

  • N = 樣本天數
  • x = 例外次數
  • p = 1 - 信心水準

判斷：

  • LR_POF 服從自由度 1 的卡方分佈
  • 95% 臨界值 = 3.84
  • 99% 臨界值 = 6.63
  • LR_POF > 3.84 → 95% 信心下拒絕模型
  • LR_POF > 6.63 → 99% 信心下拒絕模型

實例：

  • 250 天回測，95% VaR（p = 5%）
  • 觀察到 18 次例外
  • 期望例外 = 250 × 5% = 12.5 次
  • 觀察比例 = 18/250 = 7.2%（高於 5%）
  • 計算 LR_POF...如果結果 > 3.84 → 拒絕模型
  • 結論：模型低估風險，需要調整

五、Christoffersen 條件涵蓋檢定

是什麼：前面的 POF 檢定只看「總共多少次例外」，不管例外是不是集中在某段時間。Christoffersen 檢定額外檢查「例外是否獨立」——如果例外會聚集（一次例外後隔天又例外），那模型有問題。

核心邏輯：

  • 好的 VaR 模型，例外應該是隨機分散的
  • 如果例外聚集 → 模型對波動聚集的處理不好
  • 波動聚集 = 大跌之後更容易大跌（GARCH 效應）

怎麼做：
1. 建立轉移矩陣：

  • n00 = 沒例外→沒例外 的天數

2. 計算獨立性統計量 LR_CCI
3. LR_CCI 服從自由度 1 的卡方分佈
4. 可以與 POF 結合：LR_CC = LR_POF + LR_CCI（自由度 2）

實例：

  • 250 天回測中，出現了 12 次例外
  • 但其中 8 次集中在某 2 週內
  • n11 = 6（連續例外），n10 = 2
  • 這代表例外不是獨立的 → 模型對波動聚集處理不好
  • 即使 POF 可能通過，CCI 也會失敗

適用時機：如果你用歷史模擬法或變異數法算 VaR，這兩種方法都假設例外是獨立的。但真實市場有波動聚集效應，例外會成群出現。這時候 Christoffersen 檢定就能抓出問題。

六、Type I 與 Type II 錯誤

是什麼：任何統計檢定都有兩種犯錯的可能。

錯誤類型定義後果
**Type I 錯誤（α）**拒絕一個正確的模型白忙一場，改了一個其實沒壞的模型
**Type II 錯誤（β）**接受一個錯誤的模型繼續用一個低估風險的模型，可能賠大錢

散戶該怎麼想：

  • Type I 錯誤 → 你太緊張了，以為模型壞了但其實沒壞。代價是重做模型，花時間
  • Type II 錯誤 → 你太鬆懈了，以為模型沒壞但其實壞了。代價是虧大錢
  • 對散戶來說，Type II 錯誤的代價遠高於 Type I
  • 所以寧可多拒絕幾次（稍微保守），也不要漏掉壞模型

實例：

  • 95% 信心 VaR 回測 250 天，出現 18 次例外
  • 期望 12.5 次，看起來偏多
  • 如果選擇較寬鬆的拒絕標準（比如 99% 顯著水準才拒絕）→ 可能不拒絕 → 但其實模型低估風險 → Type II 錯誤
  • 如果選擇較嚴格的拒絕標準（95% 顯著水準就拒絕）→ 可能拒絕 → 但如果只是運氣差 → Type I 錯誤
  • 散戶建議用 95% 顯著水準，寧可 Type I 也不要 Type II

七、散戶 VaR 回測實戰 SOP

最簡版本：例外次數計算法

1. 準備資料：

  • 過去 250 個交易日的 VaR 預測值

2. 計算例外：

  • 每天比較：如果實際虧損 > VaR → 記一次例外

3. 判斷模型：

  • 95% VaR：期望 5% 例外 = 250 × 5% = 12.5 次

4. 調整模型：

  • 例外太多 → VaR 估計太低 → 加大波動度假設或提高信心水準

進階版本：巴塞爾紅綠燈法

1. 使用 99% 信心水準 VaR
2. 回測 250 天
3. 0-4 次例外 = 🟢，5-9 次 = 🟡，≥10 次 = 🔴
4. 黃燈→調查原因，紅燈→換模型

每月例行檢查

1. 更新回測數據（加入新月份，捨棄最舊月份）
2. 重新計算例外次數
3. 檢查例外是否有聚集現象
4. 如果連續 2 個月黃燈 → 升級為紅燈處理

八、VaR 回測的三大困難

1. 靜態組合 vs 動態組合

VaR 假設你的部位不變，但實際上你每天都在買賣。

  • 解法：用「假設性報酬」（Hypothetical Return）= 如果部位完全不動的報酬
  • 另外也看「清淨報酬」（Clean Return）= 扣掉手續費、非市價項目的報酬

2. 樣本不具代表性

回測期間可能剛好是太平盛世或剛好是金融風暴，都不能代表長期。

  • 解法：至少用 1 年以上的數據，最好包含多空循環

3. 統計檢定的根本限制

250 天的回測，統計檢定力（Power）其實不高。

  • 可能一個壞模型因為「運氣好」而通過檢定（Type II 錯誤）
  • 解法：用多種檢定方法交叉驗證，不要只依賴單一測試

心得

VaR 算出來只是第一步，驗證才是重點。這就像你買了保險，從來不理賠，你也分不清是保險公司服務好還是你運氣好。回測就是在區分這兩者。

最實用的三個層次：
1. 最簡單：數例外次數。95% VaR 一年 252 天應該約 12-13 次例外，太多太少都有問題。這連 Excel 都不用，手算就行。
2. 進階：用巴塞爾紅綠燈。雖然是銀行業的標準，但散戶可以直接套用——99% VaR 回測 250 天，0-4 次🟢，5-9 次🟡，10+ 次🔴。直覺又好用。
3. 最嚴謹：Kupiec POF + Christoffersen CCI。前者檢驗頻率，後者檢驗獨立性。但對散戶來說，z 統計量 + 紅綠燈就夠用了。

讓我警惕的是 Type I vs Type II 錯誤的取捨。散戶最容易犯的錯是「相信一個壞模型」（Type II），因為壞模型通常告訴你風險很低，讓你敢下更大部位——然後一次黑天鵝就把你幹掉。所以寧可多拒絕幾次模型，也不要對壞模型太寬容。

另一個重點是「例外聚集」。12 次例外如果分散在 12 個月份，那可能是統計波動。但如果 12 次集中在 2 週內，那代表模型對波動聚集完全沒轍。這種情況下，VaR 模型需要加入 GARCH 或其他條件波動模型，而不是單純用歷史模擬法。

核心收穫：VaR 不是算完就放著的數字，而是需要持續驗證的工具。 每個月跑一次回測，就像每個月體檢一次——你可能覺得沒必要，但等你發現問題的時候，通常已經來不及了。
