回測（Backtesting）基礎

是什麼：用歷史數據模擬策略在過去的表現，驗證策略是否有效的方法。

為什麼重要：
1. 驗證策略可行性 - 看策略在過去是否能獲利
2. 風險評估 - 了解最大虧損、波動性等風險指標
3. 參數優化 - 調整參數提高勝率
4. 排除主觀干擾 - 基於數據而非情緒做決策

回測四大用途：

  • 績效預期：預估可獲得的報酬和風險
  • 風險控管：了解最大回撤、波動性
  • 參數調整：優化策略參數
  • 客觀驗證：排除人為主觀意識

過擬合（Overfitting）陷阱

是什麼：策略對歷史數據「過度最佳化」，結果是優化雜訊而非真正的價格規律。

比喻理解：

  • 中樂透的人很成功，但成功來自運氣而非能力
  • 參數調太多，就像買越多彩券，總會中獎
  • 歷史表現好 ≠ 未來表現好

過擬合的徵兆：
1. 歷史績效極佳，但實際交易虧損
2. 參數太針對特定時期優化
3. 策略太複雜、條件太多

怎麼做：避免過擬合的方法

Hold-Out 驗證法（樣本內外切分）

是什麼：將歷史資料分成「樣本內（IS）」和「樣本外（OOS）」

怎麼做：
1. 用 IS 數據優化參數
2. 用 OOS 數據驗證績效
3. 如果 IS 最佳參數在 OOS 表現也前面 → 策略有效

缺點：

  • IS 和 OOS 人為定義，優化多次後會對 OOS 越來越熟悉
  • 切分資料會減少可用數據量
  • OOS 通常是近期資料，卻不能用來開發

實例：

  • 樣本內：2015-2020 數據，找到最佳均線參數（20日、60日）
  • 樣本外：2021-2023 數據，驗證這組參數是否仍有效
  • 如果樣本外績效明顯變差 → 可能過擬合

交叉驗證法（CSCV）

是什麼：更科學的驗證方法，產生多重 IS/OOS 組合

怎麼做：
1. 將績效時間序列切成 S 份
2. 任選 S/2 份當 IS，其他當 OOS
3. 產生大量組合（C(S, S/2) 種）
4. 觀察「最佳參數」在 OOS 的排名分布

判斷標準：

  • 過擬合機率 = 最佳參數在 OOS 表現低於中位數的比例
  • 好策略：過擬合機率 < 10%
  • 壞策略：過擬合機率 > 70%

實例：

  • 100 個樣本中，74 個最佳化後效果低於中位數 → 74% 過擬合風險（壞策略）
  • 100 個樣本中，只有 4 個低於中位數 → 4% 過擬合風險（好策略）

回測核心指標

是什麼：評估策略績效的關鍵數據

怎麼看：

指標意義判斷標準
淨利總報酬金額要扣掉手續費再看
勝率獲利交易比例不如賺賠比重要
盈虧比平均獲利/平均虧損> 1 才有正期望值
最大虧損（MDD）歷史最大虧損幅度不能超過心理承受範圍
夏普比率報酬/波動比越高代表波動越小
交易次數樣本是否足夠太少可能有運氣成分

實例：

  • 策略 A：勝率 60%，盈虧比 0.8 → 期望值 = 0.6×0.8 - 0.4×1 = 0.08（正但小）
  • 策略 B：勝率 40%，盈虧比 2 → 期望值 = 0.4×2 - 0.6×1 = 0.2（更高）

適用時機：比較不同策略時，不要只看勝率或淨利，要看完整指標

TradingView 回測操作步驟

是什麼：TradingView 提供的免費回測工具

怎麼做：

1. 選擇商品：搜尋股票/期貨代碼
2. 設定時間週期：日線、小時線等
3. 載入策略：

  • 內建策略：策略測試器 → 技術 → 選擇策略

4. 調整參數：修改均線天數、停損點等
5. 設定成本：手續費、滑價（建議先設 0，再逐步加入）
6. 查看結果：

  • 概要：淨利、勝率、MDD

實例：

  • 商品：SPX（S&P 500 指數）
  • 策略：雙均線交叉（20MA/60MA）
  • 時間：日線
  • 結果：458 次交易，淨利 0.08%
  • 判斷：交易次數多但報酬低，加入手續費後可能虧損

注意事項：

  • 免費版限制 5000 根 K 線
  • 不能上傳自己的歷史數據
  • 一次只能測一個市場

回測三大陷阱

是什麼：常見的回測錯誤，導致結果失真

怎麼避免：

陷阱問題解決方法
未扣成本手續費、滑價未計入設定實際交易成本
未來函數用當時不知道的數據確保只用當下可得資訊
存活偏差只測還存在的股票包含已下市的股票

實例：

  • 收盤突破進場 → 當天收盤才知道突破，無法在收盤前進場
  • 正確做法：收盤確認突破，隔天開盤進場

指標 vs 策略差異

是什麼：TradingView 中兩種不同功能的工具

項目指標（Indicator）策略（Strategy）
功能畫圖、標記輔助線模擬交易進出場
產出視覺化數據績效報告
回測❌ 不能✅ 可以
用途分析判斷驗證策略有效性

實例：

  • 指標：畫出 KD 線、均線交叉點
  • 策略：KD 黃金交叉買進、死亡交叉賣出 + 回測績效

心得

回測不是為了找到「歷史最好的參數」，而是驗證「策略邏輯是否真的有效」。

最關鍵的概念是過擬合檢測 — 很多策略歷史績效漂亮，實際用卻虧損，原因就是優化了雜訊。FinLab 那篇用「多重 IS/OOS 切分」的方法很實用，比單純切分樣本內外更嚴謹。

實戰建議：
1. 先用簡單策略跑回測，確認基準線
2. 加入手續費和滑價，看績效還剩多少
3. 用不同時間區間驗證，避免只測牛市
4. 關鍵指標：盈虧比 > 勝率，MDD < 心理承受範圍

TradingView 回測方便但不精確，真要跑策略還是要用專業回測軟體，但作為初步驗證已經足夠。
